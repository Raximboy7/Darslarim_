Performance Nima va Nega Muhim?

Performance (unumdorlik) — bu dasturingiz qanchalik tez ishlashi va qancha xotira ishlatishidir. Kichik dasturda farq sezilmaydi, lekin million elementli ma'lumotda sekin kod soatlab kutishga, tez kod esa soniyalarga aylanadi.
Eng muhim qoida: Avval o'lcha!

Optimizatsiyani taxmin bilan boshlamang. Avval profiling bilan qaysi joy sekin ekanini aniqlang. Aks holda muhim bo'lmagan joyni bekorga optimallashtirasiz.

1. To'g'ri Data Structure Tanlash

# Million elementda set yuzlab marta tezroq

2. Generator Bilan Xotira Tejash

List hamma elementni bir vaqtda xotirada saqlaydi. Generator esa elementlarni bittalab, kerak bo'lganda yaratadi — shuning uchun deyarli xotira ishlatmaydi.
# List: 10 million son xotirada (sekin, og'ir)
sonlar = [x * 2 for x in range(10_000_000)]

# Generator: bittalab yaratadi (yengil)
sonlar = (x * 2 for x in range(10_000_000))
# () qavs — generator, [] qavs — list
Qachon generator?

Agar siz ma'lumotni faqat bir marta, ketma-ket aylanib chiqsangiz — generator ishlating. Agar bir necha marta kerak bo'lsa yoki indeks bilan murojaat qilsangiz — list kerak.

3. Caching — functools.lru_cache

Caching (keshlash) — bir marta hisoblangan natijani eslab qolish. Funksiya bir xil argument bilan yana chaqirilsa, qaytadan hisoblamay, saqlangan javobni qaytaradi.
from functools import lru_cache

@lru_cache
def fib(n):
    if n < 2:
        return n
    return fib(n-1) + fib(n-2)
# @lru_cache javoblarni eslab qoladi — qayta hisoblamaydi
>>> fib(35) # keshsiz: ~3 soniya, kesh bilan: ~0.0001 soniya

4. List Comprehension vs Oddiy Sikl

Ro'yxat yaratishda list comprehension odatda oddiy for + append dan tezroq, chunki u ichki darajada (C tilida) optimallashtirilgan.
# Sekinroq: har safar .append() chaqiriladi
natija = []
for x in range(1000):
    natija.append(x * x)

# Tezroq: list comprehension
natija = [x * x for x in range(1000)]
# Bir xil natija, lekin comprehension tezroq va qisqaroq

5. Lokalizatsiya — Local O'zgaruvchi Tez

Python local (funksiya ichidagi) o'zgaruvchini global dan tezroq topadi. Sikl ichida ko'p ishlatadigan funksiyani local o'zgaruvchiga saqlash tezlikni oshiradi.
# Sekinroq: har aylanishda append qidiriladi
for x in data:
    natija.append(x)

# Tezroq: metodni local o'zgaruvchiga oldik
qosh = natija.append
for x in data:
    qosh(x)
# Katta sikllarda local nom qidiruvi tezroq

6. Built-in Funksiyalar Tez

sum(), map(), min(), max() kabi built-in funksiyalar C tilida yozilgan — ular qo'lda yozilgan sikldan tezroq.
# Sekinroq: qo'lda yig'indi
jami = 0
for x in sonlar:
    jami += x

# Tezroq: built-in sum()
jami = sum(sonlar)
# Built-in funksiya har doim afzal — tez va o'qishli

7. Profiling Bilan O'lchash

Kodingiz qancha vaqt olishini o'lchang. Eng oddiy yo'l — time moduli; aniqroq tahlil uchun timeit va cProfile.
import time

boshi = time.perf_counter()
natija = sum(range(10_000_000))
oxiri = time.perf_counter()

print(f"Vaqt: {oxiri - boshi:.4f} soniya")
# perf_counter() — eng aniq vaqt o'lchagich

Misol 1: list vs set Qidiruv Tezligi

Element soni qancha ko'p bo'lsa, list shuncha sekinlashadi. set esa doim deyarli bir xil tez qoladi. Sinab ko'ring.

Misol 2: lru_cache Bilan / Holda

Fibonachchi sonini keshsiz hisoblash juda ko'p takror amal qiladi. @lru_cache bilan har son faqat bir marta hisoblanadi.

Misol 3: Optimizatsiya — Oldin / Keyin

Bir xil masala (sonlar yig'indisi) ikki xil yo'l bilan: qo'lda sikl (oldin) va built-in sum() (keyin). Tezlik farqini ko'ring.
