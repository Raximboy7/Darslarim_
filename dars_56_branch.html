Branch (Tarmoqlar)

Modul 10 — Git va Version Control | Dars 56

Branch (Tarmoq) Nima?

Branch (tarmoq) — bu loyihangizning parallel ish yo'nalishi. U asosiy kodni buzmasdan yangi narsalarni sinab ko'rish imkonini beradi. Tarmoqda ishlaganingizda, siz asosiy koddan alohida nusxada ishlaysiz — xato qilsangiz ham, asosiy kod butun va xavfsiz qoladi.
Buni daraxtga o'xshatish mumkin: asosiy tana — bu main tarmog'i, undan chiqqan shoxchalar esa — yangi xususiyatlar ustida ishlaydigan tarmoqlar. Ish tugagach, shoxchani yana asosiy tanaga qaytarib qo'shasiz.

Nega Branch Kerak?

Tasavvur qiling: dasturingiz ishlayapti, lekin siz yangi xususiyat qo'shmoqchisiz. Agar to'g'ridan-to'g'ri asosiy kodda ishlasangiz va xato qilsangiz — butun dastur ishdan chiqishi mumkin. Branch buni hal qiladi.
✓
Xavfsiz tajriba

Yangi g'oyalarni asosiy kodni buzmasdan sinab ko'rasiz

✓
Alohida xususiyatlar

Har bir yangi xususiyatni o'z tarmog'ida ishlab chiqasiz

✓
Jamoa bilan ishlash

Har bir dasturchi o'z tarmog'ida ishlaydi, bir-biriga xalaqit bermaydi

✓
Tayyor bo'lganda qo'shish

Xususiyat tayyor bo'lgach, uni asosiy kodga birlashtirasiz

O'qish tartibi

C2 da yangi feature tarmog'i ochildi → unda C3 va C4 commitlar qilindi → ish tugagach M nuqtada main ga birlashtirildi (merge).

git branch — Tarmoqlar Ro'yxati va Yaratish

git branch buyrug'i barcha tarmoqlarni ko'rsatadi. Joriy (hozir turgan) tarmog'ingiz oldida * belgisi turadi.
git branch
git branch yangi-xususiyat
# 1-qator: tarmoqlar ro'yxatini ko'rsatadi
# 2-qator: "yangi-xususiyat" nomli yangi tarmoq yaratadi
>>> git branch
* main
  yangi-xususiyat

git checkout / git switch — Tarmoqqa O'tish

Tarmoq yaratganingizdan keyin unga o'tish kerak. Buning ikki yo'li bor: eski git checkout va yangi, qulayroq git switch.
git switch yangi-xususiyat
git switch -c boshqa-tarmoq
# 1-qator: "yangi-xususiyat" tarmog'iga o'tadi
# 2-qator: -c yangi tarmoq YARATADI va darrov unga O'TADI
git checkout yangi-xususiyat
git checkout -b boshqa-tarmoq
# Eski usul: switch bilan bir xil ishni bajaradi
# -b yangi tarmoq yaratib, unga o'tadi
Maslahat

Yangi loyihalarda git switch dan foydalaning — u faqat tarmoqlar uchun ishlaydi va chalkashlik kamroq bo'ladi.

git merge — Tarmoqni Birlashtirish

Tarmoqdagi ish tugagach, uni asosiy tarmoqqa qaytarib qo'shasiz. Bu jarayon merge (birlashtirish) deyiladi. Avval asosiy tarmoqqa o'ting, keyin ustiga boshqa tarmoqni birlashtiring.
git switch main
git merge yangi-xususiyat
# 1-qator: avval main (asosiy) tarmoqqa o'tamiz
# 2-qator: "yangi-xususiyat" dagi o'zgarishlarni main ga qo'shamiz
>>> git merge yangi-xususiyat
Updating a1b2c3d..e4f5g6h
Fast-forward
  app.py | 12 ++++++++++++

main / master Tarmoq

Har bir Git loyihasida bitta asosiy tarmoq bo'ladi. Avval u master deb nomlanardi, hozir esa ko'pincha main deyiladi. Bu — loyihaning ishonchli, asosiy versiyasi. Yangi xususiyatlar boshqa tarmoqlarda ishlanib, tayyor bo'lgach main ga birlashtiriladi.
BuyruqVazifasi
git branchTarmoqlar ro'yxatini ko'rsatadi
git branch nomYangi tarmoq yaratadi
git switch nomTarmoqqa o'tadi
git switch -c nomYaratadi va o'tadi
git merge nomTarmoqni joriy tarmoqqa birlashtiradi
git branch -d nomTarmoqni o'chiradi

Merge Conflict (Ziddiyat) Nima?

Ba'zan ikki tarmoqda bir xil qatorni turlicha o'zgartirsangiz, Git qaysi birini olishni bilmay qoladi. Bu — merge conflict (ziddiyat). Bunda Git sizdan qaysi versiyani saqlashni so'raydi.
<<<<<<< HEAD
salom = "Salom dunyo"
=======
salom = "Assalomu alaykum"
>>>>>>> yangi-xususiyat
# HEAD = joriy tarmoqdagi versiya
# pastdagi = qo'shilayotgan tarmoqdagi versiya
# Siz keraksiz qatorni o'chirib, birini qoldirasiz
Qo'rqmang

Conflict — bu xato emas, oddiy holat. Siz shunchaki to'g'ri versiyani tanlab, belgilarni (<<<, ===, >>>) o'chirasiz, so'ng faylni saqlab yana commit qilasiz.
