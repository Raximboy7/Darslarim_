Deployment (Joylashtirish)

Modul 13 — Web Asoslari | Dars 89

Deployment Nima?

Deployment (joylashtirish) — bu sizning loyihangizni o'z kompyuteringizdan olib, internetga chiqarish jarayoni. Shundan keyin dunyoning istalgan joyidagi odam brauzer orqali sizning saytingizni ocha oladi.

Hozirgacha loyihangiz faqat sizning kompyuteringizda — localhost da ishlardi. Deployment — uni haqiqiy serverga ko'chirish demakdir.

Local vs Production

Dasturchilar ikki muhit (environment) bilan ishlaydi:
• Local — o'z kompyuteringiz, sinab ko'rish uchun.
• Production — haqiqiy server, foydalanuvchilar uchun.

Muhim qoida

Production muhitda debug=True ni hech qachon qoldirmang! U xatolarni hammaga ko'rsatadi va xakerlarga kodingiz haqida ma'lumot beradi.

Hosting — Saytni Qayerga Joylash?

Hosting — bu loyihangizni saqlaydigan va internetga uzatadigan kompyuter (server). O'z serveringizni sotib olish shart emas — tayyor xizmatlardan foydalanasiz. Ko'pchiligida boshlash uchun bepul rejasi bor.
XizmatTavsifi
PythonAnywherePython uchun eng sodda, boshlovchilarga juda qulay
RenderBepul reja bor, Flask loyihalarni oson joylaydi
HerokuMashhur va g'oyasi sodda: kodni "push" qilasiz, ishga tushadi
RailwayTez sozlash, zamonaviy interfeys
Asosiy g'oya

Hammasida bir xil g'oya: kodingizni serverga yuborasiz (masalan git push orqali), server uni o'rnatadi va sizga manzil (URL) beradi. Tafsilotlari farq qiladi, lekin mantiq bir xil.

requirements.txt

Sizning loyihangiz qaysi kutubxonalarga (Flask, requests va h.k.) muhtoj ekanini server bilishi kerak. Buni requirements.txt faylida ro'yxat qilamiz. Server uni o'qib, barcha kutubxonalarni o'zi o'rnatadi.
# requirements.txt fayli ichida:
Flask==3.0.0
requests==2.31.0
gunicorn==21.2.0
# Har bir qatorda: kutubxona nomi va versiyasi
# Bu faylni avtomatik yaratish (terminalda):
pip freeze > requirements.txt
# pip freeze o'rnatilgan barcha kutubxonalarni ro'yxatlaydi

Environment Variables (.env)

Parol, maxfiy kalit (secret key), API token kabi maxfiy ma'lumotlarni hech qachon to'g'ridan-to'g'ri kodga yozmang! Ular .env fayliga yoki serverning sozlamalariga yashiriladi va kod ulardan environment variable (muhit o'zgaruvchisi) sifatida o'qiydi.
# .env fayli (bu fayl GitHubga yuklanmaydi!):
SECRET_KEY=mening_maxfiy_kalitim
DATABASE_URL=postgres://...
# Maxfiy qiymatlar shu yerda saqlanadi
import os

# Maxfiy kalitni muhitdan o'qish:
kalit = os.environ.get("SECRET_KEY")
print(kalit)
# os.environ.get() — muhit o'zgaruvchisini xavfsiz o'qiydi
>>> mening_maxfiy_kalitim
Nega bu muhim?

Agar parolni kodga yozsangiz va kodni GitHubga yuklasangiz — parolingizni hamma ko'radi! .env faylini esa .gitignore ga qo'shib, yuklanmasligini ta'minlaysiz.

Error Logging (Xatolarni Yozib Borish)

Production muhitda debug=False bo'lgani uchun xatolar ekranda ko'rinmaydi. Lekin xato bo'lganda biz buni bilishimiz kerak! Shuning uchun xatolarni log fayliga yozib boramiz.
import logging

logging.basicConfig(filename="xatolar.log", level=logging.ERROR)

try:
    natija = 10 / 0
except Exception as e:
    logging.error("Xato yuz berdi: " + str(e))
# Xato "xatolar.log" fayliga yoziladi, dastur to'xtamaydi
>>> xatolar.log ichiga: ERROR Xato yuz berdi: division by zero

debug=False — Production Sozlamasi

Flask loyihangizni serverga joylashdan oldin debug ni o'chiring. Eng yaxshi usul — uni muhit o'zgaruvchisidan o'qish.
import os

# Local da DEBUG=True, serverda DEBUG yo'q -> False
debug_holati = os.environ.get("DEBUG") == "True"

app.run(debug=debug_holati)
# Serverda DEBUG sozlanmagani uchun avtomatik False bo'ladi — xavfsiz
Eslab Qoling

• Local — sinash, Production — ishlatish
• requirements.txt — kutubxonalar ro'yxati
• Maxfiy kalitlar .env da, kodda emas
• os.environ.get() — muhitdan o'qiydi
• Production da debug=False
• Xatolarni logging bilan yozib boring
