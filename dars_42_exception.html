Exception Handling (try-except)

Modul 8 — Xatolar va Test | Dars 42

Xato (Exception) Nima?

Exception (xato) — bu dastur ishlayotganda yuzaga keladigan kutilmagan holat. Masalan, foydalanuvchi raqam o'rniga matn kiritsa yoki biror sonni nolga bo'lmoqchi bo'lsangiz — Python xatoga uchraydi. Agar bu xatoni tutmasangiz, dastur shu yerda qulaydi (to'xtab qoladi) va keyingi qatorlar umuman bajarilmaydi.
son = int("salom")
print("Bu qator bajarilmaydi")
# "salom" ni songa aylantirib bo'lmaydi → dastur quladi
>>> ValueError: invalid literal for int() with base 10: 'salom'

try-except Bloki

Xatoni tutish uchun kodni try blokiga o'raymiz. Agar shu blok ichida xato chiqsa, Python dasturni to'xtatmaydi, balki except blokini bajaradi.
try:
    son = int("salom")
except:
    print("Xato yuz berdi!")
# try: xavfli kod shu yerda
# except: xato bo'lsa shu blok ishlaydi → dastur qulamaydi
>>> Xato yuz berdi!
Asosiy g'oya

try ichidagi kod muvaffaqiyatli bo'lsa — except umuman ishlamaydi.
try ichida xato chiqsa — Python darhol except blokiga sakraydi.

Aniq Xato Turini Tutish

Hamma xatoni bir xil tutish o'rniga, aniq turini ko'rsatish yaxshiroq. Masalan except ValueError: — faqat noto'g'ri qiymat xatosini tutadi. Bu sizga qaysi xato bo'lganini aniq bilish imkonini beradi.
try:
    son = int(input("Raqam: "))
except ValueError:
    print("Iltimos, butun son kiriting!")
# Faqat ValueError tutiladi (matn kiritilsa)
# int(input()) ni try-except bilan o'rash — eng keng tarqalgan misol

Bir Nechta except Bloki

Bitta try uchun bir nechta except yozish mumkin — har biri boshqa xato turi uchun. Python qaysi xato chiqqaniga qarab mos blokni tanlaydi.
try:
    a = int(input("Son: "))
    natija = 10 / a
except ValueError:
    print("Bu son emas!")
except ZeroDivisionError:
    print("Nolga bo'lib bo'lmaydi!")
# Matn kiritilsa → ValueError bloki
# 0 kiritilsa → ZeroDivisionError bloki

else va finally Bloklari

else — try ichida xato bo'lmasa bajariladi. finally — xato bo'ladimi yoki yo'qmi, har doim bajariladi (masalan faylni yopish uchun).
try:
    son = int("5")
except ValueError:
    print("Xato!")
else:
    print("Hammasi joyida:", son)
finally:
    print("Tekshiruv tugadi")
# else: xato bo'lmadi → ishladi
# finally: har qanday holatda ham ishlaydi
>>> Hammasi joyida: 5
>>> Tekshiruv tugadi

Bloklar Tartibi

BlokQachon ishlaydi
tryDoim — xavfli kod shu yerda turadi
exceptFaqat try ichida xato chiqsa
elseFaqat try ichida xato bo'lmasa
finallyHar doim — xato bo'lsa ham, bo'lmasa ham
Nega Xatolarni Tutish Muhim?

• Dastur kutilmaganda qulamaydi — foydalanuvchi xafa bo'lmaydi
• Foydalanuvchiga tushunarli xabar ko'rsatasiz ("Raqam kiriting!")
• Dastur xatodan keyin ham ishlashda davom etadi
• Faqat kerakli xatoni tuting — boshqa noma'lum xatolarni yashirib qo'ymang

Misol 1: Raqamni Tekshirish (try-except oqimi)

To'g'ri raqam kiriting → natija; harf yoki bo'sh kiriting → except ishlaydi.

Misol 2: Nolga Bo'lishni Tutish

0 kiriting → ZeroDivisionError tutiladi; boshqa son → natija chiqadi.

Misol 3: else va finally Qachon Ishlaydi

To'g'ri raqam → except o'tkazib yuboriladi, else ishlaydi. Harf → except ishlaydi, else o'tkaziladi. finally har doim chiqadi.
