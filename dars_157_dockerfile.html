Image va Dockerfile

Modul 22 — DevOps (Docker va CI/CD) | Dars 157

Image va Dockerfile Nima?

O'tgan darsda image — bu ilovani ishga tushirish uchun tayyor "qadoq" ekanini ko'rdik. Dockerfile esa shu image qanday yasalishini tushuntiruvchi retsept (matn fayli). Unda qaysi tizimdan boshlash, qaysi fayllarni qo'shish va qaysi buyruqni ishga tushirish yozib qo'yiladi.
Sodda taqqoslash

🍳 Dockerfile — bu retsept (qog'ozdagi yo'riqnoma)
📦 Image — retsept bo'yicha pishirilgan tayyor taom
🚀 Container — dasturxonga qo'yilgan, ishlab turgan taom

Demak oqim shunday: Dockerfile → docker build → Image → docker run → Container.

Asosiy Buyruqlar

Dockerfile ichidagi har bir qator bitta buyruqdan boshlanadi. Buyruq nomlari odatda KATTA harf bilan yoziladi.
BuyruqVazifasiMisol
FROMAsos image (qaysi tizimdan boshlanadi)FROM python:3.12
WORKDIRIshchi papkani belgilaydiWORKDIR /app
COPYFayllarni image ichiga ko'chiradiCOPY . .
RUNQurish paytida buyruq bajaradiRUN pip install flask
EXPOSEQaysi portni ochishini bildiradiEXPOSE 5000
CMDKonteyner ishga tushganda nima bajariladiCMD ["python", "app.py"]
RUN va CMD farqi

• RUN — image qurilish paytida bir marta bajariladi (masalan, kutubxona o'rnatish).
• CMD — konteyner ishga tushganda bajariladi (asosiy ilovani ishga tushiradi).

Python Ilova Uchun Dockerfile

Quyida oddiy Python ilovasi uchun to'liq Dockerfile. Har bir qatorni izoh bilan o'qing.
FROM python:3.12-slim  # asos image
WORKDIR /app  # ishchi papka
COPY requirements.txt .  # avval ro'yxat
RUN pip install -r requirements.txt
COPY . .  # qolgan fayllar
EXPOSE 5000
CMD ["python", "app.py"]
# 1) Python o'rnatilgan tizimdan boshlaymiz
# 2) /app papkasiga o'tamiz
# 3) Avval requirements.txt ni ko'chiramiz va kutubxonalarni o'rnatamiz
# 4) Keyin barcha kodni ko'chiramiz
# 5) 5000-portni ochamiz va app.py ni ishga tushiramiz
Nega avval requirements.txt?

Avval kutubxonalar ro'yxatini ko'chirib o'rnatsak, kod o'zgarganda Docker bu qatlamni qayta ishlatadi (cache) va qayta o'rnatmaydi. Bu qurishni tezlashtiradi.

Image Qurish: docker build

Dockerfile tayyor bo'lgach, undan image yasash uchun docker build buyrug'idan foydalanamiz.
docker build -t myapp .
# -t myapp → image nomi "myapp"
# . (nuqta) → Dockerfile shu papkada (joriy papka)
>>> [+] Building 12.4s
>>> => [1/5] FROM python:3.12-slim
>>> => [2/5] WORKDIR /app
>>> => [3/5] COPY requirements.txt .
>>> => [4/5] RUN pip install -r requirements.txt
>>> => [5/5] COPY . .
>>> Successfully tagged myapp:latest
Qurib bo'lgach ilovani ishga tushiramiz: docker run -p 5000:5000 myapp

.dockerignore Fayli

COPY . . hamma narsani ko'chiradi. Lekin ba'zi fayllar image ichiga kerak emas (ular faqat joyni egallaydi). Ularni .dockerignore fayliga yozsak, Docker e'tibor bermaydi.
__pycache__/
*.pyc
venv/
.git/
.env
# Bu fayllar image ichiga ko'chirilmaydi
# → image yengilroq va xavfsizroq bo'ladi

Image Qatlamlari (Layers)

Dockerfile dagi har bir buyruq alohida qatlam (layer) hosil qiladi. Image — bu qatlamlar to'plami. Agar bir qatlam o'zgarmagan bo'lsa, Docker uni qayta ishlatadi (cache) — shuning uchun ikkinchi qurish tezroq bo'ladi.
CMD ["python", "app.py"]
COPY . .
RUN pip install -r requirements.txt
COPY requirements.txt .
WORKDIR /app
FROM python:3.12-slim
Eslab Qoling

Tez-tez o'zgaradigan narsalarni (kod) pastga, kam o'zgaradiganlarni (kutubxona o'rnatish) tepaga qo'ying. Shunda cache ko'proq ishlaydi.

Misol 1: Dockerfile Buyruqlari Lug'ati

Buyruqni tanlang — u nima qilishini va misolini ko'rsatamiz.

Misol 2: docker build Oqimi

"Qurishni boshla" tugmasini bosing — Docker har bir qatlamni qadamma-qadam quradi.

Misol 3: Python Dockerfile Tuzuvchi

Ilova nomi, Python versiyasi va portni kiriting — tayyor Dockerfile hosil bo'ladi.

Misol 4: Cache Ishlaydimi?

Nima o'zgargan? Tanlang — Docker qaysi qatlamlarni cache dan oladi va qaysilarini qayta quradi, ko'ring.
