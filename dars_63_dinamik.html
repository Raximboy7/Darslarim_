Rekursiya va Dinamik Dasturlash

Modul 11 — Algoritmlar | Dars 63

Rekursiya — Qisqa Eslatma

Rekursiya — bu funksiyaning o'zini o'zi chaqirishidir. Katta masala har safar bir oz kichikroq masalaga aylanadi va oxir-oqibat eng oddiy holatga (bazaviy holatga) yetadi.
def faktorial(n):
    if n == 0:
        return 1
    return n * faktorial(n - 1)
# Bazaviy holat: n == 0 bo'lsa 1 qaytaradi
# Aks holda o'zini kichikroq n bilan chaqiradi
>>> faktorial(4)
>>> 24

Dinamik Dasturlash Nima?

Dinamik dasturlash (DP) — bu katta masalani kichik qism masalalarga bo'lib yechish usulidir. Asosiy g'oya: bir marta hisoblangan natijani saqlab qo'yamiz va keyin uni qayta hisoblamaymiz — saqlangan javobni o'qib olamiz.
DP qachon ishlaydi?

Masala bir xil qism masalalarga qayta-qayta bo'linganda. Masalan oddiy rekursiv Fibonacci bir xil qiymatni minglab marta qayta hisoblaydi — bu juda sekin. DP shu takror hisoblardan qutqaradi.

Memoization — Natijalarni Saqlash (top-down)

Memoization — rekursiyaga kesh (xotira lug'ati) qo'shamiz. Har bir hisoblangan natijani saqlaymiz; keyin yana so'ralsa, qayta hisoblamasdan keshdan o'qiymiz.
kesh = {}
def fib(n):
    if n < 2:
        return n
    if n in kesh:
        return kesh[n]
    kesh[n] = fib(n-1) + fib(n-2)
    return kesh[n]
# Avval keshni tekshiramiz
# Yangi natijani keshga yozib qo'yamiz
# Shunda har qiymat faqat BIR marta hisoblanadi
Tayyor vosita

Python'da @lru_cache dekoratori avtomatik memoization beradi: functools dan import qilinadi va funksiya ustiga yoziladi.

Tabulation — Jadval Bilan (bottom-up)

Tabulation — rekursiyasiz. Biz kichik qiymatlardan boshlab jadvalni (ro'yxatni) to'ldirib boramiz. Har bir keyingi qiymat oldingilaridan hisoblanadi.
def fib(n):
    jadval = [0, 1]
    for i in range(2, n+1):
        jadval.append(jadval[i-1] + jadval[i-2])
    return jadval[n]
# 0 va 1 dan boshlaymiz
# Har qadamda oxirgi ikkitasini qo'shamiz
# Jadval kichikdan kattaga to'ladi
>>> fib(10)
>>> 55

Fibonacci: Oddiy vs Memoization

Oddiy rekursiv Fibonacci har bir qiymatni qayta-qayta hisoblaydi — n kattalashganda hisoblar soni portlab ketadi. Memoization esa har qiymatni faqat bir marta hisoblaydi.

Xulosa

Oddiy usul deyarli ikki barobar tezroq o'sadi (eksponensial), memoization esa chiziqli (n ga mutanosib). Ana shu DP ning kuchi.

Misol 1: Oddiy vs Memoized Fibonacci

n kiriting va ikkala usul Fibonacci'ni hisoblash uchun necha marta funksiya chaqirilishini taqqoslang.

Misol 2: Memoization Keshi To'lib Borishi

fib(n) ni memoization bilan hisoblaganda kesh qanday to'lib borishini ko'ring. Yashil kataklar — keshda saqlangan qiymatlar.

Misol 3: Fibonacci Qiymatini Hisoblash

n ning Fibonacci qiymatini tez (tabulation) usulda hisoblang va butun ketma-ketlikni ko'ring.
Eslab Qoling

• Rekursiya — funksiyaning o'zini o'zi chaqirishi
• DP — katta masalani kichiklarga bo'lib, takror hisoblardan qochish
• Memoization — rekursiya + kesh (top-down)
• Tabulation — jadval bilan (bottom-up)
• DP sekin rekursiyani tez yechimga aylantiradi
