Context Manager (with)

Modul 9 — Iterator va Generator | Dars 52

with Bloki Nima?

with — bu resurs (fayl, ulanish, qulf) bilan ishlaganda uni avtomatik ochib va yopib beradigan maxsus blok. Siz faqat "ish"ni yozasiz — tozalashni Python o'zi bajaradi.
Buni boshqaruvchi narsa context manager (konteks menejeri) deyiladi. U ikki ishni qiladi: blok boshlanganda resursni tayyorlaydi, blok tugaganda (hatto xato chiqsa ham) uni tozalaydi.

Nega with Yaxshi?

Fayl ochsangiz, uni yopish kerak. Agar yopishni unutsangiz — ma'lumot saqlanmasligi yoki resurs band qolishi mumkin. with bu muammoni butunlay yo'q qiladi.

with bo'lmagan eski usul

f = open("fayl.txt")
data = f.read()
f.close()
# close() ni o'zingiz chaqirishingiz SHART. Unutsangiz — muammo.

with bilan yangi usul

with open("fayl.txt") as f:
    data = f.read()
# Blok tugashi bilan fayl AVTOMATIK yopiladi. close() kerak emas.
>>> Fayl o'zi yopildi, hech narsa unutilmaydi

Fayllar bilan with — Eslatma

Doim with ishlating

Fayl bilan ishlaganda har doim with open(...) as f: ko'rinishidan foydalaning. Bu eng ishonchli va xavfsiz usul. Blok ichida xato chiqsa ham, fayl baribir yopiladi.

with open("ism.txt", "w") as f:
    f.write("Salom dunyo")
# Yozdik — blok tugadi — fayl yopildi va saqlandi

O'z Context Manager'ingizni Yaratish

Klassga ikkita maxsus metod qo'shsangiz, uni with bilan ishlatish mumkin bo'ladi:
MetodQachon ishlaydiVazifasi
__enter__with boshlangandaResursni tayyorlaydi, ochadi
__exit__with tugagandaResursni tozalaydi, yopadi
class Eshik:
    def __enter__(self):
        print("Eshik ochildi")
        return self
    def __exit__(self, a, b, c):
        print("Eshik yopildi")

with Eshik() as e:
    print("Ichkaridamiz")
# __enter__ avval, keyin ich, oxirida __exit__ ishlaydi
>>> Eshik ochildi
>>> Ichkaridamiz
>>> Eshik yopildi
__exit__ ning argumentlari

__exit__(self, a, b, c) dagi a, b, c — agar blok ichida xato chiqsa, xato haqida ma'lumot bo'ladi. Xato bo'lmasa — uchchalasi None bo'ladi. Eng muhimi: xato bo'lsa ham __exit__ ishlaydi.

contextlib — Qisqa Yo'l

Butun klass yozmasdan, contextlib moduli yordamida generator orqali ham context manager yasash mumkin. yield dan oldingi qism — boshlanish, keyingi qism — tugash.
from contextlib import contextmanager

@contextmanager
def eshik():
    print("ochildi")
    yield
    print("yopildi")
# yield dan oldin — __enter__, keyin — __exit__ vazifasini bajaradi
Eslab Qoling

• with resursni avtomatik yopadi/tozalaydi
• Fayllar uchun har doim with open(...) as f: ishlating
• __enter__ — boshlanishda, __exit__ — tugaganda
• Xato chiqsa ham __exit__ baribir ishlaydi
• contextlib bilan qisqaroq yozish mumkin

Misol 1: with Ketma-ketligi

Tugmani bosing va __enter__ → ish → __exit__ qadamlari qanday tartibda ishlashini ko'ring.
1. __enter__() — resurs ochilmoqda
2. with bloki ichidagi ish bajarilmoqda
3. __exit__() — resurs yopilmoqda

Misol 2: with bilan / withsiz Taqqoslash

Faylni qo'lda yopish bilan, with orqali yopishni solishtiring.

Misol 3: Xato Bo'lsa Ham __exit__ Ishlaydi

Blok ichida xato chiqsa nima bo'ladi? __exit__ baribir ishga tushadi va resursni yopadi.
__enter__() — ochildi
blok ichidagi ish
__exit__() — yopildi
