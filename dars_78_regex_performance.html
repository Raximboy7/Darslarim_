Modul 12 — Regex va Matn | Dars 78

Regex Tezligi Nega Muhim?

Regex (regular expression) — matn ichidan namuna (pattern) qidirishning kuchli vositasi. Lekin u har doim ham tez emas. Kichik matnda farq sezilmaydi, ammo katta fayllarda yoki minglab qatorda qidirayotganda yomon yozilgan regex dasturni soniyalarga, ba'zan daqiqalarga sekinlashtirishi mumkin.
Asosiy g'oya

Regex qancha "ishlash" qilishini biz boshqaramiz. To'g'ri yozilgan pattern bir o'tishda topadi, yomon yozilgani esa bir necha marta orqaga qaytib, qayta-qayta tekshiradi. Bu darsda regexni tezroq qilishning yo'llarini o'rganamiz.

1. re.compile() — Oldindan Tayyorlash

Har safar re.search() chaqirsangiz, Python pattern'ni har gal qaytadan "tushunadi" (kompilyatsiya qiladi). Agar bitta pattern'ni ko'p marta ishlatsangiz, uni bir marta re.compile() bilan tayyorlab qo'ying — bu ancha tezroq.

Sekin usul — har gal qayta tayyorlaydi

import re
natijalar = []
for qator in matnlar:
    if re.search(r"\d+", qator): # har gal qayta tayyorlaydi
        natijalar.append(qator)
# Pattern har aylanishda qaytadan tahlil qilinadi — ortiqcha ish

Tez usul — bir marta tayyorla, ko'p marta ishlat

import re
pattern = re.compile(r"\d+") # bir marta tayyorlandi
natijalar = []
for qator in matnlar:
    if pattern.search(qator): # tayyor pattern'dan foydalanadi
        natijalar.append(qator)
# Pattern bir marta tayyorlanadi, sikl ichida faqat ishlatiladi — tezroq
Qachon foydali?

Bir pattern'ni sikl ichida yoki ko'p marta ishlatsangiz — re.compile() ishlating. Faqat bir marta ishlatsangiz, oddiy re.search() ham yetarli.

2. Ochko'z (Greedy) va Dangasa (Lazy) Kvantorlar

Kvantor * va + standart holatda ochko'z (greedy) bo'ladi — iloji boricha ko'proq belgini oladi. Ularga ? qo'shsangiz dangasa (lazy) bo'ladi — iloji boricha kamroq oladi.
KvantorTuriMa'nosi
.*Ochko'zIloji boricha ko'p ol
.*?DangasaIloji boricha kam ol
.+Ochko'zKamida bitta, ko'p ol
.+?DangasaKamida bitta, kam ol

Misol: <b> teglarini topish

matn = "<b>Salom</b> va <b>Dunyo</b>"

re.findall(r"<b>.*</b>", matn) # ochko'z
re.findall(r"<b>.*?</b>", matn) # dangasa
# Ochko'z — eng oxirgi </b> gacha hamma narsani oladi
>>> ochko'z: ['<b>Salom</b> va <b>Dunyo</b>']
>>> dangasa: ['<b>Salom</b>', '<b>Dunyo</b>']
Xulosa

Ochko'z .* avval butun matnni "yutadi", keyin orqaga qaytib mos joyni qidiradi. Dangasa .*? esa kamdan boshlab kerakli joyni topadi. Ko'pincha aniq natija uchun dangasa to'g'riroq.

3. Catastrophic Backtracking (Halokatli Orqaga Qaytish)

Ba'zi yomon pattern'lar matn mos kelmaganda juda ko'p variantni tekshirishga majbur bo'ladi. Har bir variant uchun orqaga qaytib qayta urinadi — bu catastrophic backtracking deyiladi va dasturni qotirib qo'yishi mumkin.

Yomon pattern misoli

# XAVFLI: ichma-ich takrorlanuvchi kvantor
pattern = r"(a+)+b"
matn = "aaaaaaaaaaaaaaaaaaaa" # oxirida 'b' yo'q

re.search(pattern, matn) # juda sekin!
# 'b' topilmaydi, regex 'a' larni har xil bo'lib ko'rishga urinadi — variantlar soni portlaydi
Nega sekinlashadi?

(a+)+ da bitta narsa (a lar) ikki marta takrorlanyapti. 20 ta "a" ni guruhlarga ajratishning million xil yo'li bor. "b" topilmagani uchun regex bularning hammasini sinab ko'radi. Belgilar ko'paygan sayin vaqt eksponensial oshadi.

# YAXSHI: oddiy va aniq
pattern = re.compile(r"a+b")
pattern.search("aaaaaaaaaaaaaaaaaaaa") # tez False qaytaradi
# Ichma-ich guruh yo'q — orqaga qaytish minimal, natija bir zumda

4. Ba'zan Oddiy String Metodi Tezroq

Regex kuchli, lekin har joyda kerak emas. Agar siz aniq bir so'zni qidirayotgan bo'lsangiz — regex emas, oddiy string metodi tezroq va o'qishga osonroq.
MaqsadRegex (sekinroq)String metodi (tezroq)

Almashtirishre.sub("a","o",s)s.replace("a","o")

matn = "Bugun havo issiq"

# Regex shart emas:

Qoida

Regexni faqat namuna (pattern) kerak bo'lganda ishlating — masalan "har qanday raqam", "elektron pochta shakli". Aniq matn uchun string metodlari yetarli va tezroq.
