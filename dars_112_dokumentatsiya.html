Dokumentatsiya Nima va Nega Muhim?

Dokumentatsiya — bu kodning nima qilishini va qanday ishlatilishini tushuntiruvchi yozma izohlar. Kod faqat kompyuter uchun emas, balki odamlar uchun ham yoziladi. Oradan bir oy o'tib, o'z kodingizni qaytadan ochsangiz, "bu yerda nima qilmoqchi edim?" deb savol berasiz. Yaxshi dokumentatsiya shu savolga javob beradi.
Nega muhim?

• Boshqalar (va kelajakdagi siz) kodni tezroq tushunadi
• Funksiyani qanday chaqirishni eslab qolish shart emas
• Jamoada birga ishlash osonlashadi
• Xatolarni topish va tuzatish tezlashadi

Docstring — Kod Ichidagi Izoh

Docstring — bu funksiya, klass yoki modul ichidagi maxsus izoh matni. U uchta qo'shtirnoq """...""" ichida yoziladi va funksiyaning birinchi qatorida turadi. Oddiy # izohdan farqi — docstring kod bilan birga saqlanadi va help() orqali o'qish mumkin.

Funksiya docstring

def salomlash(ism):
    """Berilgan ism bilan salom matnini qaytaradi."""
    return "Salom, " + ism
# Docstring funksiyaning birinchi qatorida, """...""" ichida

Klass docstring

class Mashina:
    """Mashina haqidagi ma'lumotni saqlaydigan klass."""
    def __init__(self, rang):
        self.rang = rang
# Klass nomidan keyin docstring yoziladi

Modul docstring

"""Bu modul matematik yordamchi funksiyalarni saqlaydi."""

def qoshish(a, b):
    return a + b
# Modul docstring fayl eng tepasida, birinchi qatorda turadi
Ko'p qatorli docstring

Murakkab funksiya uchun docstring bir necha qatordan iborat bo'lishi mumkin: birinchi qatorda qisqa tavsif, keyin parametrlar va qaytaradigan qiymat tushuntiriladi.

def yuza(eni, boyi):
    """To'rtburchak yuzasini hisoblaydi.

    eni: to'rtburchak eni (son)
    boyi: to'rtburchak bo'yi (son)
    Natija: yuza qiymati (son)
    """
    return eni * boyi
# Birinchi qator qisqa tavsif, keyin batafsil tushuntirish

help() bilan Docstring O'qish

help(funksiya) chaqirilganda Python o'sha funksiyaning docstringini ekranga chiqaradi. Shuning uchun yaxshi docstring yozsangiz, boshqalar funksiyani ishga tushirmasdan uning vazifasini bilib oladi.
def salomlash(ism):
    """Berilgan ism bilan salom matnini qaytaradi."""
    return "Salom, " + ism

help(salomlash)
# help() funksiya docstringini ko'rsatadi
>>> Help on function salomlash:
>>>
>>> salomlash(ism)
>>>     Berilgan ism bilan salom matnini qaytaradi.
Maslahat

Docstringga to'g'ridan-to'g'ri funksiya.__doc__ orqali ham kirish mumkin. Masalan print(salomlash.__doc__) docstring matnini chiqaradi.

README Fayl — Loyiha Haqida

README — bu loyihaning "tashrif qog'ozi". Odatda README.md nomi bilan loyiha papkasining tepasida turadi. Yangi odam loyihani ochganda eng birinchi shuni o'qiydi: loyiha nima qiladi, qanday o'rnatiladi va qanday ishlatiladi.
Tipik README tuzilmasi

1. Loyiha nomi — qisqa va aniq sarlavha
2. Tavsif — loyiha nima qiladi, bir-ikki jumla
3. O'rnatish — qanday o'rnatish (pip install ...)
4. Foydalanish — qanday ishga tushirish, misol kod
5. Misollar — natijalar va namunalar
6. Muallif / Litsenziya — kim yozgan, qanday foydalanish mumkin

Kodni Toza Tashkil Qilish

Toza dokumentatsiya toza tuzilma bilan boshlanadi. Quyidagi tartib kodni o'qishni osonlashtiradi.
"""Modul tavsifi — eng tepada."""
import math  # importlar tepada

PI = 3.14159  # o'zgarmaslar (konstantalar)

def aylana_yuzasi(r):
    """Aylana yuzasini qaytaradi."""
    return PI * r * r
# Tartib: modul docstring → importlar → konstantalar → funksiyalar
