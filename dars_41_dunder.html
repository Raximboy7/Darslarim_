Dunder (Sehrli) Metodlar

Modul 7 — OOP (Obyektga asoslangan dasturlash) | Dars 41

Dunder Metod Nima?

Dunder — bu "double underscore" (ikki tagchiziq) so'zining qisqartmasi. Bunday metodlarning nomi ikki tagchiziq bilan boshlanib, ikki tagchiziq bilan tugaydi: __init__, __str__, __len__ kabi. Ularni sehrli (magic) metodlar ham deyishadi.
Bu metodlar Python tomonidan avtomatik chaqiriladi. Masalan siz len(obyekt) deb yozganingizda, Python ichkarida obyekt.__len__() ni chaqiradi. Demak siz o'z obyektingizni Python'ning ichki (built-in) imkoniyatlari bilan ishlata olasiz.

Nega Ular Foydali?

Asosiy g'oya

Dunder metodlar yordamida o'z klassingizni xuddi Python'ning ichki turlari (son, string, list) kabi ishlatasiz:

• print(obyekt) — chiroyli ko'rinish beradi
• len(obyekt) — uzunlikni qaytaradi
• obyekt1 + obyekt2 — qo'shishni amalga oshiradi
• obyekt1 == obyekt2 — solishtirishni amalga oshiradi
• obyekt[i] — indekslash imkonini beradi

__init__ — Konstruktor

__init__ — eng ko'p ishlatiladigan dunder metod. U obyekt yaratilgan paytda avtomatik ishga tushadi va boshlang'ich qiymatlarni o'rnatadi.
class Talaba:
    def __init__(self, ism, yosh):
        self.ism = ism
        self.yosh = yosh

t = Talaba("Ali", 20)
print(t.ism)
# __init__ obyekt yaratilganda avtomatik chaqiriladi
# Natija: Ali

__str__ va __repr__ — Tasvir

__str__ — obyekt print() qilinganda qanday ko'rinishini belgilaydi (foydalanuvchi uchun chiroyli matn). __repr__ esa dasturchi uchun aniq, texnik ko'rinishni beradi.
class Talaba:
    def __init__(self, ism):
        self.ism = ism
    def __str__(self):
        return f"Talaba: {self.ism}"

t = Talaba("Ali")
print(t)
# __str__ bo'lmasa: <__main__.Talaba object at 0x...>
# __str__ bilan natija: Talaba: Ali
>>> Talaba: Ali

__len__ — len() uchun

__len__ metodi len(obyekt) chaqirilganda nima qaytishini belgilaydi. U albatta butun son qaytarishi kerak.
class Guruh:
    def __init__(self, talabalar):
        self.talabalar = talabalar
    def __len__(self):
        return len(self.talabalar)

g = Guruh(["Ali", "Vali", "Guli"])
print(len(g))
# len(g) ichkarida g.__len__() ni chaqiradi
# Natija: 3
>>> 3

__getitem__ — Indekslash obj[i]

__getitem__ metodi obyektni obyekt[i] ko'rinishida indekslash imkonini beradi — xuddi list yoki string kabi.
class Guruh:
    def __init__(self, talabalar):
        self.talabalar = talabalar
    def __getitem__(self, i):
        return self.talabalar[i]

g = Guruh(["Ali", "Vali"])
print(g[0])
# g[0] ichkarida g.__getitem__(0) ni chaqiradi
# Natija: Ali
>>> Ali

__add__ — + Operatori

__add__ metodi ikki obyektni + belgisi bilan qo'shganda nima sodir bo'lishini belgilaydi. a + b ichkarida a.__add__(b) ga aylanadi.
class Pul:
    def __init__(self, miqdor):
        self.miqdor = miqdor
    def __add__(self, boshqa):
        return Pul(self.miqdor + boshqa.miqdor)

natija = Pul(100) + Pul(50)
print(natija.miqdor)
# Pul(100) + Pul(50) → Pul(150)
# Natija: 150
>>> 150

__eq__ — == Solishtirish

__eq__ metodi ikki obyektni == bilan solishtirganda qanday tekshirilishini belgilaydi. U True yoki False qaytaradi.
class Nuqta:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __eq__(self, boshqa):
        return self.x == boshqa.x and self.y == boshqa.y

print(Nuqta(1, 2) == Nuqta(1, 2))
# __eq__ bo'lmasa, faqat manzil solishtiriladi (False)
# __eq__ bilan natija: True
>>> True

Eslab Qoling

• Dunder = ikki tagchiziq __nom__
• Ular Python tomonidan avtomatik chaqiriladi
• __init__ obyekt yaratilganda ishlaydi
• __str__ print() uchun, __len__ len() uchun
• __add__ +, __eq__ == uchun

Misol 1: __add__ — Ikki Obyektni Qo'shish

Pul obyektlarini + bilan qo'shamiz. Python ichkarida __add__ ni chaqiradi.

Misol 2: __len__ — len(obyekt)

Guruhga talabalar ismini vergul bilan kiriting va len(guruh) natijasini ko'ring.

Misol 3: __eq__ — Ikki Obyektni Solishtirish

Ikki Nuqta obyektini == bilan solishtiramiz. Koordinatalar bir xil bo'lsa True qaytadi.
