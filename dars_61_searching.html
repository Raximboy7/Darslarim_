Qidiruv Algoritmlari (Searching)

Modul 11 — Algoritmlar | Dars 61

Qidiruv Nima?

Qidiruv (searching) — bu ro'yxat (list) ichidan kerakli elementni topish jarayoni. Masalan, telefon kontaktlaringizdan bitta ismni topish, yoki sonlar ro'yxatidan 42 raqami bor-yo'qligini aniqlash — bularning hammasi qidiruvdir.
Biz ikkita asosiy qidiruv algoritmini o'rganamiz: Linear search (chiziqli qidiruv) va Binary search (ikkilik qidiruv). Ular bir xil vazifani bajaradi, lekin tezligi juda farq qiladi.

1. Linear Search (Chiziqli Qidiruv)

Eng sodda usul: elementlarni birma-bir, boshidan oxirigacha tekshiramiz. Kerakli son topilsa — uning indeksini qaytaramiz. Hech qayerda bo'lmasa — topilmadi.
Asosiy g'oya

Har bir elementni navbat bilan ko'rib chiqamiz. Ro'yxat saralanmagan (tartiblanmagan) bo'lsa ham ishlaydi — bu uning katta afzalligi.

Python kodi

def linear_search(royxat, nishon):
    for i in range(len(royxat)):
        if royxat[i] == nishon:
            return i
    return -1

sonlar = [5, 3, 8, 1, 9]
print(linear_search(sonlar, 8))
# Birma-bir tekshiramiz
# Topilsa indeksni qaytaramiz
# Topilmasa -1 qaytaramiz
# Natija: 2 (8 raqami 2-indeksda)
>>> 2
Tezligi: O(n)

Eng yomon holatda — kerakli element oxirida bo'lsa yoki umuman bo'lmasa — barcha n ta elementni tekshirishga to'g'ri keladi. Ro'yxat 2 barobar uzaysa, ish ham taxminan 2 barobar ko'payadi.

2. Binary Search (Ikkilik Qidiruv)

Ancha aqlli usul. Lekin bitta shart bor: ro'yxat oldindan saralangan (tartiblangan) bo'lishi shart! Har qadamda ro'yxatning o'rtasiga qaraymiz va yarmini butunlay tashlab yuboramiz.

Qanday ishlaydi? (qadamma-qadam)

1
O'rtani topamiz

Ro'yxatning o'rta elementiga qaraymiz.

2
Taqqoslaymiz

O'rta element nishonga teng bo'lsa — topdik!

3
Yarmini tashlaymiz

Nishon o'rtadan kichik bo'lsa — chap yarmida qidiramiz; katta bo'lsa — o'ng yarmida.

4
Takrorlaymiz

Qolgan yarmida yana o'rtani topib, shu jarayonni davom ettiramiz.

Python kodi

def binary_search(royxat, nishon):
    chap = 0
    ong = len(royxat) - 1
    while chap <= ong:
        orta = (chap + ong) // 2
        if royxat[orta] == nishon:
            return orta
        elif royxat[orta] < nishon:
            chap = orta + 1
        else:
            ong = orta - 1
    return -1

sonlar = [1, 3, 5, 8, 9]
print(binary_search(sonlar, 8))
# chap va ong chegaralar
# to'xtash sharti
# o'rtani topamiz (butun bo'lish)
# o'rta == nishon bo'lsa topdik
# nishon kattaroq -> o'ng yarmga o'tamiz
# nishon kichikroq -> chap yarmga o'tamiz
# ro'yxat SARALANGAN bo'lishi shart!
# Natija: 3
>>> 3
Tezligi: O(log n)

Har qadamda ro'yxatning yarmi tashlanadi! 1000 ta elementdan kerakligini topish uchun atigi ~10 qadam yetadi. 1 million elementdan esa atigi ~20 qadam. Bu juda tez.

Muhim eslatma

Binary search faqat saralangan ro'yxatda ishlaydi. Agar ro'yxat tartibsiz bo'lsa, u noto'g'ri natija beradi! Avval saralash kerak (Dars 60), yoki linear search ishlating.

Misol 1: Linear Search Vizual

Pastdagi tugmani bosing — algoritm har bir sonni birma-bir tekshirib boradi. Qancha qadam ketishini sanab boring.

Misol 2: Binary Search Vizual

Bu yerda ro'yxat saralangan. Tugmani bosing — algoritm o'rtani topib, kerak bo'lmagan yarmini o'chirib (xira qilib) tashlaydi.

Misol 3: Qadamlarni Taqqoslash

Bitta sonni ikkala algoritm bilan qidiramiz va kim necha qadamda topishini ko'ramiz. Saralangan ro'yxat ustida ishlaymiz.
Eslab Qoling

• Linear search — birma-bir, har xil ro'yxatda ishlaydi, O(n)
• Binary search — yarmini tashlab boradi, faqat saralanganda, O(log n)
• Topilmasa ikkalasi ham -1 qaytaradi
• Katta saralangan ro'yxat uchun binary search ancha tez
