Capstone 3 — Web Ilova

Modul 23 — Pro daraja va Capstone | Dars 173

Loyihamiz: To'liq Web Ilova

Bu capstone darsida biz to'liq web ilova quramiz. Misol uchun oddiy blog (yoki onlayn do'kon). Ilovamizda foydalanuvchi ro'yxatdan o'tadi (auth), ma'lumotlar bazasiga yozish/o'qish bo'ladi (model), postlarni qo'shish-tahrirlash-o'chirish (CRUD), chiroyli sahifalar (template) va forma orqali ma'lumot kiritish bo'ladi. Oxirida ilovani internetga joylashtiramiz (deploy).
Bu yerda biz hamma web ko'nikmalarini birlashtiramiz: frontend (foydalanuvchi ko'radigan sahifa) va backend (server, ma'lumotlar bazasi) birga ishlaydi. Biz Flask freymvorkidan foydalanamiz, chunki u sodda va boshlovchilar uchun qulay (Django ham mumkin).

Web Ilova Qanday Ishlaydi?

Web ilova so'rov-javob (request-response) tamoyiliga asoslanadi. Foydalanuvchi brauzerda manzilga kiradi → brauzer serverga so'rov yuboradi → server kodni ishga tushiradi, kerak bo'lsa bazadan o'qiydi → tayyor sahifani (HTML) qaytaradi → brauzer uni ko'rsatadi.

Avval fayllar tuzilishini tayyorlaymiz. Har bir fayl o'z vazifasiga ega.

# Har bir narsa o'z joyida — kod tartibli va o'qishga oson bo'ladi

2-bosqich: Model (Ma'lumotlar Bazasi)

Model — bazadagi jadval tuzilishini Python klassi bilan ifodalaydi. Biz Flask-SQLAlchemy ishlatamiz. Har bir klass = bitta jadval, har bir maydon (Column) = jadval ustuni.
from flask_sqlalchemy import SQLAlchemy
db = SQLAlchemy()

class Foydalanuvchi(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    login = db.Column(db.String(50), unique=True)
    parol = db.Column(db.String(200))

class Post(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    sarlavha = db.Column(db.String(120))
    matn = db.Column(db.Text)
# Foydalanuvchi va Post — bazadagi ikki jadval. id avtomatik tartib raqami.

3-bosqich: Server va Route'lar (View)

View (yoki route) — bu manzil bilan funksiyani bog'laydi. Foydalanuvchi / manziliga kirsa, bosh sahifa funksiyasi ishlaydi va shablonni qaytaradi.
from flask import Flask, render_template, request, redirect
from models import db, Post

app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///blog.db"
db.init_app(app)

@app.route("/")
def bosh_sahifa():
    postlar = Post.query.all()
    return render_template("index.html", postlar=postlar)
# @app.route("/") — "/" manziliga kirilganda bu funksiya ishlaydi. # Bazadan barcha postlarni olib, index.html shabloniga uzatamiz.

4-bosqich: CRUD — Qo'shish, O'qish, Tahrirlash, O'chirish

CRUD = Create (qo'shish), Read (o'qish), Update (tahrirlash), Delete (o'chirish). Bu har qanday ilovaning yuragi. Mana yangi post qo'shish:
@app.route("/qoshish", methods=["POST"])
def post_qoshish():
    sarlavha = request.form["sarlavha"]
    matn = request.form["matn"]
    yangi = Post(sarlavha=sarlavha, matn=matn)
    db.session.add(yangi)
    db.session.commit()
    return redirect("/")
# Formadan ma'lumot olamiz → yangi Post yaratamiz → bazaga saqlaymiz # (add + commit) → bosh sahifaga qaytaramiz (redirect).
@app.route("/ochirish/<int:id>")
def post_ochirish(id):
    post = Post.query.get(id)
    db.session.delete(post)
    db.session.commit()
    return redirect("/")
# <int:id> — manzildan post raqamini olamiz va o'sha postni o'chiramiz.

5-bosqich: Template (HTML Shablon)

Template — bu HTML fayl, lekin ichida Python ma'lumotlarini ko'rsatish mumkin. {{ }} ichida o'zgaruvchi, {% %} ichida sikl yoki shart yoziladi (Jinja2).
<h1>Blogim</h1>
{% for post in postlar %}
    <div class="card">
        <h2>{{ post.sarlavha }}</h2>
        <p>{{ post.matn }}</p>
    </div>
{% endfor %}
# Server postlar ro'yxatini uzatadi, shablon ularni sikl bilan chiqaradi. # {{ post.sarlavha }} — har post sarlavhasi ekranga chiqadi.

6-bosqich: Forma (Foydalanuvchidan Ma'lumot)

Yangi post qo'shish uchun forma kerak. Forma yuborilganda ma'lumot serverga (yuqoridagi /qoshish) boradi.
<form action="/qoshish" method="POST">
    <input name="sarlavha" placeholder="Sarlavha">
    <textarea name="matn"></textarea>
    <button>Saqlash</button>
</form>
# method="POST" — ma'lumot xavfsiz yuboriladi. # name="sarlavha" — server request.form["sarlavha"] orqali oladi.

7-bosqich: Auth (Ro'yxatdan O'tish va Kirish)

Auth — foydalanuvchini tanish. Parolni hech qachon ochiq saqlamaymiz — uni hash qilamiz. Kirgan foydalanuvchini sessionda eslab qolamiz.
from werkzeug.security import generate_password_hash, check_password_hash
from flask import session

@app.route("/kirish", methods=["POST"])
def kirish():
    u = Foydalanuvchi.query.filter_by(login=request.form["login"]).first()
    if u and check_password_hash(u.parol, request.form["parol"]):
        session["user_id"] = u.id
        return redirect("/")
    return "Login yoki parol xato"
# Bazadan foydalanuvchini topamiz, parolni tekshiramiz. # To'g'ri bo'lsa, session ichiga uning id sini yozamiz (eslab qolamiz).

8-bosqich: Deploy (Internetga Joylash)

Ilova tayyor bo'lgach, uni boshqalar ham ko'rishi uchun deploy qilamiz. Bunga oddiy platformalar: Render, Railway, PythonAnywhere. Avval kerakli kutubxonalarni ro'yxatlaymiz.
# requirements.txt
Flask
Flask-SQLAlchemy
gunicorn

# terminalda ishga tushirish (mahalliy)
python app.py
# serverda (deploy)
gunicorn app:app
# requirements.txt — platforma qaysi kutubxonalar kerakligini biladi. # gunicorn — haqiqiy serverda Flask'ni ishga tushiradigan dastur.
Deploy qadamlari

1. Kodni GitHub'ga yuklang
2. Render/Railway'da yangi loyiha oching va GitHub'ga ulang
3. Ishga tushirish buyrug'i: gunicorn app:app
4. Platforma sizga manzil beradi — tayyor!
