Modul 23 — Pro daraja va Capstone | Dars 167

pytest Nima?

pytest — bu Python uchun eng mashhur test kutubxonasi. U unittest dan ancha qulayroq: oddiy assert ishlatasiz, klasslar yozish shart emas, va natijalar chiroyli rangli ko'rinishda chiqadi.
Nega test yozamiz?

Test — bu kodingiz to'g'ri ishlayotganini avtomatik tekshiradigan kod. Bir marta yozasiz, keyin har safar kodni o'zgartirganingizda bir tugma bilan hammasini qayta sinab ko'rasiz. Xato kiritsangiz, test darhol "qizil" beradi.

O'rnatish

pytest standart kutubxonada yo'q — uni pip orqali o'rnatasiz:
pip install pytest
# pytest ni kompyuteringizga o'rnatadi
pytest --version
# O'rnatilganini tekshirish (versiyani chiqaradi)

Birinchi Test Fayli

pytest fayl va funksiyalarni nom orqali topadi: fayl nomi test_*.py bilan boshlanishi, test funksiyalari esa def test_*() bilan boshlanishi kerak.
# test_math.py
def qoshish(a, b):
    return a + b

def test_qoshish():
    assert qoshish(2, 3) == 5
# Fayl nomi test_ bilan, funksiya nomi test_ bilan boshlanadi
assert nima qiladi?

assert ifoda — agar ifoda True bo'lsa, test o'tadi (yashil). Agar False bo'lsa, test yiqiladi (qizil).

Testni Ishga Tushirish

Terminalda papkada turib shunchaki pytest deb yozasiz. pytest barcha test_*.py fayllarini topib ishga tushiradi.
pytest
# Barcha testlarni topib ishga tushiradi
test_math.py . [100%]

===== 1 passed in 0.01s =====
Har bir . (nuqta) — bu o'tgan test. F harfi esa yiqilgan testni bildiradi. Batafsil ko'rish uchun pytest -v ishlatishingiz mumkin.

fixture — Tayyorgarlik

@pytest.fixture — bu testlardan oldin tayyorgarlik qiladigan funksiya. Masalan, namuna ma'lumot tayyorlaydi va uni bir nechta testga beradi. Test funksiyasi fixture nomini parametr sifatida oladi.
import pytest

@pytest.fixture
def savat():
    return ["olma", "nok"]

def test_uzunlik(savat):
    assert len(savat) == 2
# savat fixture tayyor ma'lumot beradi, test uni ishlatadi

parametrize — Ko'p Holat

@pytest.mark.parametrize — bitta testni turli kiritmalar bilan ko'p marta ishga tushiradi. Har bir holat alohida test sifatida hisoblanadi.
import pytest

@pytest.mark.parametrize("a, b, natija", [
    (2, 3, 5),
    (10, 5, 15),
    (0, 0, 0),
])
def test_qoshish(a, b, natija):
    assert a + b == natija
# Bitta test 3 marta ishlaydi — har bir qator alohida holat
===== 3 passed in 0.01s =====

Coverage — Qamrov

Coverage — bu testlaringiz kodning necha foizini sinab ko'rganini o'lchaydi. pytest-cov kutubxonasi orqali ishlaydi.
pip install pytest-cov
pytest --cov
# Kodning necha foizi testlar bilan qamralganini ko'rsatadi
Maslahat

100% coverage kod xatosizligini kafolatlamaydi, lekin yuqori qamrov ko'p xatolarni oldindan ushlab qolishga yordam beradi. Maqsad — muhim qismlarni sinab ko'rish.
