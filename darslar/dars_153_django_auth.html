Autentifikatsiya Nima?

Autentifikatsiya — bu foydalanuvchining kim ekanligini tekshirish jarayoni. Ya'ni sayt: "Sen haqiqatan ham o'sha odammisan?" deb so'raydi. Foydalanuvchi login (ism) va parol kiritadi, sayt esa ularni bazadagi ma'lumotlar bilan solishtiradi.
Oddiy misol

Telegramga kirganda telefon raqam va parol so'raydi-ku? Mana shu — autentifikatsiya. Sayt seni tanib oladi va "bu Ali" deydi.

Django Auth Tizimi

Django'da tayyor autentifikatsiya tizimi bor — django.contrib.auth. Sen parolni qo'lda tekshirish, sessiya yaratish kabi murakkab ishlarni yozishing shart emas — hammasi tayyor.
# settings.py ichida allaqachon bor:
INSTALLED_APPS = [
    'django.contrib.auth',
    'django.contrib.contenttypes',
]
# Bu ilovalar autentifikatsiyani ta'minlaydi

User Modeli

Har bir foydalanuvchi User modelida saqlanadi. Unda username, password, email kabi maydonlar bor.
from django.contrib.auth.models import User

# Yangi foydalanuvchi yaratish
user = User.objects.create_user(
    username="ali",
    password="maxfiy123"
)
# create_user() parolni avtomatik hash qiladi (himoyalaydi)

Ro'yxatdan O'tish (register)

Yangi foydalanuvchi birinchi marta kelganda ro'yxatdan o'tadi — ya'ni o'z login va parolini yaratadi.
from django.contrib.auth.models import User

def register(request):
    if request.method == "POST":
        User.objects.create_user(
            username=request.POST["username"],
            password=request.POST["password"]
        )
    return render(request, "register.html")
# Forma yuborilsa — yangi User yaratiladi

Kirish (login)

Kirish uchun authenticate() parolni tekshiradi, login() esa foydalanuvchini saytga kiritadi.
from django.contrib.auth import authenticate, login

def login_view(request):
    user = authenticate(
        username=request.POST["username"],
        password=request.POST["password"]
    )
    if user is not None:
        login(request, user)
        return redirect("home")
# authenticate parolni tekshiradi, login kiritadi

Chiqish (logout)

Foydalanuvchi ishni tugatganda logout() uni saytdan chiqaradi — sessiyani o'chiradi.
from django.contrib.auth import logout

def logout_view(request):
    logout(request)
    return redirect("login")
# logout sessiyani tozalaydi va login sahifaga qaytaradi

login_required Dekorator

Ba'zi sahifalarni faqat kirgan foydalanuvchilar ko'rishi kerak. @login_required dekoratori sahifani himoya qiladi — kirmagan odam avtomatik login sahifaga yuboriladi.
from django.contrib.auth.decorators import login_required

@login_required
def profile(request):
    return render(request, "profile.html")
# Kirmagan foydalanuvchi bu sahifani ko'ra olmaydi

Sessiyalar (Sessions)

Sessiya — bu sayt sening kirganingni "eslab qolishi". Login qilganingdan keyin sayt senga maxsus cookie beradi. Shu cookie tufayli har safar parol so'ramaydi — seni tanib oladi.
Sessiya qanday ishlaydi

Login → sayt cookie beradi → brauzer cookie'ni saqlaydi → keyingi so'rovlarda cookie yuboriladi → sayt seni taniydi. Logout qilsang — cookie o'chadi.

Parol Xavfsizligi (hash)

Django parolni ochiq matn sifatida saqlamaydi! U parolni hash qiladi — ya'ni o'qib bo'lmaydigan uzun belgilarga aylantiradi. Shuning uchun baza o'g'irlansa ham, parollarni hech kim bila olmaydi.
parol: "maxfiy123"
bazada: pbkdf2_sha256$870000$kT9x...$h8Lq2v...
Muhim qoida

Parolni hech qachon to'g'ridan-to'g'ri yozma. Doimo create_user() yoki set_password() ishlat — ular avtomatik hash qiladi.

Eslab Qoling

• Autentifikatsiya = kim ekanligini tekshirish (login)
• create_user() — ro'yxatdan o'tkazadi
• authenticate() + login() — kiritadi
• logout() — chiqaradi
• @login_required — sahifani himoya qiladi
• Parol doimo hash holatda saqlanadi
