Unit Testing (unittest)

Modul 8 — Xatolar va Test | Dars 45

Test Nima va Nega Kerak?

Test — bu kodingiz to'g'ri ishlayotganini tekshiradigan boshqa bir kod. Siz funksiya yozasiz, keyin "bu funksiya kerakli javobni qaytaryaptimi?" degan savolga avtomatik javob beradigan test yozasiz.
Tasavvur qiling, siz qoshish(2, 3) funksiyasini yozdingiz. U 5 qaytarishi kerak. Har safar kodni o'zgartirganingizda qo'lda tekshirish o'rniga, bir marta test yozib, uni xohlagancha qayta ishlatishingiz mumkin. Test o'tsa — kod sog'lom; yiqilsa — biror joyda xato bor.
Nega test foydali?

• Xatolarni tez topasiz
• Kodni o'zgartirganda nimadir buzilganini darrov bilasiz
• Qo'lda qayta-qayta tekshirish kerak emas
• Kodingizga ishonch ortadi

unittest Moduli

Python'da test yozish uchun tayyor unittest moduli bor. Uni o'rnatish shart emas — Python bilan birga keladi. Faqat import qilish kifoya.
import unittest
# Test moduli — Python ichida tayyor turadi

Birinchi Test: Klass va Metodlar

Avval tekshiriladigan oddiy funksiyani yozamiz. Keyin testlarni unittest.TestCase dan meros olgan klass ichiga joylaymiz. Har bir test — test_ bilan boshlanadigan metod bo'lishi shart.
import unittest

def qoshish(a, b):
    return a + b

class TestQoshish(unittest.TestCase):
    def test_qoshish(self):
        self.assertEqual(qoshish(2, 3), 5)
# class TestX(unittest.TestCase) — test klassi
# def test_...(self): — har bir test alohida metod
# assertEqual(a, b) — a va b teng bo'lishini kutadi
Muhim qoidalar

• Klass unittest.TestCase dan meros olishi shart
• Test metodi nomi test_ bilan boshlanishi shart
• Har metod birinchi parametri self bo'ladi

assert Metodlari

Test ichida assert ("tasdiqla") metodlari natijani tekshiradi. Agar shart bajarilsa — test o'tadi; bajarilmasa — test yiqiladi.
MetodTekshiradiO'tadi (PASS) agar...
assertEqual(a, b)a va b tenga == b
assertNotEqual(a, b)a va b teng emasa != b
assertTrue(x)x rostx — True
assertFalse(x)x yolg'onx — False
assertIn(a, b)a, b ichida bora in b
self.assertEqual(2 + 2, 4)
self.assertTrue(5 > 3)
self.assertFalse(1 > 9)
# 2+2 == 4 → o'tadi
# 5 > 3 rost → o'tadi
# 1 > 9 yolg'on → o'tadi (assertFalse yolg'onni kutadi)

Testni Ishga Tushirish

Fayl oxiriga unittest.main() yozsangiz, fayl ishga tushganda barcha test_ metodlari avtomatik bajariladi.
if __name__ == "__main__":
    unittest.main()
# Bu qator faylni ishga tushirganda testlarni boshlaydi
Terminalda natija shunday ko'rinadi. Har bir nuqta . — bitta o'tgan test. OK — hammasi o'tdi degani.
.
----------------------------------------
Ran 1 test in 0.001s

OK

O'tdi (PASS) va Yiqildi (FAIL)

Agar funksiyada xato bo'lsa, test yiqiladi (FAIL) va Python sizga nima kutilgani-yu nima chiqqanini ko'rsatadi. Masalan qoshish funksiyasida + o'rniga - yozib qo'ysangiz:
def qoshish(a, b):
    return a - b   
# Xato: + o'rniga - yozildi → qoshish(2, 3) = -1
F
========================================
FAIL: test_qoshish
AssertionError: -1 != 5
----------------------------------------
FAILED (failures=1)
Tushuntirish

• . → test o'tdi (PASS)
• F → test yiqildi (FAIL)
• -1 != 5 → kutilgan 5 edi, lekin -1 chiqdi

Misol 1: Testlarni Ishga Tushiring

Quyida qoshish(a, b) funksiyasi uchun 3 ta test bor. "Testlarni ishga tushir" tugmasini bosing — qaysi test o'tgani (yashil) va qaysi yiqilgani (qizil) ko'rinadi.
def qoshish(a, b): return a + b

assertEqual(qoshish(2, 3), 5)
assertEqual(qoshish(0, 0), 0)
assertEqual(qoshish(-1, 1), 0)
# 3 ta test — hammasi to'g'ri javobni kutmoqda

Misol 2: assertEqual To'g'ri yoki Noto'g'ri

Ikkita qiymat kiriting. assertEqual ularni solishtiradi: teng bo'lsa test o'tadi (PASS), teng bo'lmasa yiqiladi (FAIL).

Misol 3: Xato Bo'lsa Test Yiqiladi

kopaytir(a, b) funksiyasini o'zingiz "yozasiz". To'g'ri amalni (*) tanlasangiz test o'tadi, xato amalni tanlasangiz test yiqiladi — xuddi haqiqiy unittest kabi.
kopaytir(4, 3) test qiladi → kutilgan natija: 12

Misol 4: assertTrue va assertFalse

Bir shart yozing (masalan 7 > 3). assertTrue shart rost bo'lishini kutadi, assertFalse esa yolg'on bo'lishini kutadi.
Eslab Qoling

• import unittest — modulni chaqirish
• Klass unittest.TestCase dan meros oladi
• Test metodi test_ bilan boshlanadi
• assertEqual / assertTrue / assertFalse — natijani tekshiradi
• unittest.main() — testlarni ishga tushiradi
• . = o'tdi (PASS), F = yiqildi (FAIL)
