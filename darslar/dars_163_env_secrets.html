Environment Variables Nima?

Environment variable (qisqacha env) — bu dasturning tashqarisida saqlanadigan sozlama. Ya'ni qiymat kod ichida yozilmaydi, balki dastur ishlayotgan muhitda turadi.

Misol uchun: ma'lumotlar bazasi paroli, API kaliti, port raqami — bularning hammasi har bir kompyuterda yoki serverda har xil bo'lishi mumkin. Shuning uchun ularni muhit o'zgaruvchisi sifatida saqlaymiz.
import os

port = os.environ.get("PORT")
print(port)
# os.environ — muhitdagi barcha o'zgaruvchilar lug'ati
# .get("PORT") — PORT nomli o'zgaruvchini oladi
>>> 8000

Nega Kalitlarni Kodga Yozmaymiz?

Xavfsizlik xatosi!

Agar parol yoki API kalitini to'g'ridan-to'g'ri kodga yozsangiz, u GitHub ga yuklanganda hammaga ochiq bo'ladi. Yomon niyatli odam kalitingizni o'g'irlab, hisobingizdan foydalanishi mumkin.

Yomon usul (kalit kod ichida)

api_key = "sk-1234-MAXFIY-PAROL"
# Bu kalit GitHubga ochiq chiqadi!
# XAVFLI: kalit hammaga ko'rinadi

Yaxshi usul (kalit env ichida)

import os
api_key = os.environ.get("API_KEY")
# Kalit kodda emas, muhitda turadi
# XAVFSIZ: kalit kodda yo'q, faqat o'qiladi

.env Fayli

Sozlamalarni qulay saqlash uchun loyiha ichida .env nomli fayl yaratamiz. Unda har bir qatorda NOM=qiymat ko'rinishida yoziladi. Bu fayl maxfiy bo'lib, GitHubga yuklanmasligi kerak.
.env fayli misoli
API_KEY=sk-1234-MAXFIY-PAROL
DB_PASSWORD=mening_parolim
PORT=8000
DEBUG=True
Diqqat

.env faylida tirnoq qo'yish shart emas, bo'sh joy ham qo'ymang: NOM=qiymat deb yozing.

python-dotenv Kutubxonasi

python-dotenv kutubxonasi .env faylidagi qiymatlarni o'qib, ularni muhit o'zgaruvchilariga yuklaydi. Shundan keyin ularni os.environ.get() orqali olamiz.
# Avval o'rnatamiz: pip install python-dotenv
import os
from dotenv import load_dotenv

load_dotenv()

api_key = os.environ.get("API_KEY")
port = os.environ.get("PORT", "8000")
print(api_key, port)
# load_dotenv() — .env faylini o'qiydi
# .get("PORT", "8000") — agar topilmasa, 8000 beradi (standart qiymat)
>>> sk-1234-MAXFIY-PAROL 8000

.gitignore ga .env Qo'shish

Eng muhim qadam! .gitignore fayliga .env ni qo'shamiz. Shunda Git uni e'tiborsiz qoldiradi va GitHubga hech qachon yuklanmaydi.
.gitignore fayli
.env
__pycache__/
*.pyc
venv/
Maslahat

Boshqalar uchun namuna sifatida .env.example faylini yaratib, unga faqat nomlarni qoldiring (qiymatsiz). Bu faylni esa GitHubga yuklash mumkin.

GitHub Secrets (CI/CD)

Server yoki CI/CD (GitHub Actions) da .env fayli bo'lmaydi. U yerda maxfiy kalitlarni GitHub Secrets da saqlaymiz. Bular GitHub sozlamalarida shifrlangan holda turadi va faqat workflow ishlayotganda o'qiladi.

Secret qo'shish joyi

GitHub repozitoriyasi → Settings → Secrets and variables → Actions → New repository secret. Masalan: API_KEY nomli secret yaratamiz.

Workflow ichida ishlatish

# .github/workflows/deploy.yml
steps:
  - name: Dasturni ishga tushirish
    env:
      API_KEY: ${{ secrets.API_KEY }}
    run: python main.py
# secrets.API_KEY — GitHub Secrets dan olinadi
# python kod ichida os.environ.get("API_KEY") orqali o'qiladi

12-Factor App (Qisqacha)

12-Factor App — bu zamonaviy dasturlar uchun mashhur qoidalar to'plami. Uning bir muhim qoidasi: "Config in the environment" — ya'ni barcha sozlamalar (parol, kalit, manzil) kodda emas, muhitda saqlanishi kerak. Buning sababi: bir xil kod har xil joyda (kompyuter, test serveri, ishlab chiqarish serveri) ishlasin, faqat sozlamalar o'zgarsin.
Asosiy fikr

Bitta kod — ko'p muhit. Kodni o'zgartirmasdan, faqat env o'zgaruvchilarini almashtirib, dasturni har qanday joyda ishlatish mumkin.

Eslab Qoling

• Maxfiy kalitlarni hech qachon kodga yozmang
• Sozlamalarni .env faylida saqlang
• .env ni .gitignore ga qo'shing
• Kodda os.environ.get() bilan o'qing
• CI/CD da GitHub Secrets dan foydalaning

Misol 1: Kodga Yozilgan vs .env (Xavfsizlik)

Pastdagi tugmalar ikki usulni taqqoslaydi: kalit kod ichida bo'lsa nima bo'ladi, .env ichida bo'lsa nima bo'ladi.

Misol 2: .env Faylini O'qish (load_dotenv)

Quyida .env faylidagi qiymatlarni o'zgartiring, so'ng load_dotenv() ni ishga tushiring va kod ularni qanday o'qishini ko'ring.

Misol 3: GitHub Secrets Ishlashi

Workflow ishga tushganda GitHub Secrets dagi kalit muhitga yuklanadi, dastur esa uni o'qiydi. Tugmani bosib, jarayonni qadamma-qadam ko'ring.

Misol 4: .gitignore Tekshiruvchi

Fayl nomini kiriting va u GitHubga yuklanadimi yoki yo'qmi tekshiring (.env va boshqa maxfiy fayllar e'tiborsiz qolishi kerak).
