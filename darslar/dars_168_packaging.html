Packaging Nima?

Packaging — bu o'z kodingizni boshqalar pip install orqali o'rnatib ishlatishi mumkin bo'lgan paket ko'rinishiga keltirish. Ya'ni siz yozgan funksiyalarni butun dunyo bilan ulashasiz!
Oddiy misol

Siz requests, numpy yoki flask ni o'rnatganingizda — ular ham xuddi shunday paketlar. Endi siz ham shunday paket yaratasiz!

Paket Tuzilmasi

Paket — bu papka ichidagi fayllar yig'indisi. Eng muhim fayl — __init__.py. Aynan u papkani Python uchun "paket" qiladi.
mening_paketim/ # asosiy papka (loyiha)
├── salompaket/ # paketning o'zi
│   ├── __init__.py # papkani paket qiladi
│   └── salom.py # sizning kodingiz
├── pyproject.toml # metama'lumotlar
└── README.md # tavsif
# salompaket/salom.py
def salom(ism):
    return f"Salom, {ism}!"
# Paket ichidagi oddiy funksiya

__init__.py Nimaga Kerak?

__init__.py fayli bo'sh ham bo'lishi mumkin. Lekin unga funksiyalarni import qilsangiz, paketdan foydalanish osonlashadi.
# salompaket/__init__.py
from .salom import salom

__version__ = "1.0.0"
# Endi: from salompaket import salom — to'g'ridan-to'g'ri ishlaydi

pyproject.toml — Paket Pasporti

pyproject.toml — bu paketning pasporti. Unda paket nomi, versiyasi, muallifi va kerakli kutubxonalar yoziladi. Bu zamonaviy usul (eski setup.py o'rniga).
# pyproject.toml
[build-system]
requires = ["setuptools"]
build-backend = "setuptools.build_meta"

[project]
name = "salompaket"
version = "1.0.0"
description = "Oddiy salom paketi"
authors = [{ name = "Ali Valiyev" }]
requires-python = ">=3.8"
# name = pip install qiladigan nom, version = semantik versiya

Build va Upload Oqimi

Paketni tayyorlab bo'lgach, 3 qadam bilan nashr qilasiz: build → twine upload → PyPI.

PyPI nima?

PyPI (Python Package Index) — bu Python paketlarining ulkan ombori (pypi.org). Aynan shu yerdan pip install paketlarni yuklab oladi. Sinab ko'rish uchun test.pypi.org bor.

Semantik Versiyalash (v1.0.0)

Versiya 3 raqamdan iborat: MAJOR.MINOR.PATCH. Har bir o'zgarish turi o'z raqamini oshiradi.
