Version control (VCS) nima?

Version Control System (VCS) — bu loyihangizdagi fayllarning o'zgarishlar tarixini saqlaydigan tizim. U kim, qachon, nimani o'zgartirganini eslab qoladi va istalgan vaqtda oldingi holatga qaytish imkonini beradi. Eng mashhur VCS — bu Git.

Tasavvur qiling, kodni yozayotib fayllarni shunday nomlaysiz: loyiha.py, loyiha_yangi.py, loyiha_oxirgi.py, loyiha_oxirgi_TUZATILGAN.py... Bu chalkashlik! VCS aynan shu muammoni hal qiladi.

Nega Git kerak?

1
Tarix va qaytish

Har bir o'zgarishni saqlaydi. Xato qilsangiz — bir buyruq bilan oldingi holatga qaytasiz.

2
Jamoada ishlash

Bir necha dasturchi bir loyihada bir vaqtda ishlashi va o'zgarishlarni birlashtirishi mumkin.

3
Zaxira (backup)

Kodni GitHub kabi serverga yuklaysiz — kompyuter buzilsa ham kod yo'qolmaydi.

4
Tajriba (branch)

Asosiy kodga tegmasdan yangi g'oyalarni alohida "shox"da sinab ko'rasiz.

Asosiy tushunchalar

Atama Ma'nosi
Repository (repo) Loyiha va uning butun tarixi saqlanadigan "ombor".
Commit O'zgarishlarning saqlangan "suratlari" — tarixdagi bir nuqta.
Branch Asosiy koddan ajralgan mustaqil ishlash yo'nalishi (shox).
Remote Internetdagi nusxa — masalan, GitHub'dagi repository.
Push / Pull Kodni serverga yuklash / serverdan olib kelish.

Git vs GitHub — farqi nima?

Adashtirmang!

• Git — kompyuteringizda ishlaydigan dastur (tizim).
• GitHub — Git repozitoriylarini internetda saqlaydigan xizmat (sayt).
Ya'ni Git — g'ildirak, GitHub — mashinani qo'yadigan garaj. GitHub'dan tashqari GitLab, Bitbucket kabi muqobillar ham bor.

Git o'rnatilganini tekshirish

Terminal (buyruqlar oynasi)da quyidagini yozing:

git --version
Agar Git o'rnatilgan bo'lsa, uning versiyasi chiqadi. Chiqmasa — git-scm.com dan yuklab o'rnatasiz.
$ git --version
git version 2.43.0

Birinchi marta ishlatishdan oldin o'zingizni tanishtiring:

git config --global user.name "Ismingiz"
git config --global user.email "email@example.com"
Bu ma'lumot har bir commit'ga "imzo" sifatida yoziladi — kim o'zgartirganini bildiradi.

Xulosa

Esda tuting

• VCS — kod tarixini saqlaydigan tizim; Git — eng mashhuri.
• Git qaytish, jamoada ishlash, zaxira va tajriba imkonini beradi.
• Repo, commit, branch, remote — asosiy atamalar.
• Git ≠ GitHub: biri dastur, ikkinchisi onlayn xizmat.
• Keyingi darslarda amalda init, add, commit, push ni o'rganamiz.
