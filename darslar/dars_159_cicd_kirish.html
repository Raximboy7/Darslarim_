CI/CD Nima?

CI/CD — bu dasturni avtomatik tekshirish va yetkazib berish usuli. Ikki qismdan iborat: CI (Continuous Integration — uzluksiz integratsiya) va CD (Continuous Deployment — uzluksiz joylashtirish). Maqsadi — kod yozilgandan keyin uni odam qo'l bilan tekshirib o'tirmasin, hammasi avtomatik bajarilsin.
Oddiy qilib aytganda

Siz kod yozasiz va GitHub'ga yuborasiz. Shu zahoti maxsus robot ishga tushadi: testlarni o'tkazadi, dasturni yig'adi va serverga joylaydi. Hammasi siz tugma bosmasdan, o'zi bajariladi.

CI — Continuous Integration (Uzluksiz Integratsiya)

CI — kod qo'shilganda avtomatik test ishga tushiriladi. Har bir dasturchi o'z kodini umumiy loyihaga qo'shganda, sistema darhol tekshiradi: kod ishlayaptimi, testlar o'tdimi, xato bormi.
# Dasturchi kod yozadi
git add .
git commit -m "yangi funksiya"
git push
# --> CI avtomatik testlarni boshlaydi
# Kod yuborilishi bilan testlar avtomatik ishga tushadi
CI nima beradi?

• Xato kodni erta topadi (server'ga chiqmasdan oldin)
• Testlarni qo'lda ishga tushirish shart emas
• Jamoa kodini birlashtirish xavfsiz bo'ladi

CD — Continuous Deployment (Uzluksiz Joylashtirish)

CD — testlardan o'tgan kod avtomatik deploy qilinadi, ya'ni serverga joylanadi va foydalanuvchilarga yetib boradi. Siz serverga kirib, fayllarni qo'lda ko'chirib o'tirmaysiz — sistema o'zi qiladi.
# Testlar o'tdi --> CD ishga tushadi
docker build -t myapp .
docker push myapp
# --> Dastur serverda yangilandi
# Test muvaffaqiyatli bo'lsa, dastur avtomatik serverga chiqadi

Pipeline Tushunchasi

Pipeline (quvur) — bu ketma-ket bajariladigan qadamlar zanjiri. Kod har bir qadamdan o'tib boradi: test → build → deploy. Agar bir qadam muvaffaqiyatsiz bo'lsa, keyingi qadamlar bajarilmaydi.
📝CommitKod yuborildi
→
🧪TestTestlar tekshiriladi
→
🔨BuildDastur yig'iladi
→
🚀DeployServerga chiqadi

Eslab Qoling

• CI — kod qo'shilganda avtomatik test
• CD — testdan o'tgan kod avtomatik deploy
• Pipeline: test → build → deploy
• Bir qadam yiqilsa — keyingilari bajarilmaydi
• Vositalar: GitHub Actions, GitLab CI, Jenkins
