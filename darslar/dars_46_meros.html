Meros (Inheritance)

Modul 9 — OOP — obyektga asoslangan dasturlash | Dars 46

Meros Nima?

Meros — bu bitta klass o'zining xususiyat va metodlarini boshqa klassga uzatishi. Ya'ni yangi klass eskisidan barcha imkoniyatlarni tayyor holda oladi va ustiga yangilarini qo'shadi. Buni hayotdan misol bilan tasavvur qiling: bola ota-onadan ko'z rangi, bo'yi kabi xususiyatlarni meros oladi.
Real analogiya: Hayvon → It / Mushuk

Hamma hayvonlar nafas oladi va ovqatlanadi. It ham, Mushuk ham hayvon — demak ular ham nafas oladi va ovqatlanadi. Bizga buni har bir hayvon uchun qaytadan yozish shart emas — Hayvon klassida bir marta yozamiz, It va Mushuk esa undan meros oladi.

Ota Klass va Bola Klass

Xususiyatlarni beruvchi klass — ota klass (parent / asosiy klass). Xususiyatlarni oluvchi klass — bola klass (child / voris klass).
Kishi (ota klass)
↓
Talaba (bola klass)
O'qituvchi (bola klass)

Talaba va O'qituvchi — ikkalasi ham Kishi. Demak ular Kishi klassidan meros oladi.

Sintaksis: class Talaba(Kishi)

Bola klass nomidan keyin qavs ichida ota klass nomi yoziladi: class Bola(Ota):
class Kishi:
    def __init__(self, ism):
        self.ism = ism
    def salomlash(self):
        print("Salom, men", self.ism)

class Talaba(Kishi):
    pass
# Talaba klassi Kishi'dan meros oladi.
# pass = "hozircha yangi narsa qo'shma".
# Lekin salomlash() metodi allaqachon ishlaydi!
t = Talaba("Ali")
t.salomlash()
# salomlash() Talaba'da yozilmagan,
# lekin Kishi'dan meros olingan.
>>> Salom, men Ali

super() — Ota Klass __init__ ni Chaqirish

Agar bola klassga yangi xususiyat qo'shmoqchi bo'lsak, o'z __init__ ini yozamiz. Lekin otaning xususiyatlari ham kerak. super().__init__() — bu ota klassning __init__ ini chaqiradi va eski ishni takrorlamaydi.
class Talaba(Kishi):
    def __init__(self, ism, guruh):
        super().__init__(ism)
        self.guruh = guruh
# super().__init__(ism) → Kishi self.ism ni o'rnatadi.
# Keyin biz yangi self.guruh ni qo'shamiz.
t = Talaba("Vali", "IT-21")
print(t.ism, t.guruh)
# ism — ota klassdan, guruh — bola klassdan.
>>> Vali IT-21

Metodni Qayta Ta'riflash (Override)

Override — bola klass otadan olgan metodni o'ziga moslab qaytadan yozishi. Bola klassda bir xil nomli metod bo'lsa, otaning metodi o'rniga bolaning metodi ishlaydi.
class Hayvon:
    def ovoz(self):
        print("Qandaydir ovoz")

class It(Hayvon):
    def ovoz(self):
        print("Vov-vov!")
# It klassi ovoz() ni qayta ta'rifladi (override).
# Endi It uchun otaning ovoz() i emas, o'ziniki ishlaydi.
>>> Hayvon().ovoz() → Qandaydir ovoz
>>> It().ovoz() → Vov-vov!

Meros Nega Kodni Qisqartiradi (DRY)

DRY = Don't Repeat Yourself ("O'zingni takrorlama"). Bir xil kodni qayta-qayta yozish o'rniga, umumiy qismni ota klassda bir marta yozamiz, bolalar esa undan meros oladi.

Eslab Qoling

• class Bola(Ota): — meros sintaksisi
• Bola otaning barcha metod va xususiyatlarini oladi
• super().__init__() — otaning __init__ ini chaqiradi
• Override — bola metodni qaytadan yozsa, o'ziniki ishlaydi
• Meros kodni qisqartiradi (DRY)

Misol 1: Meros Olingan Metod

Kishi klassida salomlash() bor. Talaba undan meros oladi — o'zida yozmasa ham ishlaydi. Ismni kiriting va ko'ring.

Misol 2: Override — Bola Metodni O'zgartiradi

Hayvon turini tanlang. Har bir bola klass ovoz() metodini o'ziga moslab qayta ta'riflagan (override).

Misol 3: super() Qadamma-Qadam

Talaba yaratganda super().__init__(ism) qanday ishlashini qadamlab kuzating. Ism va guruhni kiriting.
