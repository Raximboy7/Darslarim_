Debugging (Xatolarni Topish)

Modul 15 — Concurrency va Advanced | Dars 102

Debugging Nima?

Debugging — bu dasturdagi xatolarni topib tuzatish jarayoni. Xato — inglizcha bug (qo'ng'iz) deyiladi, shuning uchun ularni topib yo'qotish debugging deb ataladi.
Kod ishlamay qolsa yoki noto'g'ri natija bersa, vahima qilmang. Debugging — bu detektiv ishi: kodni qadamma-qadam kuzatib, xato qayerda va nima uchun yuzaga kelganini topasiz.

Xato Turlarini Tushunish

Avval xatoning qaysi turdan ekanini bilish kerak. Uch asosiy tur bor:
TurMa'nosiMisol
SyntaxErrorYozuvda xato — Python kodni o'qiy olmaydiqavs yopilmagan
NameErrorAniqlanmagan o'zgaruvchi ishlatildiprint(x) — x yo'q
TypeErrorMos kelmaydigan turlar qo'shildi"5" + 3
ZeroDivisionErrorNolga bo'lish10 / 0
IndexErrorMavjud bo'lmagan indeksbo'sh ro'yxatdan [0]

Traceback (Xato Izi) ni O'qish

Xato yuz berganda Python traceback (xato izi) chiqaradi. Uni pastdan yuqoriga emas, balki oxirgi qatordan o'qing — eng muhim ma'lumot eng pastda turadi.
Traceback (most recent call last):
  File "main.py", line 3, in <module>
    natija = bol(10, 0)
  File "main.py", line 1, in bol
    return a / b
ZeroDivisionError: division by zero
Tracebackni qanday o'qish kerak?

• Eng oxirgi qator — xato turi va sababi (ZeroDivisionError: division by zero)
• line 1 — xato qaysi qatorda yuz berdi
• File "main.py" — qaysi faylda
• Yuqori qatorlar — qaysi yo'l orqali shu xatoga yetib kelindi

1-usul: print() bilan Debugging

Eng sodda usul — shubhali joylarga print() qo'yib, o'zgaruvchilar qiymatini ekranga chiqarish. Tez va oson, lekin keyin ularni tozalash kerak — shuning uchun bu tartibsiz usul.
def narx_hisobla(soni, narx):
    print("soni =", soni)  # tekshiruv
    print("narx =", narx)  # tekshiruv
    jami = soni * narx
    return jami
# print() bilan har bir o'zgaruvchi qiymatini ko'ramiz
>>> soni = 3
>>> narx = 5000
>>> 15000

2-usul: pdb — Python Debugger

pdb — Pythonning rasmiy debug vositasi. U dasturni belgilangan joyda to'xtatadi va siz o'zgaruvchilarni tekshirib, kodni qadamma-qadam bajarishingiz mumkin.

breakpoint() — to'xtash nuqtasi

def bol(a, b):
    breakpoint()  # shu yerda to'xtaydi
    return a / b
# breakpoint() — Python 3.7+ da eng oson usul
Eski usul ham bor (xuddi shu vazifani bajaradi):
import pdb
pdb.set_trace()  # breakpoint() bilan bir xil
# import pdb; pdb.set_trace() — klassik to'xtash nuqtasi

pdb Buyruqlari (Qadamma-qadam)

Dastur to'xtaganda (Pdb) ko'rinadi. Endi bu buyruqlar bilan kodni kuzatasiz:
BuyruqVazifasi
n (next)Keyingi qatorga o'tish
s (step)Funksiya ichiga kirib qadam tashlash
c (continue)Davom etish (keyingi to'xtashgacha)
p o'zgaruvchiO'zgaruvchi qiymatini ko'rsatish
l (list)Atrofdagi kodni ko'rsatish
q (quit)Debuggerdan chiqish
O'zgaruvchilarni Kuzatish

To'xtash nuqtasida p soni deb yozsangiz, Python o'sha paytdagi soni qiymatini ko'rsatadi. Shu tariqa xato qaysi o'zgaruvchidan kelganini aniqlaysiz.

Misol 1: Tracebackni O'qish

Quyidagi tugmalarni bosing — Python tracebackni qanday o'qishni ko'rsatadi. Har bir qator nimani anglatishini tushuntiramiz.
Traceback (most recent call last):
  File "main.py", line 3, in <module>
    natija = bol(10, 0)
  File "main.py", line 2, in bol
    return a / b
ZeroDivisionError: division by zero

Misol 2: breakpoint da To'xtash

Bu demo breakpoint() ishini simulyatsiya qiladi. Dastur to'xtaganda o'zgaruvchini tanlab, uning qiymatini p buyrug'i bilan ko'ring.
def bol(a, b):
    breakpoint()  # <-- dastur shu yerda to'xtadi
    return a / b

natija = bol(10, 0)

Misol 3: Qadamma-qadam Bajarish

Keyingi qadam (n) tugmasini bosib, kodni qator-qator bajaring. Har qadamda qaysi qator bajarilayotgani va o'zgaruvchilar qiymati ko'rinadi.
jami = 0
narxlar = [100, 200, 300]
for n in narxlar:
    jami = jami + n
print(jami)

Misol 4: Xato Turini Aniqlash

Kodni tanlang — Python qanday xato berishini va sababini ko'rsatamiz.
Eslab Qoling

• Tracebackni oxirgi qatordan o'qing — xato turi va sababi shu yerda
• print() — sodda, tez, lekin tartibsiz usul
• breakpoint() — dasturni to'xtatib, ichini ko'rish imkonini beradi
• pdb da n — keyingi qator, p — qiymatni ko'rish, c — davom etish
