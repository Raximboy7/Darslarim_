Yakuniy Proyekt: To'liq Sayt

Bu modulda Django'ning barcha qismlarini alohida o'rgandik. Endi ularni bitta to'liq saytga birlashtiramiz. Biz oddiy Blog sayti yasaymiz: foydalanuvchi postlar ro'yxatini ko'radi, har bir postni ochib o'qiy oladi, hatto yangi post qo'sha oladi.
Nimani birlashtiramiz?

Project + App · Model · Migration · View · URL · Template · Forma · Admin panel. Hammasi bitta loyihada ishlaydi.

1-Qadam: Project va App Yaratish

Avval loyiha (project) yaratamiz, keyin uning ichida blog (app) hosil qilamiz.
django-admin startproject mysite
cd mysite
python manage.py startapp blog
# mysite — loyiha, blog — uning ichidagi app
Yangi app'ni settings.py da ro'yxatga olamiz:
# mysite/settings.py
INSTALLED_APPS = [
    # ... boshqalar
    "blog",
]
# blog app'ini Django'ga tanitamiz

Loyiha tuzilishi

mysite/
  manage.py
  mysite/
    settings.py
    urls.py
  blog/
    models.py
    views.py
    urls.py
    admin.py
    templates/

2-Qadam: Model Yaratish

Model — bu bazadagi jadval. Har bir post sarlavha, matn va sana saqlaydi.
# blog/models.py
from django.db import models

class Post(models.Model):
    sarlavha = models.CharField(max_length=200)
    matn = models.TextField()
    sana = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.sarlavha
# Post jadvali: sarlavha, matn, sana ustunlari

3-Qadam: Migration (Bazaga Yozish)

Model faqat kod. Uni haqiqiy jadvalga aylantirish uchun migration qilamiz.
python manage.py makemigrations
python manage.py migrate
# makemigrations — rejani tayyorlaydi
# migrate — bazaga haqiqatan yozadi
>>> Migrations for 'blog':
>>>   blog/migrations/0001_initial.py
>>> Applying blog.0001_initial... OK

4-Qadam: View Yozish

View — saytning miyasi. U bazadan postlarni oladi va template'ga uzatadi.
# blog/views.py
from django.shortcuts import render
from .models import Post

def post_royxati(request):
    postlar = Post.objects.all()
    return render(request, "blog/royxat.html",
        {"postlar": postlar})
# Bazadan barcha postlarni olib, template'ga beradi

5-Qadam: URL Bog'lash

URL manzilni view bilan bog'laydi. Foydalanuvchi manzilni ochsa, mos view ishlaydi.
# blog/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path("", views.post_royxati, name="royxat"),
]
# Bosh sahifa ("") -> post_royxati view'iga ulanadi
Bu URL'larni asosiy mysite/urls.py ga ulaymiz:
# mysite/urls.py
from django.urls import path, include

urlpatterns = [
    path("", include("blog.urls")),
]
# blog app'ining URL'larini asosiy loyihaga ulaymiz

6-Qadam: Template (HTML Sahifa)

Template — foydalanuvchi ko'radigan HTML. View bergan postlarni {% for %} sikli bilan chiqaramiz.
<!-- blog/templates/blog/royxat.html -->
<h1>Mening Blogim</h1>
{% for post in postlar %}
  <div>
    <h2>{{ post.sarlavha }}</h2>
    <p>{{ post.matn }}</p>
  </div>
{% endfor %}
# Har bir post sarlavha va matnini ko'rsatadi

7-Qadam: Forma — Yangi Post Qo'shish

Forma orqali foydalanuvchi yangi post yozadi. View formani qabul qilib bazaga saqlaydi.
# blog/views.py (qo'shimcha)
from django.shortcuts import redirect

def post_qoshish(request):
    if request.method == "POST":
        Post.objects.create(
            sarlavha=request.POST["sarlavha"],
            matn=request.POST["matn"])
        return redirect("royxat")
    return render(request, "blog/qoshish.html")
# POST kelsa saqlaydi, aks holda forma ko'rsatadi

8-Qadam: Admin Panel

Admin panelda postlarni brauzerdan boshqarish uchun modelni ro'yxatga olamiz.
# blog/admin.py
from django.contrib import admin
from .models import Post

admin.site.register(Post)
# Endi /admin sahifasida Post'larni boshqarish mumkin

9-Qadam: Saytni Ishga Tushirish

python manage.py runserver
# Brauzerda http://127.0.0.1:8000 ni ochamiz
>>> Starting development server at http://127.0.0.1:8000/
>>> Quit the server with CONTROL-C.
