Commit Nima?

Commit — bu loyihangizning ma'lum bir paytdagi surati (snapshot). Har bir commit "o'zgarishlar suratga olingan nuqta"dir. Siz biror ish bajarasiz, keyin uni commit qilasiz — Git esa o'sha holatni abadiy eslab qoladi. Keyinchalik istalgan commitga qaytishingiz mumkin.
Oddiy taqqoslash

O'yin o'ynaganingizda save qilasiz-ku? Commit ham xuddi shunday — kodingizning "save nuqtasi". Xato qilsangiz, oxirgi commitga qaytib olasiz.

Uchta Asosiy Holat

Git'da fayl uchta bosqichdan o'tadi. Buni tushunish commit'ning kalitidir.

git add — Staging'ga Qo'shish

git add faylni staging areaga (commitga tayyorlash maydoniga) qo'shadi. Bu — "men shu faylni keyingi commitga qo'shmoqchiman" degani.
git add main.py
# Faqat bitta faylni staging'ga qo'shish
git add .
# Nuqta (.) — hamma o'zgargan fayllarni qo'shish

git commit — O'zgarishlarni Saqlash

git commit -m "xabar" staging'dagi fayllarni tarixga saqlaydi. -m degani message (xabar) — har commitga qisqa izoh yoziladi.
git commit -m "Birinchi sahifa qo'shildi"
# Staging'dagi o'zgarishlar xabar bilan saqlanadi
>>> [main a1b2c3d] Birinchi sahifa qo'shildi
>>> 1 file changed, 12 insertions(+)

git status — Hozirgi Holat

git status — "hozir nima bo'lyapti?" degan savolga javob beradi. Qaysi fayllar o'zgargan, qaysilari staging'da, qaysilari hali kuzatilmagan — barchasini ko'rsatadi.
git status
# Repozitoriyaning hozirgi holatini ko'rsatadi

git log — Commit Tarixi

git log — barcha commitlar ro'yxatini ko'rsatadi. Eng yangisi tepada turadi. Har commitning hash (yagona kodi), muallifi, sanasi va xabari ko'rinadi.
git log
git log --oneline
# --oneline har commitni bitta qatorda qisqa ko'rsatadi

Yaxshi Commit Xabari

Commit xabari kelajakdagi siz uchun yoziladi. "Nima qildim?" degan savolga aniq javob bo'lsin. Qisqa, aniq va ish-harakat shaklida yozing.
Yomon ✗Yaxshi ✓
"o'zgartirish""Login tugmasi qo'shildi"
"asdf""Narx hisoblash xatosi tuzatildi"
"fix""Bosh sahifa dizayni yangilandi"
Eslab Qoling

• git add . — hamma o'zgarishlarni staging'ga
• git commit -m "..." — xabar bilan saqlash
• git status — hozirgi holatni ko'rish
• git log — commit tarixini ko'rish
• Tartib: add → commit, doimo shu ketma-ketlik

Misol 1: add → commit Ketma-ketligi

"Boshla" tugmasini bosing va terminal qadamma-qadam git add dan git commit gacha bo'lgan jarayonni ko'rsatadi.
bash — loyiham
# "Boshla" tugmasini bosing

Misol 2: git status Holati

Bir faylni o'zgartiring, keyin uni staging'ga qo'shing va git status qanday o'zgarishini kuzating.
git status
$ git status
nothing to commit, working tree clean

Misol 3: git log Tarixi

Har "Commit qo'shish" bosilganda yangi commit tarixga qo'shiladi. git log ro'yxatini real vaqtda ko'ring.
