Best Practices (Eng Yaxshi Amaliyotlar)

Modul 17 — Qo'shimcha Mavzular | Dars 113

Best Practices Nima?

Best practices — bu tajribali dasturchilar yillar davomida to'plagan, kodni toza, o'qiladigan va ishonchli qiladigan amaliyotlar majmuasi. Kodingiz nafaqat ishlashi, balki boshqa odamlar (va kelajakdagi siz!) uni oson tushunishi ham muhim.
Nega muhim?

Kod bir marta yoziladi, lekin o'n marta o'qiladi. Toza kod kamroq xato keltirib chiqaradi, uni tuzatish va kengaytirish oson bo'ladi. Yomon kod esa ertaga sizning eng katta dushmaningizga aylanadi.

PEP 8 — Python Uslub Qo'llanmasi

PEP 8 — bu Python kodini qanday yozish bo'yicha rasmiy qo'llanma. U barcha Python dasturchilari kelishgan umumiy qoidalar to'plami. PEP 8 ga amal qilsangiz, kodingiz boshqa dasturchilarning kodiga o'xshaydi va hamma uni oson o'qiydi.
QoidaTavsiya
Chekinish (indent)4 ta bo'sh joy (tab emas)
Satr uzunligi79 belgidan oshmasin
Operatorlar atrofidabo'sh joy: x = a + b
Bo'sh qatorlarfunksiyalar orasida 2 ta bo'sh qator
Importlarfayl boshida, har biri alohida qatorda
Izohlaraniq va foydali bo'lsin

Maslahat

Nom mazmunli bo'lsin. x yoki a emas, balki narx yoki foydalanuvchi_soni kabi nom tanlang. Bir harfli nomlar faqat qisqa sikllarda (masalan, for i in ...) mos keladi.

Yomon Kod vs Yaxshi Kod

Quyida bir xil ishni bajaradigan ikkita kod. Birinchisi ishlaydi, lekin o'qish qiyin. Ikkinchisi PEP 8 ga amal qiladi va tushunarli.
❌ Yomon
def f(a,b):
  x=a+b
  return x
r=f(5,3)
✅ Yaxshi
def summani_hisobla(birinchi, ikkinchi):
    natija = birinchi + ikkinchi
    return natija

yigindi = summani_hisobla(5, 3)
Farqi nimada?

Yaxshi kodda: mazmunli nomlar, 4 bo'sh joy chekinish, operator atrofida bo'sh joy, vergullardan keyin bo'sh joy. Kod o'zini o'zi tushuntiradi.

DRY — Takrorlanmang

DRY = Don't Repeat Yourself (O'zingni takrorlama). Agar bir xil kodni bir necha marta yozayotgan bo'lsangiz, uni funksiyaga ajrating. Shunda o'zgartirish kerak bo'lsa, faqat bitta joyni tahrirlaysiz.
❌ Takror
print("Salom Ali")
print("Salom Vali")
print("Salom Guli")
✅ DRY
def salomlash(ism):
    print("Salom " + ism)

for ism in ["Ali", "Vali", "Guli"]:
    salomlash(ism)

KISS — Sodda Saqlang

KISS = Keep It Simple, Stupid (Soddaroq qil). Eng oddiy yechim ko'pincha eng yaxshisi. Kodingizni kerakdan ortiq murakkablashtirmang. Murakkab kod chiroyli emas — tushunarli kod chiroyli.
# Murakkab
if son % 2 == 0:
    juftmi = True
else:
    juftmi = False

# Sodda (KISS)
juftmi = (son % 2 == 0)
# Ikkala kod bir xil ishlaydi, lekin pastki variant qisqaroq va tozaroq

Refactoring — Kodni Yaxshilash

Refactoring — bu kodning ishlashini o'zgartirmasdan, uning ichki tuzilishini yaxshilash. Ya'ni natija bir xil qoladi, lekin kod tozaroq va tushunarliroq bo'ladi. Bu doimiy jarayon — yaxshi dasturchilar kodini muntazam yaxshilab boradi.
Eslab Qoling

• PEP 8 — Python uslub qo'llanmasi, 4 bo'sh joy chekinish
• snake_case — o'zgaruvchi va funksiya, PascalCase — klass, UPPER — konstanta
• DRY — takrorlanma, kodni funksiyaga ajrat
• KISS — soddaroq saqla
• Mazmunli nom tanla, toza kod yoz
