Big O nima uchun kerak?

Big O — algoritm qancha tez ishlashini emas, balki ma'lumot hajmi (n) oshgani sari qancha sekinlashishini tasvirlaydi. U kompyuter tezligiga bog'liq emas — sof matematik o'sish tezligini beradi.

Ikki dastur bir xil natija beradi, lekin biri 1000 ta element uchun 0.01 soniya, ikkinchisi 10 soniya ishlashi mumkin. Big O aynan shu farqni oldindan aytib beradi.

O(1) — Doimiy vaqt

Ma'lumot hajmidan qat'i nazar amal doim bir xil vaqt oladi. Eng tez turdagi. 

def birinchi(royxat):
    return royxat[0]   # indeks bo'yicha o'qish

songa = [10, 20, 30, 40]
print(birinchi(songa))
Ro'yxatda 4 ta yoki 4 million element bo'lsin — birinchi elementga kirish bir xil qadamda bo'ladi. Bu O(1).
>>> 10

O(n) — Chiziqli vaqt

Elementlar soni ikki barobar oshsa, qadamlar ham ikki barobar oshadi. Ko'pincha bitta for siklidan iborat. 

def yigindi(royxat):
    natija = 0
    for son in royxat:   # har element bir marta ko'riladi
        natija += son
    return natija
n ta element uchun n ta qo'shish. n=100 bo'lsa 100 qadam, n=1000 bo'lsa 1000 qadam. Bu O(n).

O(n^2) — Kvadratik vaqt

Sikl ichida sikl bo'lganda paydo bo'ladi. Element soni 10 barobar oshsa, ish hajmi 100 barobar oshadi — juda tez sekinlashadi. 

def juftliklar(royxat):
    for a in royxat:        # tashqi sikl: n marta
        for b in royxat:    # ichki sikl: yana n marta
            print(a, b)   # jami n * n = n^2 marta
n=1000 bo'lsa — 1 000 000 amal! Bubble sort va oddiy izlash algoritmlari aynan O(n^2). Iloji boricha bundan qochish kerak.
Diqqat Agar kodda ichma-ich ikkita sikl bir xil ro'yxat ustidan yursa — bu deyarli har doim O(n^2).

O(log n) va O(n log n)

O(log n) — har qadamda muammo yarmiga qisqaradi. Binary search bunga misol: 1 million elementni atigi ~20 qadamda tekshiradi. 

def binary_search(royxat, x):
    chap, ong = 0, len(royxat) - 1
    while chap <= ong:
        orta = (chap + ong) // 2
        if royxat[orta] == x:
            return orta
        elif royxat[orta] < x:   # o'ng yarmiga o'tamiz
            chap = orta + 1
        else:
            ong = orta - 1       # chap yarmiga o'tamiz
    return -1
Har taqqoslashda qidiruv maydoni yarmiga kesiladi — shuning uchun O(log n). Faqat saralangan ro'yxatda ishlaydi.

O(n log n) — samarali saralash algoritmlari (Merge sort, Quick sort, Python sorted()) darajasi. O(n^2) dan ancha tez.

Amaliyot

Mashq: har birining Big O sini toping

1. Ro'yxatning oxirgi elementini qaytaruvchi funksiya → ?
2. Ro'yxatdagi eng katta sonni topuvchi for sikl → ?
3. Ikki ro'yxatning barcha juftliklarini chop etuvchi ichma-ich sikl → ?
4. Saralangan ro'yxatda binary search → ?

Javoblar: 1) O(1), 2) O(n), 3) O(n^2), 4) O(log n).

Xulosa

Esda saqlang

• Big O — o'sish tezligi, aniq vaqt emas.
• Doimiylar tashlanadi: O(2n) → O(n), O(n+5) → O(n).
• Ichma-ich sikl → ehtiyot bo'ling, ko'pincha O(n^2).
• Yarmiga bo'lish → O(log n), eng arzon qidiruv.
• Katta ma'lumotda O(n log n) dan yomonrog'idan qoching.
