ORM Nima?

ORM (Object-Relational Mapping — Obyekt-Relyatsion Bog'lash) — bu ma'lumotlar bazasi bilan SQL yozmasdan, sof Python kodi orqali ishlash usuli. Siz jadval bilan emas, oddiy Python obyektlari bilan ishlaysiz, Django esa ularni avtomatik SQL so'rovlariga aylantiradi.
Oddiy qilib aytganda

Jadvaldagi har bir qator — bu Python obyekti.
Har bir ustun — bu obyektning xususiyati (atribut).
Siz Model.objects orqali so'rov berasiz, Django esa orqada SQL ishlatadi.

Misol Model

Quyidagi darslarda shu modeldan foydalanamiz. Bu — bizning Talaba jadvalimiz.
from django.db import models

class Talaba(models.Model):
    ism = models.CharField(max_length=100)
    yosh = models.IntegerField()
    shahar = models.CharField(max_length=50)
# Talaba jadvali: ism, yosh va shahar ustunlari bilan

QuerySet Nima?

QuerySet — bu ma'lumotlar bazasidan olingan obyektlar to'plami. U ro'yxatga o'xshaydi: uni aylantirish (loop), sanash, filtrlash mumkin. Eng muhimi — QuerySet dangasa (lazy): baza so'rovi faqat ma'lumot kerak bo'lganda yuboriladi.
talabalar = Talaba.objects.all() # QuerySet
for t in talabalar:
    print(t.ism)
# all() barcha qatorlarni QuerySet sifatida qaytaradi

Diqqat: .get() va .filter() farqi

.get() — har doim bitta obyekt qaytaradi. Agar topilmasa yoki bir nechta topilsa — xatolik beradi.
.filter() — har doim QuerySet qaytaradi (0 ta, 1 ta yoki ko'p bo'lishidan qat'i nazar).

# Barcha talabalar
hammasi = Talaba.objects.all()

# id=1 bo'lgan bitta talaba
ali = Talaba.objects.get(id=1)

# Yoshi 20 bo'lganlar
yoshlar = Talaba.objects.filter(yosh=20)

# Toshkentlik BO'LMAGANLAR
boshqalar = Talaba.objects.exclude(shahar="Toshkent")
# filter shartga mos keladi, exclude esa teskarisi

Yangi Qator Yaratish

Bazaga yangi ma'lumot qo'shishning ikki yo'li bor: .create() — bir qadamda, yoki .save() — obyekt yasab, keyin saqlash.
.create() — bir qadamda
Talaba.objects.create(
  ism="Ali",
  yosh=20,
  shahar="Toshkent"
)
.save() — ikki qadamda
t = Talaba(
  ism="Ali",
  yosh=20
)
t.save()

Yangilash va O'chirish

# YANGILASH: obyektni olib, o'zgartirib, saqlash
t = Talaba.objects.get(id=1)
t.yosh = 21
t.save()

# Ko'p qatorni birdan yangilash
Talaba.objects.filter(shahar="Toshkent").update(yosh=25)

# O'CHIRISH
t.delete()
# save() bitta obyektni, update() ko'p qatorni yangilaydi

Tartiblash (order_by)

.order_by() natijani tartiblaydi. Oldiga - qo'yilsa — kamayish tartibida (teskari).
# Yosh bo'yicha o'sish tartibida
Talaba.objects.order_by("yosh")

# Yosh bo'yicha kamayish tartibida (kattadan kichikka)
Talaba.objects.order_by("-yosh")
# "-" belgisi teskari (kamayuvchi) tartibni bildiradi

ORM vs SQL

Bir xil natija — ikki xil yo'l bilan. Pastda ORM va u yaratadigan SQL so'rovi yonma-yon ko'rsatilgan.
ORM (Python)SQL
Talaba.objects.all()SELECT * FROM talaba;
.filter(yosh=20)WHERE yosh = 20;
.get(id=1)WHERE id = 1;
.order_by("-yosh")ORDER BY yosh DESC;
.count()SELECT COUNT(*);
Nega ORM yaxshi?

• Sof Python — SQL tilini bilish shart emas
• Xavfsiz — SQL injection hujumlaridan himoyalaydi
• Ko'chma — bazani (SQLite, PostgreSQL) o'zgartirsangiz ham kod o'zgarmaydi
