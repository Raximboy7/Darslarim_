Modul 3 — Shartlar va sikllar | Dars 15

Nega Bu Muhim?

Sikllar (while va for) — dasturchining eng kuchli quroli. Lekin aynan shu yerda boshlang'ichlar eng ko'p xatoga yo'l qo'yadi. Yaxshi xabar: bu xatolar bir necha turga bo'linadi, va ularni bilsangiz — oson tuzatasiz. Bu darsda 4 ta eng keng tarqalgan xatoni ko'ramiz.

1. Cheksiz Sikl (Infinite Loop)

while sharti hech qachon False bo'lmasa, sikl abadiy davom etadi. Eng ko'p uchraydigan sabab — sanagichni (masalan i) oshirishni unutish.
❌ Xato: i o'zgarmaydi → cheksiz sikl
i = 1
while i <= 5:
    print(i) # i hech qachon o'zgarmaydi!

i doim 1 bo'lib qoladi, shart doim rost — ekranga 1 cheksiz chiqaveradi.

✓ To'g'ri: i ni oshiramiz
i = 1
while i <= 5:
    print(i)
    i = i + 1 # i oshadi → shart oxiri buziladi
>>> 1
>>> 2
>>> 3
>>> 4
>>> 5
⚠️ Dastur qotib qolsa?

Agar dastur to'xtamay qolsa — terminalda Ctrl + C bosing. Bu sikldan majburiy chiqishning eng tez yo'li.

2. Off-by-One Xatosi (Bir qadam adashish)

range(a, b) da b qadam ICHIGA KIRMAYDI. Shuning uchun 1 dan n gacha (n ham kirsin) sanash kerak bo'lsa — range(1, n+1) yozish lozim. Bu eng ko'p uchraydigan "kichik" xato.
KodBeradiIzoh
range(1, 5)1, 2, 3, 45 KIRMAYDI
range(1, 5+1)1, 2, 3, 4, 55 ham kiradi ✓
range(0, 3)0, 1, 23 ta son, 0 dan
range(3)0, 1, 2boshlanish 0
❌ Xato: 1 dan 5 gacha desa-yu, 5 chiqmaydi
for i in range(1, 5):
    print(i) # 5 chiqmaydi!
>>> 1 2 3 4   # 5 yo'q
✓ To'g'ri: n+1 bilan
for i in range(1, 5 + 1):
    print(i)
>>> 1 2 3 4 5   # 5 ham bor ✓

3. break va continue ni Noto'g'ri Ishlatish

break — sikldan butunlay chiqadi. continue — shu qadamni tashlab, keyingisiga o'tadi. Ularni adashtirish keng tarqalgan xato.
BuyruqVazifasi
breakSikl butunlay to'xtaydi
continueFaqat shu qadam tashlanadi, sikl davom etadi
❌ Xato: juftlarni tashlamoqchi, lekin break qo'ygan
for i in range(1, 6):
    if i % 2 == 0:
        break # 2 da to'xtab qoladi!
    print(i)
>>> 1   # 2 ga yetganda sikl o'ladi
✓ To'g'ri: continue bilan toqlarni chiqaramiz
for i in range(1, 6):
    if i % 2 == 0:
        continue # juftni tashlaydi
    print(i)
>>> 1 3 5   # faqat toq sonlar ✓

4. Sikl Ichida Ro'yxatni O'zgartirish

Ro'yxat (list) bo'ylab aylanayotganda, aynan o'sha ro'yxatdan element o'chirish yoki qo'shish — ba'zi elementlarni "sakrab o'tib" ketishga olib keladi. Bu juda nozik xato.
❌ Xato: aylanayotgan ro'yxatdan o'chirish
sonlar = [1, 2, 2, 3]
for x in sonlar:
    if x == 2:
        sonlar.remove(x) # bitta 2 qoladi!
>>> [1, 2, 3]   # ikkinchi 2 sakrab o'tildi
✓ To'g'ri: yangi ro'yxat yasaymiz
sonlar = [1, 2, 2, 3]
natija = [x for x in sonlar if x != 2]
>>> [1, 3]   # barcha 2 to'g'ri olib tashlandi ✓
Eslab Qoling

• Cheksiz sikldan qutulish: while ichida sanagichni oshiring (i = i + 1)
• range(a, b) da b kirmaydi — oxirini ham olish uchun b+1
• break — chiqadi, continue — tashlab o'tadi
• Aylanayotgan ro'yxatni o'zgartirmang — yangi ro'yxat yasang

Misol 1: Xatoni Top

Quyidagi kodni o'qing va qanday xato borligini tanlang:

i = 1
while i <= 3:
    print(i)
# (i ni oshirish yo'q)

Misol 2: range() Vizualizatori

range(a, b) qaysi sonlarni beradi? O'zingiz sinab ko'ring.

Misol 3: Off-by-One — 1 dan n gacha

n gacha (n ham kirsin) sanash uchun range(1, n+1) kerakligini ko'ring.

Misol 4: Cheksiz Sikl Ogohlantirgichi

while siklida sanagich oshadimi yoki yo'q? Tekshirib ko'ring.

Kod: i = 1; while i <= 5: i = i + (qadam)

Misol 5: break va continue

1 dan 6 gacha sonlarda juft son uchraganda nima bo'ladi?
