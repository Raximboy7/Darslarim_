Tree va Graph (Daraxt va Graf)

Modul 11 — Algoritmlar | Dars 66

Tree (Daraxt) Nima?

Tree — bu ma'lumotlarni pog'onali (ierarxik) ko'rinishda saqlaydigan tuzilma. Xuddi haqiqiy daraxtga o'xshaydi, lekin teskari: ildiz tepada, shoxlar pastga tarqaladi.
Asosiy atamalar

• Ildiz (root) — eng tepadagi tugun, hammasi shundan boshlanadi
• Tugun (node) — daraxtdagi har bir element
• Ota (parent) — tepada turgan tugun
• Bola (child) — pastda turgan, ulangan tugun
• Barg (leaf) — bolasi yo'q tugun (eng pastdagilar)

Misol: oddiy daraxt diagrammasi

Bu yerda A — ildiz, B va C uning bolalari, D, E, F — barglar (bolasi yo'q).

Binary Tree (Ikkilik Daraxt)

Binary Tree — bu maxsus daraxt: har bir tugunda ko'pi bilan 2 ta bola bo'ladi. Ularni chap bola va o'ng bola deb ataymiz.
class Node:
    def __init__(self, qiymat):
        self.qiymat = qiymat
        self.chap = None
        self.ong = None
# Har tugunda qiymat, chap va o'ng bola bor
# Bola bo'lmasa None turadi
ildiz = Node(10)
ildiz.chap = Node(5)
ildiz.ong = Node(15)
# 10 — ildiz, 5 — chap bola, 15 — o'ng bola

Tree Traversal — Daraxtni Aylanib Chiqish

Daraxtdagi barcha tugunlarni bir marta ko'rib chiqishga traversal (ziyorat) deyiladi. Ikki asosiy usul bor: DFS va BFS.

DFS — Depth First Search (Chuqurlik bo'yicha)

G'oya: avval chuqurga tushamiz

Bir shoxni oxirigacha kuzatib boramiz, so'ng orqaga qaytib boshqa shoxga o'tamiz. Xuddi labirintda bir yo'lni oxirigacha tekshirib, keyin orqaga qaytgandek.

BFS — Breadth First Search (Kenglik bo'yicha)

G'oya: avval pog'onama-pog'ona

Daraxtni qavat-qavat ko'rib chiqamiz: avval ildiz, keyin uning barcha bolalari, keyin keyingi qavat. Xuddi suvga tosh tashlasak, to'lqin tarqalgandek.

UsulYo'nalishYordamchi tuzilma
DFSChuqurga (pastga)Stack (yoki rekursiya)
BFSKenglikka (qavat-qavat)Queue (navbat)
def dfs(tugun):
    if tugun is None:
        return
    print(tugun.qiymat)
    dfs(tugun.chap)
    dfs(tugun.ong)
# DFS rekursiya bilan: avval o'zi, keyin chap, keyin o'ng
# Bu "preorder" tartibi deyiladi

Graph (Graf) Nima?

Graph — bu tugunlar (nodes) va ularni bog'lovchi ulanishlar (edges) dan iborat tuzilma. Daraxtdan farqi: grafda ilmoqlar (sikllar) bo'lishi mumkin va ildiz shart emas.
Tree vs Graph

• Tree — bu maxsus graf: ildizi bor, sikli yo'q
• Graph — istalgan tugunlar bir-biriga ulanishi mumkin
• Grafda tugun bir nechta "qo'shni" tugunga bog'lanadi

Eslab Qoling

• Tree — pog'onali tuzilma: ildiz, tugun, barg
• Binary Tree — har tugunda ko'pi bilan 2 bola
• DFS — chuqurga, BFS — qavat-qavat
• Graph — tugunlar + ulanishlar, sikl bo'lishi mumkin
• Tree — bu sodda graf ko'rinishi

Misol 1: Daraxt Tuzilmasi Vizualizatori

Tugmani bosing — daraxt qismlari yoritiladi.

Misol 2: DFS va BFS Ziyorat Tartibi

Yuqoridagi daraxt uchun ziyorat tartibini ko'ring. Tartib raqamlari pastda chiqadi.

Misol 3: Graf Qo'shnilarini Ko'rish

Tugunni tanlang — uning to'g'ridan-to'g'ri ulangan qo'shnilari yoritiladi.
