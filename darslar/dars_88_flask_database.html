Database Nega Kerak?

Oldingi darslarda formadan kelgan ma'lumotni o'zgaruvchiga saqladik. Lekin dastur qayta ishga tushganda yoki server o'chganda — hamma ma'lumot yo'qoladi.

Database (ma'lumotlar bazasi) — ma'lumotni doimiy saqlash uchun maxsus joy. Foydalanuvchilar, postlar, buyurtmalar — hammasi shu yerda yashaydi va server o'chsa ham yo'qolmaydi.
Oddiy taqqoslash

O'zgaruvchi — bu qo'lyozma qog'oz: ish tugagach tashlanadi.
Database — bu daftar: yozganing doim qoladi, keyin qayta o'qiy olasan.

SQLite — Eng Oddiy Baza

SQLite — bu butun bazani bitta faylga saqlaydigan oddiy ma'lumotlar bazasi. Alohida server o'rnatish kerak emas — Python bilan birga keladi. O'rganish va kichik loyihalar uchun ideal.
# Baza shunchaki bitta fayl bo'ladi:
sqlite:///baza.db
# baza.db fayli loyiha papkasida paydo bo'ladi

Flask-SQLAlchemy — ORM

Baza bilan ishlash uchun odatda SQL tili yoziladi (SELECT * FROM users). Buni o'rganish vaqt oladi.

ORM (Object-Relational Mapping) bu ishni soddalashtiradi: SQL o'rniga oddiy Python class va obyektlar bilan ishlaysiz. Flask uchun bu vosita — Flask-SQLAlchemy.
from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///baza.db"
db = SQLAlchemy(app)
# db — bu bazaga ulanish. Endi model yaratamiz.

Model Yaratish

Model — bu baza jadvalini tasvirlovchi Python class. Class bazadagi jadvalga, har bir db.Column jadval ustuniga, har bir obyekt esa bitta qatorga mos keladi.
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    ism = db.Column(db.String(80))
    yosh = db.Column(db.Integer)
# id — har qatorning yagona raqami (primary key)
# ism — matn ustuni, yosh — son ustuni
with app.app_context():
    db.create_all()
# Bu modelga qarab bazada jadval yaratadi

CRUD Amallari

Baza bilan ishlashda 4 ta asosiy amal bor. Ularning bosh harflari CRUD so'zini beradi:
AmalMa'nosiVazifasi
C — CreateQo'shishYangi qator yaratish
R — ReadO'qishMa'lumotni olish
U — UpdateYangilashMavjud qatorni o'zgartirish
D — DeleteO'chirishQatorni o'chirish

Create — qo'shish

yangi = User(ism="Ali", yosh=20)
db.session.add(yangi)
db.session.commit()
# add — navbatga qo'yadi, commit — bazaga yozadi

Read — o'qish

hammasi = User.query.all()
bittasi = User.query.get(1)
# all() — barcha qator, get(1) — id=1 bo'lgan qator

Update — yangilash

u = User.query.get(1)
u.yosh = 21
db.session.commit()
# Obyektni o'zgartirib, commit() qilamiz

Delete — o'chirish

u = User.query.get(1)
db.session.delete(u)
db.session.commit()
# delete — o'chirishga belgilaydi, commit — tasdiqlaydi
Eng Muhimi: commit()

db.session.add/delete — faqat rejani tayyorlaydi.
db.session.commit() — o'zgarishlarni bazaga haqiqatdan yozadi.
commit() qilmasang — hech narsa saqlanmaydi!

Misol 1: CRUD Amallari (jonli jadval)

Quyida soxta User jadvali. Qo'shish, o'qish, yangilash va o'chirish tugmalarini bosib, bazaga nima bo'lishini ko'ring.

Misol 2: Model → Jadval Mosligi

Model ustunlarini o'zgartiring va o'ngda bazada qanday jadval hosil bo'lishini ko'ring. Class — jadvalga, har bir db.Column — ustunga aylanadi.

Misol 3: Yangi Yozuv Qo'shib Ro'yxatni Yangilash

Bu xuddi haqiqiy ilovadagidek: yangi foydalanuvchi qo'shasiz va ro'yxat darhol yangilanadi. add() + commit() + query.all().

Sizning Mashqingiz

1
Bazani ulang

Flask-SQLAlchemy o'rnating va SQLITE URI ni sozlang (sqlite:///baza.db)

2
Model yarating

class Talaba(db.Model) — id, ism va ball ustunlari bilan, so'ng db.create_all()

3
Yozuv qo'shing (Create)

db.session.add(...) va db.session.commit() bilan 2-3 ta talaba qo'shing

4
O'qing va yangilang (Read + Update)

Talaba.query.all() bilan ro'yxatni chiqaring, bittasining ballini o'zgartirib commit() qiling
