Inkapsulatsiya Nima?

Inkapsulatsiya — bu obyekt ichidagi ma'lumotlarni himoyalash va yashirish degani. Ya'ni ba'zi xususiyatlarga tashqaridan to'g'ridan-to'g'ri tegib bo'lmaydi — ularga faqat maxsus metodlar orqali kirish mumkin.
Buni bank hisobi bilan tasavvur qiling. Bankdagi pulingizni (balansni) o'z qo'lingiz bilan o'zgartira olmaysiz — siz pul qo'yasiz yoki yechasiz, bank esa har bir amalni tekshiradi. Inkapsulatsiya ham aynan shunday ishlaydi.

Public Xususiyatlar

Public (ochiq) xususiyatlarga hamma joydan erkin kirish mumkin. Ularning oldida hech qanday belgi bo'lmaydi.
class Hisob:
    def __init__(self, egasi):
        self.egasi = egasi  # public

h = Hisob("Ali")
print(h.egasi)
h.egasi = "Vali"  # erkin o'zgartirish mumkin
# egasi — public xususiyat: o'qish ham, yozish ham mumkin
>>> Ali

Private Xususiyatlar (__balans)

Xususiyat nomi oldiga __ (ikkita pastki chiziq) qo'yilsa, u private (yashirin) bo'ladi. Unga klass tashqarisidan to'g'ridan-to'g'ri tegib bo'lmaydi.
class Hisob:
    def __init__(self, balans):
        self.__balans = balans  # private

h = Hisob(1000)
print(h.__balans)  # XATO!
# __balans — private. Tashqaridan kirish AttributeError beradi
>>> AttributeError: 'Hisob' object has no attribute '__balans'
Nega bu kerak?

Agar balansga hamma erkin tegsa, kimdir uni manfiy songa aylantirib qo'yishi mumkin. Private qilib, biz balansni noto'g'ri qiymatlardan himoyalaymiz — unga faqat tekshiruvdan o'tgan metodlar orqali kiriladi.

Metod Orqali Kirish (Getter / Setter)

Private xususiyatga kirish uchun maxsus metodlar yoziladi:
• getter — qiymatni o'qib beradi;
• setter — qiymatni tekshirib o'rnatadi.
class Hisob:
    def __init__(self):
        self.__balans = 0

    def pul_qoy(self, summa):
        if summa > 0:
            self.__balans += summa

    def balansni_kor(self):
        return self.__balans
# pul_qoy — setter (tekshiruv bilan), balansni_kor — getter
h = Hisob()
h.pul_qoy(500)
>>> print(h.balansni_kor()) → 500

@property Dekorator

@property — bu metodni xuddi oddiy xususiyatdek qavssiz chaqirish imkonini beradi. Shunda foydalanuvchi h.balans() emas, balki h.balans deb yozadi — lekin ortda baribir tekshiruvli metod ishlaydi.
class Hisob:
    def __init__(self):
        self.__balans = 0

    @property
    def balans(self):
        return self.__balans

    @balans.setter
    def balans(self, qiymat):
        if qiymat >= 0:
            self.__balans = qiymat
# h.balans → getter ishlaydi (qavssiz)
# h.balans = 200 → setter ishlaydi (tekshiruv bilan)
h = Hisob()
h.balans = 200  # setter chaqiriladi
>>> print(h.balans) → 200

Nega Metod Orqali Kirish Afzal?

• Metodga tekshiruv qo'shish mumkin (manfiy summani rad etish)
• Ma'lumotni noto'g'ri qiymatlardan himoyalaydi
• Keyinchalik ichki mantiqni o'zgartirsangiz, tashqi kod buzilmaydi
• Kod ishonchli va xavfsiz bo'ladi

Misol 1: Bank Hisobi (Private Balans)

Balans private. Unga to'g'ridan-to'g'ri tegib bo'lmaydi — faqat pul qo'yish / yechish metodlari orqali. Manfiy summa rad etiladi.

Misol 2: @property Getter

Balansni qavssiz o'qiymiz — h.balans. Ortda getter metodi ishlaydi.

Misol 3: Setter Validatsiyasi

h.balans = qiymat deganda setter ishlaydi va qiymatni tekshiradi. Manfiy son kiritsangiz — rad etiladi.

Sizning Mashqingiz

1
Hisob klassini yarating

__init__ ichida self.__balans = 0 (private) qiling

2
pul_qoy(summa) metodini yozing

Agar summa > 0 bo'lsa balansga qo'shing, aks holda rad eting

3
@property balans qo'shing

Getter __balans ni qaytarsin (qavssiz o'qiladi)

4
Setter validatsiyasini sinab ko'ring

Manfiy qiymat o'rnatishga urinib, rad etilishini tekshiring
