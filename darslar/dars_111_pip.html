Package Management (pip)

Modul 17 — Qo'shimcha Mavzular | Dars 111

pip Nima?

pip — bu Python uchun paket menejeri. U boshqalar yozgan tayyor kutubxonalarni (paketlarni) internetdan yuklab, kompyuteringizga o'rnatadi. Endi har bir narsani noldan yozish shart emas — kimdir allaqachon yozib qo'ygan!
Oddiy misol

Web saytga so'rov yubormoqchimisiz? requests paketini o'rnating.
Jadval bilan ishlamoqchimisiz? pandas paketini o'rnating.
Grafik chizmoqchimisiz? matplotlib paketini o'rnating.

PyPI nima?

Bu paketlar qayerda saqlanadi? PyPI (Python Package Index) — bu juda katta internet ombori, unda 500 000 dan ortiq tayyor paket bor. pip aynan shu ombordan paketlarni yuklab oladi. Sayti: pypi.org

Paket O'rnatish (install)

Paketni o'rnatish uchun terminalda (kodda emas!) pip install nomi buyrug'ini yozasiz.
$ pip install requests
# "requests" paketini internetdan yuklab o'rnatadi
Collecting requests
Downloading requests-2.31.0-py3-none-any.whl (62 kB)
Installing collected packages: requests
Successfully installed requests-2.31.0

O'rnatgandan keyin uni kodingizda import qila olasiz:

import requests
javob = requests.get("https://example.com")
print(javob.status_code)
# Endi paket tayyor — saytga so'rov yubordik

requirements.txt — Loyiha Bog'liqliklari

Loyihangiz qaysi paketlarga bog'liq (muhtoj)? Ularning hammasini bitta requirements.txt faylida ro'yxat qilib yozish odat. Shunda boshqa odam loyihangizni olganda, bitta buyruq bilan barcha paketlarni o'rnata oladi.

1) Mavjud paketlarni ro'yxatga olish (pip freeze)

$ pip freeze > requirements.txt
# O'rnatilgan paketlarni versiyalari bilan faylga yozadi

requirements.txt ichi shunday ko'rinadi:

requests==2.31.0
pandas==2.0.3
matplotlib==3.7.2

2) Ro'yxatdan o'rnatish (pip install -r)

$ pip install -r requirements.txt
# Fayldagi barcha paketlarni bir vaqtda o'rnatadi
Nega kerak?

Loyihangizni GitHub'ga qo'ysangiz, do'stingiz uni yuklab olib faqat pip install -r requirements.txt yozadi — va hamma kerakli paketlar bir zumda o'rnatiladi. Qaysi paket kerakligini esdan chiqarmaysiz!

Virtual Environment (venv)

venv — bu har bir loyiha uchun alohida, mustaqil muhit. Unda o'rnatilgan paketlar faqat shu loyihaga tegishli bo'ladi, boshqa loyihalarga aralashmaydi.
Nega kerak?

Tasavvur qiling: A loyiha pandas 1.0 ni, B loyiha esa pandas 2.0 ni talab qiladi. Agar hammasini bitta joyga o'rnatsangiz — ular bir-biriga xalal beradi! venv har bir loyihaga o'z "qutisini" beradi, shuning uchun ular tinch yashaydi.

1) Muhit yaratish

$ python -m venv venv
# "venv" nomli yangi muhit papkasini yaratadi

2) Muhitni faollashtirish (activate)

# Windows:
$ venv\Scripts\activate

# macOS / Linux:
$ source venv/bin/activate
# Faollashgach, terminal boshida (venv) yozuvi paydo bo'ladi
(venv) $ pip install requests

3) Muhitdan chiqish (deactivate)

(venv) $ deactivate
# Muhitdan chiqasiz, (venv) yozuvi yo'qoladi

• Paketlar PyPI (pypi.org) ombordan yuklanadi
