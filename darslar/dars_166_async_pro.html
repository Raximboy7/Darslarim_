Async — Ilg'or

Modul 23 — Pro daraja va Capstone | Dars 166

Avval Eslab Olamiz

Dars 100 da async / await bilan tanishgan edik. async funksiya — bu "kutish mumkin bo'lgan" funksiya. Uning ichida await bilan biror ish tugashini kutamiz, va o'sha kutish paytida Python boshqa ishni bajara oladi. Bu ayniqsa I/O (internet so'rovlari, fayl, baza) ishlarida juda foydali.
import asyncio

async def salom():
    await asyncio.sleep(1)  # 1 soniya kutamiz
    print("Salom!")

asyncio.run(salom())  # dasturni ishga tushirish
# async funksiya faqat asyncio.run() yoki await orqali ishlaydi
>>> Salom!

Muammo: Ketma-ket Kutish Sekin

Agar har bir ishni navbatma-navbat await qilsak, ular birin-ketin bajariladi. Uchta 1 soniyalik ish = 3 soniya. Bu sekin!
async def ish(n):
    await asyncio.sleep(1)
    return n

async def main():
    await ish(1)  # 1 soniya
    await ish(2)  # yana 1 soniya
    await ish(3)  # yana 1 soniya
# Jami: 3 soniya — chunki har biri oldingisini kutadi

Yechim 1: asyncio.gather (Parallel)

asyncio.gather() bir nechta async ishni bir vaqtda ishga tushiradi va hammasi tugashini kutadi. Uchta 1 soniyalik ish endi taxminan 1 soniyada tugaydi!
async def main():
    natijalar = await asyncio.gather(
        ish(1),
        ish(2),
        ish(3),
    )
    print(natijalar)  # tartibda qaytadi
# gather natijalarni siz bergan tartibda ro'yxat qilib qaytaradi
>>> [1, 2, 3]  # ~1 soniyada tayyor!

Yechim 2: asyncio.create_task

asyncio.create_task() bir ishni fonda darhol ishga tushiradi va sizga task obyektini beradi. Siz boshqa ishlar bilan davom etishingiz, keyin esa await task bilan natijani olishingiz mumkin.
async def main():
    t1 = asyncio.create_task(ish(1))
    t2 = asyncio.create_task(ish(2))
    # ikkalasi ham allaqachon ishlay boshladi
    a = await t1
    b = await t2
    print(a, b)
# create_task — ishni "boshlab qo'yish", await — natijani "yig'ib olish"
Farqi nimada?

• gather — bir joyda ko'p ishni boshlab, hammasini birga kutish uchun qulay.
• create_task — ishni boshlab qo'yib, oraliqda boshqa kod yozmoqchi bo'lsangiz qulay.

async with va async for

Oddiy with va for ni eslang. Async dunyoda ularning maxsus versiyalari bor: async with — async resursni ochib-yopish uchun, async for — async oqimdan element-element o'qish uchun.
async with session.get(url) as javob:
    data = await javob.json()

async for qator in oqim:
    print(qator)
# async with — resurs avtomatik yopiladi; async for — har element kelishini kutadi

aiohttp — Async HTTP So'rovlar

Dars 110 dagi requests kutubxonasi sinxron edi — har bir so'rov javob kelguncha kutib turardi. aiohttp esa async: ko'p so'rovlarni bir vaqtda yuborib, javoblarni parallel kutadi.
import aiohttp
import asyncio

async def olib_kel(session, url):
    async with session.get(url) as r:
        return await r.json()

async def main():
    async with aiohttp.ClientSession() as s:
        natijalar = await asyncio.gather(
            olib_kel(s, url1),
            olib_kel(s, url2),
        )
# Bitta ClientSession ochib, undan ko'p so'rovni parallel yuboramiz

Semaphore — Bir Vaqtda Nechta?

Agar 1000 ta so'rovni bir vaqtda yuborsangiz, server yoki kompyuteringiz ko'tara olmasligi mumkin. asyncio.Semaphore(n) bir vaqtda eng ko'pi bilan n ta ishga ruxsat beradi — qolganlari navbatda kutadi.
sem = asyncio.Semaphore(5)  # bir vaqtda 5 ta

async def olib_kel(url):
    async with sem:  # navbat kutadi
        return await sorov(url)
# async with sem: — bo'sh joy bo'lguncha kutadi, keyin kiradi

Qachon Async Ishlatamiz?

VaziyatAsync kerakmi?
Ko'p internet so'rovlari (API, sayt)Ha — juda foydali
Ko'p fayl yoki baza o'qish/yozishHa — I/O kutishni tejaydi
Og'ir matematik hisoblash (CPU)Yo'q — bu yerda foyda bermaydi
Bitta oddiy, tez so'rovYo'q — keraksiz murakkablik
Asosiy qoida

Async — kutish (I/O) ko'p bo'lganda foydali. Dastur kutib turgan paytda boshqa ishni qiladi. Hisoblash (CPU) og'ir bo'lsa, async yordam bermaydi — u yerda boshqa yondashuv (masalan, ko'p protsess) kerak.

Misol 1: Ketma-ket vs gather (Vaqt)

Uchta 1 soniyalik ishni ikki usulda solishtiramiz. Ketma-ket — jami 3 soniya; gather bilan parallel — taxminan 1 soniya.

Misol 2: create_task Parallel Ishi

create_task bilan boshlangan vazifalar bir vaqtda bajariladi. Tugmani bosing — har bir vazifaning progressi parallel to'lib boradi.

Misol 3: Ko'p So'rovni Parallel

Beshta "API so'rovi" ni soxta tarzda yuboramiz. Har biri har xil vaqt oladi. Sinxron — hammasining yig'indisi; async gather — eng uzunining vaqti.
