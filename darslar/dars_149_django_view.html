View va URL

Modul 21 — Django | Dars 149

View Nima?

View — bu foydalanuvchi so'roviga (request) javob qaytaruvchi funksiya. Brauzer biror manzilga (URL) kirsa, Django o'sha manzilga bog'langan view funksiyasini chaqiradi. View ishini bajaradi va javob (response) qaytaradi — brauzer shu javobni ekranga ko'rsatadi.
Sodda qilib aytganda

View = "agar shu manzilga kirilsa, mana shu javobni ber" degan ko'rsatma.

Function-Based View (FBV)

Eng oddiy view — bu oddiy Python funksiyasi. U doim birinchi argument sifatida request ni qabul qiladi va javob qaytaradi. Bunday view function-based view (funksiyaga asoslangan view) deyiladi.
# views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("Salom, dunyo!")
# request — so'rov; HttpResponse — oddiy matnli javob
>>> Brauzerda: Salom, dunyo!

HttpResponse va render()

HttpResponse — to'g'ridan-to'g'ri matn qaytaradi. render() esa HTML shabloni (template) ni va ma'lumotlarni birlashtirib chiroyli sahifa qaytaradi. Aksariyat hollarda biz render() dan foydalanamiz.
from django.shortcuts import render

def salom(request):
    ism = "Ali"
    return render(request, "salom.html", {"ism": ism})
# render(request, shablon nomi, ma'lumotlar lug'ati)

URL Routing — urls.py

Django qaysi manzilga qaysi view bog'langanini urls.py faylidagi urlpatterns ro'yxatidan biladi. Har bir manzil path() orqali bitta view ga ulanadi.
# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path("", views.home),
    path("salom/", views.salom),
]
# path(manzil, view) — "" bo'sh manzil bosh sahifa
URL → view bog'lanishi

Brauzer /salom/ ga kirsa → Django urlpatterns ni qatorma-qator tekshiradi → moslik topgan views.salom ni chaqiradi → javob qaytadi.

URL Parametrlar

Manzil ichida o'zgaruvchi qism bo'lishi mumkin. Masalan har bir foydalanuvchi uchun /user/1/, /user/2/ ... Buni <int:id> ko'rinishida yozamiz va view ga argument bo'lib uzatiladi.
# urls.py
path("user/<int:id>/", views.profil),

# views.py
def profil(request, id):
    return HttpResponse(f"Foydalanuvchi: {id}")
# <int:id> — manzildagi raqam id nomli argumentga tushadi
>>> /user/5/ → Foydalanuvchi: 5
PatternTuriMosligi
<int:id>Butun son/user/7/
<str:nom>Matn/user/ali/
<slug:s>slug (matn-raqam)/post/yangi-maqola/
Eslab Qoling

• View — request oladi, javob qaytaradi
• HttpResponse — matn, render() — HTML shablon
• urls.py manzilni view ga bog'laydi (path())
• <int:id> — manzildan parametr olish
