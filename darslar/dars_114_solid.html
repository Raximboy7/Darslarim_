SOLID nima?

SOLID — yaxshi OOP dizayn uchun 5 ta tamoyilning bosh harflaridan tuzilgan qisqartma. Bu tamoyillar kodni tushunarli, o'zgartirish oson va xatosiz qilishga yordam beradi.

S — Single Responsibility

Har bir klass faqat bitta ishga javob bersin. Talabani saqlash va uni faylga yozish — ikki xil vazifa, ikki xil klass bo'lishi kerak.
# YAXSHI: har klass bitta ish qiladi
class Talaba:
    def __init__(self, ism):
        self.ism = ism

class TalabaSaqlovchi: # faqat saqlash
    def saqla(self, t): ...
# Ma'lumot (Talaba) va uni saqlash (TalabaSaqlovchi) ajratildi

O — Open/Closed

Klass yangi imkoniyat qo'shishga ochiq, lekin mavjud kodni o'zgartirishga yopiq bo'lsin. Yangi shakl qo'shsangiz, eski kodni buzmang — meros orqali kengaytiring.
class Shakl:
    def yuza(self): pass

class Doira(Shakl): # yangi shakl
    def yuza(self): return 3.14 * self.r ** 2
# Yangi Doira qo'shildi — Shakl klassi o'zgarmadi

L — Liskov Substitution

Bola klass otasi o'rniga muammosiz ishlatilishi kerak. Agar Qush uchadi deb belgilangan bo'lsa, undan meros olgan Pingvin ucholmasa — bu tamoyil buziladi.
Oddiy qoida ✅

Otani ishlatgan har qanday joyda bolani qo'ysangiz ham dastur to'g'ri ishlashi kerak. Agar bola otaning va'dasini buzsa (masalan, xato bersa) — meros noto'g'ri tuzilgan.

I — Interface Segregation

Klassga kerak bo'lmagan metodlarni majburlamang. Bitta "hamma narsani qiladigan" katta interfeys o'rniga, kichik va aniq interfeyslar yasang.
# YAXSHI: kichik, aniq interfeyslar
class Ishchi:
    def ishla(self): ...
class Ovqatlanuvchi:
    def ovqatlan(self): ...
# Robot faqat Ishchi bo'ladi — unga ovqatlan() majburlanmaydi

D — Dependency Inversion

Klasslar aniq (konkret) klasslarga emas, abstraksiyaga bog'lansin. Masalan, Xabar klassi to'g'ridan "SMS" ga emas, umumiy "Yuboruvchi" interfeysiga bog'lansa — keyin SMS ni Email ga oson almashtirasiz.
class Yuboruvchi: # abstraksiya
    def yubor(self, matn): pass

class Xabar:
    def __init__(self, yuboruvchi):
        self.y = yuboruvchi # istalgan Yuboruvchi
# Xabar aniq SMS ga emas, umumiy Yuboruvchi'ga bog'landi → almashtirish oson

Xulosa

SOLID — 5 tamoyil

• S — bitta klass, bitta vazifa
• O — kengaytirishga ochiq, o'zgartirishga yopiq
• L — bola otani almashtira olsin
• I — keraksiz metod majburlanmasin
• D — abstraksiyaga bog'lan
Bu tamoyillar katta loyihalarni tartibli va kengaytiriladigan qiladi.
