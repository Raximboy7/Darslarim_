__str__ va __repr__ Metodlari

Modul 7 — OOP (Obyektga asoslangan dasturlash) | Dars 36

Obyektni print() qilsak nima bo'ladi?

Tasavvur qiling, biz class yaratdik va undan obyekt yasadik. Endi shu obyektni print() qilamiz. Python bizga chiroyli matn emas, balki xunuk, tushunarsiz bir narsa chiqaradi.
class Kishi:
    def __init__(self, ism, yosh):
        self.ism = ism
        self.yosh = yosh

p = Kishi("Ali", 25)
print(p)
# Biz "Ali, 25" kutamiz, lekin...
>>> <__main__.Kishi object at 0x7f8a1c0d2e50>
Bu nima degani? Python: "Bu Kishi obyekti, xotirada 0x7f8a1c0d2e50 manzilda turibdi" deydi. Bu foydalanuvchi uchun butunlay foydasiz! Aynan shu muammoni hal qilish uchun __str__ va __repr__ metodlari kerak.

__str__ — foydalanuvchi uchun chiroyli matn

__str__() metodi obyektni chiroyli, o'qishga qulay matn ko'rinishiga aylantiradi. Bu metod print() va str() chaqirilganda ishlaydi. U doimo matn (string) qaytarishi shart.
class Kishi:
    def __init__(self, ism, yosh):
        self.ism = ism
        self.yosh = yosh

    def __str__(self):
        return f"{self.ism}, {self.yosh} yoshda"

p = Kishi("Ali", 25)
print(p)
# Endi print() chiroyli matn chiqaradi
>>> Ali, 25 yoshda

__repr__ — dasturchi va debug uchun aniq tasvir

__repr__() metodi obyektning aniq, texnik tasvirini beradi. U asosan dasturchi uchun — xatolarni topish (debug) va konsolda tekshirish uchun. Yaxshi qoida: __repr__ shunday matn qaytarsinki, uni nusxalab yana shu obyektni yaratish mumkin bo'lsin.
class Kishi:
    def __init__(self, ism, yosh):
        self.ism = ism
        self.yosh = yosh

    def __repr__(self):
        return f"Kishi('{self.ism}', {self.yosh})"

p = Kishi("Ali", 25)
print(repr(p))
# repr() texnik tasvirni chiqaradi
>>> Kishi('Ali', 25)

Ikkisining farqi va qachon qaysi biri ishlaydi

Eng muhim qoida: print() avval __str__ ni qidiradi. Agar topmasa, __repr__ ga o'tadi. Demak, __repr__ — zaxira (asosiy) tasvir, __str__ esa foydalanuvchi uchun chiroyli ustki qatlam.

Maslahat

Agar ikkalasidan faqat bittasini yozmoqchi bo'lsangiz, __repr__ ni yozing. Chunki u print() uchun ham zaxira bo'lib xizmat qiladi.

To'liq misol: ikkalasi birga

Ko'p hollarda bitta classda ikkala metod ham yoziladi: biri foydalanuvchi uchun, biri dasturchi uchun.
class Kishi:
    def __init__(self, ism, yosh):
        self.ism = ism
        self.yosh = yosh

    def __str__(self):
        return f"{self.ism}, {self.yosh} yoshda"

    def __repr__(self):
        return f"Kishi('{self.ism}', {self.yosh})"

p = Kishi("Ali", 25)
print(p)
print(repr(p))
# print → __str__, repr() → __repr__
>>> Ali, 25 yoshda
>>> Kishi('Ali', 25)
Eslab Qoling

• __str__ — foydalanuvchi uchun chiroyli matn (print)
• __repr__ — dasturchi uchun aniq tasvir (debug)
• print() avval __str__ ni, topmasa __repr__ ni ishlatadi
• Ikkalasi ham doimo return bilan matn qaytaradi
• Faqat bittasini yozsangiz — __repr__ ni yozing

Misol 1: __str__ siz vs bilan

Bir tugma __str__ siz classni, ikkinchisi __str__ bilan classni print qilganda nima chiqishini ko'rsatadi. Farqni o'z ko'zingiz bilan ko'ring.

Misol 2: __str__ vs __repr__ yonma-yon

Bitta obyekt, ikki xil chiqish. print(p) __str__ ni, repr(p) __repr__ ni chaqiradi. Tugmani bosing va ikkalasini taqqoslang.
print(p) → __str__
repr(p) → __repr__

Misol 3: O'z Kishingizni yarating

Ism va yoshni kiriting. Python __str__ va __repr__ natijasini siz uchun hosil qiladi.
