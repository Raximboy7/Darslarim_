Modul 22 — DevOps (Docker va CI/CD) | Dars 162

Monitoring Nima?

Monitoring — bu ishlab turgan ilovangiz holatini doimiy kuzatib borish. Xuddi shifokor bemorning yurak urishini, harorati va qon bosimini kuzatgani kabi, biz ham serverdagi ilovamizning "salomatligini" kuzatamiz: u ishlayaptimi, tez javob beryaptimi, xato bermayaptimi?

Ilova foydalanuvchiga ko'rinmaydigan serverlarda ishlaydi. Monitoring bo'lmasa, biror narsa buzilganini faqat foydalanuvchi shikoyat qilgandan keyin bilamiz. Monitoring bilan esa muammoni foydalanuvchidan oldin sezamiz.

Asosiy maqsad

Monitoring muammoni tez topish va ideal holatda oldini olish uchun kerak. Tez topilgan muammo — arzon muammo.

1) Loglar — Ilovaning Kundaligi

Log — bu ilova nima qilayotganini yozib boradigan matn satrlari. Xuddi kundalik kabi: "soat 14:05 da foydalanuvchi kirdi", "soat 14:07 da xato yuz berdi". Har satr odatda vaqt, daraja va xabar dan iborat.

Loglar darajalari

DarajaMa'nosiMisol
INFOOddiy ma'lumot, hammasi joyida"Foydalanuvchi kirdi"
WARNINGDiqqat, e'tibor bering"Xotira 80% to'ldi"
ERRORXato yuz berdi"Bazaga ulanib bo'lmadi"

Pythonda log yozish

import logging
logging.basicConfig(level=logging.INFO)

logging.info("Foydalanuvchi kirdi")
logging.error("Bazaga ulanib bo'lmadi")
# logging moduli loglarni chiroyli yozadi
# info — oddiy xabar, error — xato xabar

Docker loglarini ko'rish

docker logs mening_ilovam
docker logs -f mening_ilovam
# konteyner chiqargan barcha loglarni ko'rsatadi
# -f (follow) — yangi loglarni jonli kuzatadi

2) Metriklar — Raqamli Ko'rsatkichlar

Metrik — bu vaqt o'tishi bilan o'lchanadigan raqam. Loglar "nima bo'ldi" deb aytsa, metriklar "qancha" degan savolga javob beradi.
MetrikNimani o'lchaydiYaxshi qiymat
CPUProtsessor band darajasi< 70%
Xotira (RAM)Ishlatilgan operativ xotira< 80%
So'rovlar soniSoniyada nechta so'rov keladikuzatib turiladi
Javob vaqtiSo'rovga qancha vaqtda javob< 300 ms

3) Alerting — Ogohlantirish

Alerting — metrik xavfli chegaradan oshganda sizga avtomatik xabar yuborish. Masalan: "CPU 90% dan oshsa, menga SMS yubor". Shunda siz ekranni doimiy kuzatib o'tirmaysiz — tizim o'zi sizni chaqiradi.
Chegara (threshold)

Alert qoidasi: agar metrik > chegara bo'lsa → ogohlantir. Chegarani juda past qo'ysangiz — keraksiz xabarlar ko'payadi; juda baland qo'ysangiz — muammoni kech bilasiz.

Asosiy Vositalar

VositaNima qiladi (sodda)
PrometheusMetriklarni yig'adi va saqlaydi (raqamlar ombori)
GrafanaMetriklarni chiroyli grafik va dashboardlarda ko'rsatadi
SentryIlovadagi xatolarni tutib oladi va sizga xabar beradi

Health Check — Salomatlik Tekshiruvi

Health check — ilova "men tirikman" deb javob beradigan maxsus manzil (masalan /health). Tizim har necha soniyada shu manzilga so'rov yuboradi: 200 OK kelsa — ilova sog'lom; javob kelmasa — ilova o'chgan, qayta ishga tushiriladi.
# Oddiy health check endpoint (Flask)
@app.route("/health")
def health():
    return {"status": "ok"}, 200
# /health manziliga kirilganda "ok" qaytaradi
# tizim shu javobga qarab ilova tirikligini biladi
Eslab Qoling

• Log — nima bo'ldi (matn kundalik)
• Metrik — qancha (raqamli ko'rsatkich)
• Alert — chegaradan oshsa ogohlantirish
• Health check — ilova tirikmi degan tekshiruv
• Vositalar: Prometheus (yig'ish), Grafana (ko'rsatish), Sentry (xatolar)

Misol 1: Loglardan Muammoni Topish

Quyida ilova loglari. Faqat ERROR satrlarini ajratib ko'rsataylik — xuddi docker logs | grep ERROR kabi.

Misol 2: Metriklar Dashboard

"Yangilash" tugmasi serverdan yangi o'lchovlar olishni taqlid qiladi. Bar yashil — xavfsiz, sariq — diqqat, qizil — xavfli.
