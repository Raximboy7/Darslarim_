TDD nima?

TDD (Test Driven Development) — "test yetakchiligidagi dasturlash". Bu yondashuvda avval test yoziladi, keyin uni o'tkazadigan kod yoziladi. Ya'ni "avval kutilgan natijani belgila, keyin kodni yoz".

Odatda biz avval kod yozib, keyin sinaymiz. TDD buni teskarisiga qiladi: avval test, keyin kod. Bu g'alati tuyulishi mumkin, lekin kodni ancha ishonchli qiladi.

Red — Green — Refactor sikli

TDD uch qadamdan iborat va bu qadamlar takrorlanadi:

🔴
Red — test yoz (u yiqiladi)

Hali kod yo'q, shuning uchun test o'tmaydi. Bu normal.

🟢
Green — testni o'tkazadigan kod yoz

Eng oddiy kod bilan testni "yashil" qil.

🔵
Refactor — kodni yaxshila

Testlar hali ham o'tayotganiga ishonch hosil qilib, kodni tozala.

1-qadam: RED (avval test)

Aytaylik, ikki sonni qo'shadigan qosh() funksiyasi kerak. Avval testini yozamiz — funksiya hali yo'q!

# test_qosh.py
from hisob import qosh

def test_qosh():
    assert qosh(2, 3) == 5
    assert qosh(0, 0) == 0
# qosh() hali mavjud emas, shuning uchun test yiqiladi (RED). Biz avval "nima kutayotganimizni" belgiladik.
>>> ImportError: cannot import name 'qosh'

2-qadam: GREEN (testni o'tkaz)

Endi testni o'tkazadigan eng oddiy kodni yozamiz:

# hisob.py
def qosh(a, b):
    return a + b
# Endi test o'tadi (GREEN). Ortiqcha narsa yozmadik — faqat testni yashil qiladigan minimal kod.
>>> test_qosh PASSED ✓

3-qadam: REFACTOR (yaxshilash)

Kod ishlayapti va testlar o'tyapti. Endi kodni tozalash, nomlarni yaxshilash, takrorlarni yo'q qilish mumkin — testlar bizni himoya qiladi.

Refactor nima uchun xavfsiz?

Kodni o'zgartirgandan keyin testlarni qayta ishlatasiz. Agar hammasi yashil bo'lsa — o'zgarish hech narsani buzmagan. Test bo'lmasa, har o'zgarishda qo'rqib ishlar edingiz.

TDD ning afzalliklari

✅ Ishonch — kod kutilganday ishlashiga aminsiz

✅ Kam xato — muammolar erta bosqichda topiladi

✅ Qo'rqmasdan o'zgartirish — testlar buzilishni darhol ko'rsatadi

✅ Aniq talab — test o'zi kod nima qilishi kerakligini hujjatlaydi

Xulosa

Eslab qoling

• TDD — avval test, keyin kod
• Sikl: Red → Green → Refactor
• Red — test yiqiladi (kod yo'q)
• Green — testni o'tkazadigan minimal kod
• Refactor — testlar himoyasida kodni yaxshilash

TDD boshida sekin tuyuladi, lekin katta loyihalarda ko'p vaqt va asabni tejaydi.
