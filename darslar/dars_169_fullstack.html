1. Ma'lumotlar Bazasini Tayyorlash

SQLite Python bilan birga keladi (o'rnatish shart emas). Avval vazifalar jadvalini yaratamiz — har vazifada id va matn bo'ladi.
import sqlite3

def baza_tayyorla():
    conn = sqlite3.connect("vazifalar.db")
    conn.execute('''
      CREATE TABLE IF NOT EXISTS vazifalar (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        matn TEXT NOT NULL)
    ''')
    conn.commit()
    conn.close()
# IF NOT EXISTS — jadval bo'lsa qayta yaratmaydi; id avtomatik oshadi; commit — o'zgarishni saqlaydi

2. app.py — Flask Routes + CRUD

Endi asosiy fayl. Bosh sahifada barcha vazifalar ko'rsatiladi (Read), forma orqali yangi vazifa qo'shiladi (Create), tugma orqali o'chiriladi (Delete).
from flask import Flask, request, redirect
import sqlite3

app = Flask(__name__)

def db():
    return sqlite3.connect("vazifalar.db")

@app.route("/")
def bosh():
    conn = db()
    rows = conn.execute("SELECT id, matn FROM vazifalar").fetchall()
    conn.close()
    html = "<h1>Vazifalar</h1>"
    for id, matn in rows:
        html += f"<p>{matn} <a href='/ochir/{id}'>x</a></p>"
    html += '''<form method="POST" action="/qoshish">
      <input name="matn"><button>Qo'shish</button></form>'''
    return html
# SELECT — vazifalarni o'qiydi (Read); har vazifa yonida o'chirish havolasi; pastda qo'shish formasi

3. Qo'shish va O'chirish Route'lari

Forma POST qilganda /qoshish ishlaydi va bazaga yozadi. O'chirish havolasi /ochir/<id> ni chaqiradi. Har ikkalasi ish tugagach redirect("/") bilan bosh sahifaga qaytaradi.
@app.route("/qoshish", methods=["POST"])
def qoshish():
    matn = request.form["matn"]
    conn = db()
    conn.execute("INSERT INTO vazifalar (matn) VALUES (?)", (matn,))
    conn.commit()
    conn.close()
    return redirect("/")

@app.route("/ochir/<int:id>")
def ochir(id):
    conn = db()
    conn.execute("DELETE FROM vazifalar WHERE id = ?", (id,))
    conn.commit()
    conn.close()
    return redirect("/")

if __name__ == "__main__":
    baza_tayyorla()
    app.run(debug=True)
# ? va (matn,) — xavfsiz parametr (SQL injection dan himoya); <int:id> — URL dan sonni oladi; redirect — bosh sahifaga qaytaradi
Muhim: xavfsizlik

SQL ga qiymatni hech qachon to'g'ridan-to'g'ri qo'shmang. Doim ? belgisi va tuple ishlating: execute("... VALUES (?)", (matn,)) — bu SQL injection hujumidan himoya qiladi.
