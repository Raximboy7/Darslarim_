Modul 21 — Django | Dars 148

Model Nima?

Model — bu ma'lumotlar bazasidagi (database) jadvalni Python klassi sifatida ifodalash usuli. Ya'ni siz SQL tilini bilmasangiz ham, oddiy Python klassi yozib, Django o'zi uni bazadagi jadvalga aylantiradi.
Oddiy taqqoslash

• Klass → bazadagi jadval (table)
• Klass maydoni (field) → jadvaldagi ustun (column)
• Obyekt (nusxa) → jadvaldagi qator (row)

Model Yaratish

Modellar har bir Django ilovaning models.py faylida yoziladi. Har bir model models.Model dan meros oladi.
from django.db import models

class Maqola(models.Model):
    sarlavha = models.CharField(max_length=200)
    matn = models.TextField()
    sana = models.DateTimeField(auto_now_add=True)
# Maqola nomli model — bazada "maqola" jadvali bo'ladi.
# sarlavha, matn, sana — jadvaldagi ustunlar (maydonlar).
Eslatma

Django avtomatik ravishda har bir modelga id degan maxsus maydon qo'shadi — bu har bir qatorning takrorlanmas raqami (primary key). Siz uni yozishingiz shart emas.

Maydon Turlari (Field Types)

Har bir maydon qanday turdagi ma'lumot saqlashini bildiradi. Eng ko'p ishlatiladigan turlar:
Maydon turiNima saqlaydiMisol
CharFieldQisqa matn (uzunligi cheklangan)ism, sarlavha
TextFieldUzun matn (cheksiz)maqola matni, izoh
IntegerFieldButun sonyosh, miqdor
DateTimeFieldSana va vaqtyaratilgan sana
BooleanFieldTrue yoki Falsefaolmi, e'lon qilinganmi
CharField va TextField farqi

CharField da max_length majburiy (masalan, ism uchun 100 belgi yetarli). TextField esa uzunlik cheklamasiz — uzun matnlar (maqola, izoh) uchun ishlatiladi.

__str__ Metodi

__str__ metodi obyekt chop etilganda yoki admin panelida ko'rsatilganda qanday matn ko'rinishida chiqishini belgilaydi. U bo'lmasa, Django "Maqola object (1)" kabi tushunarsiz matn ko'rsatadi.
class Maqola(models.Model):
    sarlavha = models.CharField(max_length=200)

    def __str__(self):
        return self.sarlavha
# Endi har bir maqola o'z sarlavhasi bilan ko'rsatiladi.
# Masalan: "Birinchi maqolam" — chiroyli va tushunarli.

Migration — Modelni Bazaga Aylantirish

Model faqat Python kodi — bazada hali jadval yo'q. Modelni haqiqiy jadvalga aylantirish uchun migration (ko'chirish) jarayoni kerak. Bu ikki buyruq orqali bajariladi.
python manage.py makemigrations
# 1-qadam: Django modeldagi o'zgarishlarni topadi va
# migration faylini (o'zgarish rejasini) tayyorlaydi.
python manage.py migrate
# 2-qadam: Migration rejasini bajaradi va
# bazada haqiqiy jadvalni yaratadi (yoki o'zgartiradi).
Oddiy qilib aytganda

• makemigrations = "Nima o'zgardi?" deb rejani yozadi
• migrate = rejani bajaradi va bazani yangilaydi
Modelni har safar o'zgartirsangiz, bu ikki buyruqni qaytadan ishga tushirasiz.
