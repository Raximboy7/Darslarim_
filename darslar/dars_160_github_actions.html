GitHub Actions Nima?

GitHub Actions — bu GitHub'ning ichiga qurilgan CI/CD tizimi. Ya'ni kodingizni GitHub'ga yuborganingizda (push) u avtomatik ravishda testlarni ishga tushiradi, loyihani yig'adi va kerak bo'lsa serverga joylashtiradi — barchasini siz hech narsa qilmasdan.
Oddiy qilib aytganda

Siz kod yozasiz → GitHub'ga yuborasiz → GitHub Actions o'zi:
✅ testlarni ishga tushiradi
✅ xatolarni topadi
✅ natijani ko'rsatadi (yashil ✓ yoki qizil ✗)

Eng yaxshi tomoni — GitHub Actions GitHub ichida ishlaydi. Boshqa xizmatga ro'yxatdan o'tish shart emas. Repozitoriya ichida bitta YAML fayl yaratsangiz kifoya.

Workflow Fayli Qayerda Bo'ladi?

GitHub Actions workflow (ish jarayoni) deb ataladigan fayllar bilan ishlaydi. Bu fayllar repozitoriyaning maxsus papkasida — .github/workflows/ ichida saqlanadi va YAML formatida yoziladi.
loyiham/
├── .github/
│   └── workflows/
│      └── ci.yml  # workflow fayli
├── main.py
└── test_main.py
# Workflow fayli .github/workflows/ ichida bo'lishi shart
YAML nima?

YAML — sozlamalarni yozish uchun oddiy matn formati. Unda chekinish (bo'sh joy) muhim: har bir daraja 2 ta bo'sh joy bilan ichkariga suriladi. Fayl kengaytmasi .yml yoki .yaml bo'ladi.

Asosiy Tushunchalar

Kalit so'zMa'nosi
nameWorkflow nomi (GitHub'da ko'rinadi)
onTrigger — qachon ishga tushishi (masalan push)
jobsBajariladigan ishlar to'plami
runs-onQaysi operatsion tizimda ishlashi (masalan ubuntu-latest)
stepsIsh ichidagi qadamlar (ketma-ket bajariladi)
usesTayyor action (qadam) ishlatish
runTerminal buyrug'ini bajarish
Eslab qoling: on — bu "qachon", jobs — bu "nima qilinadi", steps — bu "qanday qadamlar bilan".

To'liq Misol: Push'da Testlarni Ishga Tushirish

Quyidagi .github/workflows/ci.yml fayli har safar kod GitHub'ga push qilinganda Python testlarini avtomatik ishga tushiradi.
name: CI

on: [push]  # push bo'lganda ishga tush

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install pytest
      - run: pytest
# checkout → kodni yuklab oladi · setup-python → Python o'rnatadi · run → buyruqni bajaradi
>>> GitHub: CI workflow ishga tushdi...
>>> ✓ checkout · ✓ setup-python · ✓ pip install · ✓ pytest
>>> Natija: Hammasi muvaffaqiyatli (yashil ✓)

Marketplace Actions (Tayyor Qadamlar)

Har bir qadamni noldan yozish shart emas. GitHub'da Marketplace bor — u yerda minglab tayyor action'lar mavjud. Ularni uses: orqali chaqirasiz.
ActionVazifasi
actions/checkout@v4Repozitoriya kodini ish muhitiga yuklab oladi
actions/setup-python@v5Kerakli versiyadagi Python'ni o'rnatadi
actions/setup-node@v4Node.js o'rnatadi
actions/cache@v4Kutubxonalarni keshlab tezlashtiradi
@v4 nimani anglatadi?

@v4 — bu action'ning versiyasi. Doimo aniq versiya yozish tavsiya etiladi, shunda workflow kutilmaganda buzilmaydi.

Misol 1: Workflow YAML Tuzilmasi

Quyidagi tugmalarni bosib, workflow faylining har bir qismi nima ish qilishini ko'ring.

Misol 2: Push → Workflow Ishga Tushadi

Trigger'ni tanlang va "Hodisa yuz berdi" tugmasini bosing. Workflow shu triggerda ishga tushishini ko'rasiz.

Misol 3: Steps Ketma-ket Bajariladi

"Workflow'ni ishga tushir" tugmasini bosing. Qadamlar checkout → setup → test tartibida birin-ketin bajarilishini kuzating.
📥checkout
🐍setup-python
📦pip install
🧪pytest
Eslab Qoling

• Workflow fayli .github/workflows/ci.yml da bo'ladi
• on — qachon, jobs — nima, steps — qanday qadamlar
• runs-on ish muhitini (masalan ubuntu-latest) belgilaydi
• Tayyor action'lar: actions/checkout, actions/setup-python
• Steps yuqoridan pastga, ketma-ket bajariladi
