Stack va Queue

Modul 11 — Algoritmlar | Dars 64

Bu Darsda Nima O'rganamiz?

Stack (uyum) va Queue (navbat) — bu ma'lumotlarni saqlash va ulardan foydalanishning ikkita eng muhim usuli. Ular oddiy ko'rinadi, lekin deyarli har bir dasturda ishlatiladi: undo tugmasidan tortib navbat tizimigacha. Farqi faqat bitta savolda: elementlarni qaysi tartibda olib chiqamiz?

Stack Nima? (LIFO)

Stack — bu elementlar ustma-ust to'planadigan tuzilma. Faqat eng tepadagi element bilan ishlaymiz. U LIFO qoidasiga bo'ysunadi: Last In, First Out — oxirgi kirgan birinchi chiqadi.
Tarelka uyumi analogiyasi

Oshxonadagi tarelkalar uyumini tasavvur qiling. Yangi tarelkani eng tepaga qo'yasiz. Tarelka kerak bo'lsa, yana eng tepadagisini olasiz. Eng pastdagi tarelka — eng oxirida olinadi. Stack ham xuddi shunday ishlaydi.

push va pop

AmalVazifasi
pushTepaga yangi element qo'shadi
popTepadagi elementni olib chiqadi

Python list bilan Stack

Python'da stack uchun maxsus narsa kerak emas — oddiy list yetarli. .append() bilan tepaga qo'shamiz, .pop() bilan tepadan olamiz.
stack = []
stack.append("A")
stack.append("B")
stack.append("C")
print(stack)
# Uchta elementni tepaga qo'shdik
# Stack endi: A (pastda), B, C (tepada)
>>> ['A', 'B', 'C']
oxirgi = stack.pop()
print(oxirgi)
print(stack)
# pop() tepadagini ("C") olib chiqadi va qaytaradi
# LIFO: oxirgi kirgan birinchi chiqdi
>>> C
>>> ['A', 'B']

Queue Nima? (FIFO)

Queue — bu navbat. Elementlar bir tomondan kiradi, boshqa tomondan chiqadi. U FIFO qoidasiga bo'ysunadi: First In, First Out — birinchi kirgan birinchi chiqadi.
Do'kondagi navbat analogiyasi

Do'kon kassasidagi navbatni tasavvur qiling. Kim birinchi kelgan bo'lsa, u birinchi xizmat oladi va birinchi ketadi. Yangi kelgan odam navbat oxiriga turadi. Bu adolatli tartib — Queue ham shunday.

enqueue va dequeue

AmalVazifasi
enqueueNavbat oxiriga element qo'shadi
dequeueNavbat boshidan elementni olib chiqadi

collections.deque bilan Queue

Queue uchun oddiy list ishlatish mumkin, lekin boshidan element olish (list.pop(0)) sekin bo'ladi. Shuning uchun Python'da maxsus deque (double-ended queue) bor — u ikki tomondan ham tez ishlaydi.
from collections import deque

navbat = deque()
navbat.append("Ali")
navbat.append("Vali")
navbat.append("Guli")
print(navbat)
# enqueue = append() — oxiriga qo'shamiz
# Navbat: Ali (bosh), Vali, Guli (oxir)
>>> deque(['Ali', 'Vali', 'Guli'])
birinchi = navbat.popleft()
print(birinchi)
print(navbat)
# dequeue = popleft() — boshidan olamiz
# FIFO: birinchi kelgan ("Ali") birinchi chiqdi
>>> Ali
>>> deque(['Vali', 'Guli'])

Real Misollar

Stack ishlatiladigan joylar

• Undo (orqaga qaytarish) — oxirgi amalni birinchi bekor qilamiz
• Brauzer "Orqaga" tugmasi — oxirgi ochilgan sahifaga qaytamiz
• Qavslarni tekshirish — kod yozishda ( ) mosligi

Queue ishlatiladigan joylar

• Printer navbati — birinchi yuborilgan birinchi bosiladi
• Xabarlar navbati — kelgan tartibda qayta ishlanadi
• Mijozlarga xizmat — kim oldin kelsa, oldin xizmat oladi

Eslab Qoling

• Stack = LIFO — oxirgi kirgan birinchi chiqadi
• Queue = FIFO — birinchi kirgan birinchi chiqadi
• Stack: append() + pop()
• Queue: append() + popleft() (deque)

Misol 1: Stack Vizual (push / pop)

Element qo'shing va oling. Yangi element doim tepaga qo'shiladi, pop() esa har doim tepadagini oladi.
⬆ Tepa (bu yerdan qo'shamiz va olamiz)

Misol 2: Queue Vizual (enqueue / dequeue)

Element navbat oxiriga qo'shiladi, dequeue esa navbat boshidan oladi. Bu adolatli tartib — birinchi kelgan birinchi ketadi.
⬅ Bosh (bu yerdan chiqadi)Oxir (bu yerga kiradi) ➡

Misol 3: LIFO va FIFO Farqi

Bir xil elementlarni 1, 2, 3 tartibda qo'shamiz. Endi hammasini birma-bir olib chiqsak — Stack va Queue qaysi tartibda beradi? Solishtiring!
