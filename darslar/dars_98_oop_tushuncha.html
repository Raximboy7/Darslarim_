OOP nima?

OOP (Object-Oriented Programming) — obyektga yo'naltirilgan dasturlash. Bunda biz dasturni obyektlar (real dunyodagi narsalarga o'xshash) ko'rinishida quramiz. Har bir obyekt o'zining ma'lumoti (xususiyat) va harakati (metod)ga ega.
Nega OOP kerak? 🏗️

Katta dasturlarda minglab funksiya va o'zgaruvchi bo'ladi. OOP ularni mantiqiy guruhlarga (klasslarga) bo'lib, kodni tartibli, qayta ishlatiladigan va oson kengaytiriladigan qiladi. OOP asosida 4 ta ustun (tamoyil) turadi.

1-ustun: Inkapsulyatsiya

Obyektning ichki ma'lumotlarini tashqaridan berkitish. Ma'lumotga to'g'ridan-to'g'ri emas, balki metodlar orqali murojaat qilinadi. Bu ma'lumotni noto'g'ri o'zgartirishdan himoya qiladi.
class Hisob:
    def __init__(self, pul):
        self.__balans = pul  # yashirin

    def balans_ko'r(self):
        return self.__balans
# __balans — ikki pastki chiziq bilan "yashirin" qilinadi.
# Tashqaridan hisob.__balans ishlamaydi — faqat metod orqali

2-ustun: Meros (Inheritance)

Bir klass (bola) boshqa klassdan (ota) barcha xususiyat va metodlarni meros qilib oladi. Kodni takrorlamaslikka yordam beradi.
class Hayvon:
    def nafas_ol(self):
        print("Nafas olyapman")

class It(Hayvon):  # meros
    def vov(self):
        print("Vov-vov!")

rex = It()
rex.nafas_ol()  # Hayvondan meros
rex.vov()
# It — Hayvondan meros oldi. nafas_ol() ni qayta yozmasdan ishlatadi
>>> Nafas olyapman
>>> Vov-vov!

3-ustun: Polimorfizm

Bir nom — ko'p shakl. Turli klasslar bir xil nomdagi metodni o'zicha bajaradi. Masalan, har xil hayvon ovoz() metodini boshqacha bajaradi.
class Mushuk:
    def ovoz(self): return "Miyov"
class It:
    def ovoz(self): return "Vov"

for h in [Mushuk(), It()]:
    print(h.ovoz())
# Bir xil ovoz() chaqiruvi — lekin har obyekt o'zicha javob beradi
>>> Miyov
>>> Vov

4-ustun: Abstraksiya

Foydalanuvchiga faqat kerakli qismni ko'rsatib, ichki murakkab ishlarni yashirish. Masalan, mashina haydashda gaz pedalini bosasiz — dvigatel qanday ishlashini bilishingiz shart emas.
Hayotiy misol 🚗

mashina.yur() deb chaqirasiz. Ichida yonilg'i, porshen, motor bilan bo'layotgan yuzlab jarayonni bilish shart emas — abstraksiya ularni yashiradi. Bu Python'da abstract klasslar orqali amalga oshiriladi (keyingi darslar).
