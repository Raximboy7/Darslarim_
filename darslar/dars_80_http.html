HTTP Nima?

HTTP (HyperText Transfer Protocol) — bu internetda ma'lumot almashish uchun ishlatiladigan protokol (qoidalar to'plami). Brauzeringiz biror saytni ochganda, u HTTP orqali serverga so'rov yuboradi va server unga javob qaytaradi.
Oddiy misol

HTTP — bu xuddi restorandagi suhbat kabi: siz (mijoz) ofitsiantdan ovqat so'raysiz, oshxona (server) tayyorlab sizga olib keladi. So'rash va javob qaytarish — HTTP ham shunday ishlaydi.

Client-Server Modeli

Internet ikki tomondan iborat: client (mijoz) va server. Client so'rov yuboradi, server esa unga javob beradi.
TomonKim?Vazifasi
ClientBrauzer, telefon, dasturSo'rov (request) yuboradi
ServerSaytni saqlab turuvchi kompyuterJavob (response) qaytaradi
Client ──── so'rov ────► Server
Client ◄─── javob ───── Server
# Client so'raydi, server javob beradi

Request va Response

Request (so'rov) — client serverga yuboradigan xabar. Response (javob) — server qaytaradigan natija. Har bir so'rovga bitta javob keladi.
QismRequest ichidaResponse ichida

Status kod—200, 404, 500 ...

Status Kodlar

Har bir javobda status kod bo'ladi — bu raqam so'rov qanday natija bilan tugaganini bildiradi. Birinchi raqamiga qarab guruhlarga bo'linadi.
KodMa'nosiTushuntirish
200OKHammasi muvaffaqiyatli o'tdi
301RedirectSahifa boshqa manzilga ko'chgan
404Not FoundSahifa topilmadi
500Server ErrorServerda xatolik yuz berdi
Guruhlar (birinchi raqam)

• 2xx — Muvaffaqiyat (masalan 200)
• 3xx — Yo'naltirish / redirect (masalan 301)
• 4xx — Client xatosi (masalan 404)
• 5xx — Server xatosi (masalan 500)

HTTP Metodlari

Metod — so'rovning maqsadini bildiradi: ma'lumot olishni, yuborishni, yangilashni yoki o'chirishni. Eng ko'p ishlatiladiganlari quyida.
MetodVazifasiMisol
GETMa'lumot olishSahifani ochish
POSTMa'lumot yuborishForma to'ldirib yuborish
PUTMa'lumotni yangilashProfilni tahrirlash
DELETEMa'lumotni o'chirishRasmni o'chirish
GET /maqolalar # maqolalarni olish
POST /maqolalar # yangi maqola qo'shish
PUT /maqolalar/5 # 5-maqolani yangilash
DELETE /maqolalar/5 # 5-maqolani o'chirish
# Bir manzil, lekin har xil metod — har xil amal
Eslab Qoling

• HTTP — internetda ma'lumot almashish protokoli
• Client so'rov yuboradi, server javob qaytaradi
• Status kod javob natijasini bildiradi: 200 yaxshi, 404 topilmadi
• Metodlar: GET olish, POST yuborish, PUT yangilash, DELETE o'chirish

Misol 1: Status Kod Tekshiruvchi

Status kodni kiriting va uning ma'nosini ko'ring (masalan 200, 404, 500, 301).

Misol 2: Request → Response Oqimi

"So'rov yubor" tugmasini bosing va client'dan server'ga so'rov, so'ng serverdan javob qanday qaytishini bosqichma-bosqich kuzating.

Misol 3: HTTP Metodini Tanlash

Metodni tanlang va u nima qilishini ko'ring.

Sizning Mashqingiz

1
Status kodlarni yodlang

200, 404, 500 va 301 ning ma'nosini ayting

2
Oqimni chizib chiqing

Client → so'rov → server → javob ketma-ketligini yozing

3
Metodlarni moslang

"forma yuborish", "sahifa ochish", "o'chirish" uchun qaysi metod kerakligini toping

4
Demolarni sinab ko'ring

Yuqoridagi 3 ta demoni turli qiymatlar bilan sinab ko'ring
