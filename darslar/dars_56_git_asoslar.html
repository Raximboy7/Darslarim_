Git Nima?

Git — bu sizning kodingizdagi o'zgarishlarni kuzatuvchi tizim. U har bir saqlangan holatni eslab qoladi: kim, qachon va nimani o'zgartirgani. Git yordamida siz kodning tarixini ko'rishingiz, eski holatga qaytishingiz va boshqalar bilan birga ishlashingiz mumkin.

Oddiy qilib aytganda, Git — bu kodingiz uchun vaqt mashinasi. Har bir muhim o'zgarishni "rasmga olib" qo'yadi va kerak bo'lsa istalgan eski rasmga qaytadi.

Version Control nima?

Version Control (versiyalarni boshqarish) — bu fayllaringizning turli holatlarini (versiyalarini) saqlab borish tizimi. Git — eng mashhur version control tizimi.

Repository (Repo) Nima?

Repository (qisqacha repo) — bu Git kuzatib turadigan loyiha papkasi. Repo ichida sizning fayllaringiz va ularning butun tarixi saqlanadi. Tarix yashirin .git papkasida turadi.

git init — Yangi Repo Yaratish

git init buyrug'i joriy papkani Git repozitoriyiga aylantiradi. Bu buyruqdan keyin Git papkadagi o'zgarishlarni kuzata boshlaydi.
$ git init
# Joriy papkada yangi bo'sh repo yaratadi
Initialized empty Git repository in /loyiham/.git/

git config — Foydalanuvchini Sozlash

Git har bir o'zgarishni kim qilganini bilishi kerak. Shuning uchun birinchi marta ismingiz va emailingizni sozlaysiz. --global degani — bu sozlama kompyuteringizdagi barcha repolar uchun amal qiladi.
$ git config --global user.name "Ali Valiyev"
$ git config --global user.email "ali@mail.com"
# Ism va emailni bir marta sozlaymiz
$ git config --global user.name
# Sozlamani tekshirish — qiymatni chiqaradi
Ali Valiyev

Gitning 3 Joyi (Eng Muhim Tushuncha)

Git bilan ishlaganda faylingiz 3 ta joy orqali o'tadi. Buni tushunish Gitni o'rganishning kalitidir:
JoyVazifasiBuyruq
Working Directory
(ishchi katalog)		Siz fayllarni tahrirlaydigan joy—
Staging Area
(tayyorlash maydoni)		Saqlashga tayyorlangan o'zgarishlargit add
Repository
(.git tarixi)		Doimiy saqlangan tarixgit commit
Oqim shunday

Fayl yozasiz (Working) → git add bilan tayyorlaysiz (Staging) → git commit bilan tarixga saqlaysiz (Repository).

Eslab Qoling

• Git — o'zgarishlarni kuzatuvchi tizim (version control)
• Repo — Git kuzatadigan loyiha papkasi
• git init — yangi repo yaratadi
• git config — ism va emailni sozlaydi
• 3 joy: Working → Staging (add) → Repository (commit)

Misol 1: git init Simulyatsiyasi

Tugmani bosing — terminalda git init qanday natija berishini ko'ring. (Bu haqiqiy git emas, brauzerdagi simulyatsiya.)

terminal — loyiham
$ git init buyrug'ini ishga tushiring...

Misol 2: git config Sozlash Demosi

Ismingiz va emailingizni yozing, so'ng "Sozlash"ni bosing — git config buyruqlari ketma-ket qanday ishlashini ko'ring.

terminal — git config
$ Ism va emailni kiriting, so'ng "Sozlash"ni bosing...
