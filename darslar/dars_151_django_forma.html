Forma Nima Uchun Kerak?

Forma — bu foydalanuvchi sizga ma'lumot yuboradigan oyna. Ro'yxatdan o'tish, izoh qoldirish, qidirish, kontakt yuborish — bularning barchasi formalar orqali bo'ladi. Django formalarni o'zi yaratadi, tekshiradi va xavfsiz qabul qiladi.
Django nima ish qiladi?

• HTML maydonlarini (input) avtomatik chizadi
• Yuborilgan ma'lumotni tekshiradi (validatsiya)
• Xatolarni o'zbekcha/inglizcha xabar bilan ko'rsatadi
• Toza ma'lumotni cleaned_data ichiga joylaydi

forms.py — Forma Fayli

Django'da formalar alohida forms.py faylida yoziladi. Forma — bu forms.Form dan meros olgan klass. Har bir maydon bitta o'zgaruvchi bo'ladi.
# forms.py
from django import forms

class KontaktForm(forms.Form):
    ism = forms.CharField(max_length=50)
    email = forms.EmailField()
    xabar = forms.CharField(widget=forms.Textarea)
# ism — matn maydoni, email — email maydoni, xabar — katta matn maydoni

ModelForm — Modeldan Forma

Agar sizda model (jadval) bo'lsa, undan to'g'ridan-to'g'ri forma yasash mumkin. ModelForm maydonlarni model asosida o'zi tuzadi.
from django import forms
from .models import Maqola

class MaqolaForm(forms.ModelForm):
    class Meta:
        model = Maqola
        fields = ["sarlavha", "matn"]
# Maqola modelidan faqat sarlavha va matn maydonlari uchun forma yasaladi

GET va POST — Forma Oqimi

Formaga ikki marta murojaat bo'ladi. Birinchi marta foydalanuvchi sahifani ochadi (GET) — bo'sh forma chiqadi. Ikkinchi marta tugmani bosadi (POST) — ma'lumot serverga ketadi.
GET

Sahifa ochiladi.
Bo'sh forma ko'rsatiladi.

POST

Tugma bosiladi.
Ma'lumot tekshiriladi va saqlanadi.

# views.py
def kontakt(request):
    if request.method == "POST":
        form = KontaktForm(request.POST)
        if form.is_valid():
            ism = form.cleaned_data["ism"]
            # ... ma'lumotni saqlash
    else:
        form = KontaktForm()
    return render(request, "kontakt.html", {"form": form})
# POST bo'lsa — ma'lumotni oladi va tekshiradi. GET bo'lsa — bo'sh forma yasaydi.

is_valid() va cleaned_data

form.is_valid() — formani tekshiradi va True yoki False qaytaradi. Agar hammasi to'g'ri bo'lsa, toza ma'lumot form.cleaned_data lug'atiga joylanadi.
if form.is_valid():
    ism = form.cleaned_data["ism"]
    email = form.cleaned_data["email"]
    print(ism, email)
# cleaned_data — tekshirilgan, ishonchli ma'lumot. To'g'ridan-to'g'ri request.POST emas!

CSRF Token — Xavfsizlik

Muhim qoida

Har bir POST formada {% csrf_token %} bo'lishi SHART. U formani begona saytlardan kelgan soxta yuborishlardan himoya qiladi. Agar uni unutsangiz, Django formani qabul qilmaydi va xato beradi (403 Forbidden).

<!-- kontakt.html -->
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">Yuborish</button>
</form>
# {{ form.as_p }} — barcha maydonlarni avtomatik chizadi. csrf_token — himoya kaliti.

Ma'lumotni Saqlash

Forma to'g'ri bo'lsa, ma'lumotni bazaga saqlaymiz. ModelForm bilan bu juda oson — bitta form.save() yetadi.
if form.is_valid():
    form.save()
    return redirect("rahmat")
# form.save() — ma'lumotni bazaga yozadi. So'ng "rahmat" sahifasiga yo'naltiradi.
Eslab Qoling

• Forma forms.py da, forms.Form dan meros oladi
• GET — bo'sh forma ko'rsatadi, POST — ma'lumotni qabul qiladi
• is_valid() tekshiradi, cleaned_data toza ma'lumotni beradi
• POST formada {% csrf_token %} SHART
• form.save() ModelForm ma'lumotini bazaga saqlaydi

Misol 1: Ishlaydigan Forma

Quyidagi forma — POST yuborilganda ma'lumotni "qabul qildi" deb ko'rsatadi (xuddi Django cleaned_data kabi).

Misol 2: Validatsiya (Tekshirish)

Django is_valid() bilan ma'lumotni tekshiradi. Bu yerda ism bo'sh bo'lmasligi va email @ belgisiga ega bo'lishi tekshiriladi. Xato bo'lsa — qizil xabar chiqadi.

Misol 3: GET va POST Oqimi

Tugmalarni bosib ko'ring. GET — bo'sh forma, POST — ma'lumotni qabul qilish. Django ichida nima bo'lishini ko'rasiz.

Misol 4: ModelForm Maydonlarini Yasash

Maydonlarni tanlang — Django ularni HTML formaga aylantiradi. Bu {{ form.as_p }} qiladigan ishning soddalashtirilgan ko'rinishi.
