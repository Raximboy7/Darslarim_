Flask: Routes — GET/POST

Modul 14 — Web Asoslari | Dars 157

GET va POST — Ikki Xil So'rov

Brauzer serverga so'rov yuborganda uning metodi (turi) bo'ladi. Eng ko'p ishlatiladigani — GET va POST. Oddiy qilib aytganda: GET — ma'lumot olish uchun, POST — ma'lumot yuborish (jo'natish) uchun.
XususiyatGETPOST
VazifasiSahifani ochish, ma'lumot olishForma to'ldirib yuborish

Route uchun Metodni Belgilash

Har bir @app.route() odatda faqat GET ni qabul qiladi. Agar route POST ni ham qabul qilishini xohlasak, methods parametrini beramiz.
@app.route("/login", methods=["GET", "POST"])
def login():
    return "Login sahifasi"
# Bu route endi GET (sahifani ochish) va POST (forma yuborish) so'rovlarini qabul qiladi
Diqqat

Agar methods yozilmasa, route faqat GET ni qabul qiladi. POST so'rov kelsa, Flask 405 Method Not Allowed xatosini qaytaradi.

So'rov Turini Aniqlash — request.method

Bitta funksiya ichida GET va POST ni ajratish kerak. Buning uchun request obyektining method xususiyatini tekshiramiz. Avval request ni import qilamiz.
from flask import Flask, request

app = Flask(__name__)

@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        return "Forma yuborildi!"
    return "Login formasi (GET)"
# POST bo'lsa — forma yuborilgan; aks holda (GET) — sahifani ko'rsatamiz

Forma Ma'lumotini Olish — request.form

Foydalanuvchi formadagi maydonlarni to'ldirib POST qilganda, ularning qiymatini request.form orqali olamiz. U lug'atga (dict) o'xshaydi: kalit — name atributi, qiymat — foydalanuvchi kiritgan matn.
@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        ism = request.form["ism"]
        parol = request.form.get("parol")
        return f"Salom, {ism}!"
    return "Formani to'ldiring"
# request.form["ism"] — to'g'ridan-to'g'ri; request.form.get("parol") — maydon yo'q bo'lsa None qaytaradi (xato bermaydi)
Maslahat

request.form["ism"] — agar maydon bo'lmasa xato beradi. request.form.get("ism") — xavfsizroq, maydon bo'lmasa None qaytaradi.

To'liq Misol — HTML Forma + Flask

Endi HTML forma va uni qabul qiluvchi Flask route ni birlashtiramiz. Formadagi action — qayerga yuborilishi, method — qanday yuborilishi, har bir input ning name — server oladigan kalit.
from flask import Flask, request

app = Flask(__name__)

@app.route("/", methods=["GET", "POST"])
def forma():
    if request.method == "POST":
        ism = request.form.get("ism")
        return f"Xush kelibsiz, {ism}!"
    return '''
    <form method="POST">
      <input name="ism" placeholder="Ismingiz">
      <button>Yuborish</button>
    </form>
    '''

if __name__ == "__main__":
    app.run(debug=True)
# GET — forma ko'rsatiladi; foydalanuvchi ismini kiritib "Yuborish" bosgach POST bo'ladi va salom chiqadi
>>> Sahifa ochildi (GET): forma ko'rindi
>>> "Ali" kiritildi, Yuborish bosildi (POST)
>>> Xush kelibsiz, Ali!
