Multiprocessing (Ko'p Jarayon)

Modul 15 — Concurrency va Advanced | Dars 99

Jarayon (Process) Nima?

Jarayon (process) — bu operatsion tizim ishga tushiradigan mustaqil dasturning bir nusxasi. Har bir jarayonning o'ziga xos alohida xotirasi bo'ladi. Ular bir-birining o'zgaruvchilarini ko'rmaydi — har biri o'z dunyosida ishlaydi. Bu thread'dan eng katta farqi: thread'lar bitta xotirani bo'lishadi, jarayonlar esa yo'q.
Oddiy tasavvur

Tasavvur qiling, bir nechta oshpaz bor. Thread — bu bitta oshxonada (umumiy xotira) ishlayotgan oshpazlar; ular bir xil idishlardan foydalanadi. Process — har biri o'z alohida oshxonasiga ega oshpazlar; hech narsani bo'lishmaydi, shu sababli bir-biriga xalal bermaydi.

multiprocessing Moduli

Python'da jarayonlar bilan ishlash uchun standart multiprocessing moduli mavjud. Uni import qilib, yangi jarayonlar yaratamiz.
import multiprocessing
# Jarayonlar bilan ishlash kutubxonasi

Thread vs Process: Eng Muhim Farq

Python'da GIL (Global Interpreter Lock) degan mexanizm bor. U bir vaqtda faqat bitta thread Python kodini bajarishiga ruxsat beradi. Shuning uchun thread'lar CPU-og'ir (hisob-kitob ko'p) ishlarda haqiqiy tezlik bermaydi. Jarayonlar esa har biri alohida Python interpretatoriga ega bo'lgani uchun GIL ni chetlab o'tadi va bir nechta yadroda haqiqiy parallel ishlay oladi.

Jarayon Yaratish (Process)

Process(target=funksiya) bilan jarayon yaratamiz. .start() uni ishga tushiradi, .join() esa jarayon tugashini kutadi.
import multiprocessing

def ish(raqam):
    print(f"Jarayon {raqam} ishladi")

if __name__ == "__main__":
    p1 = multiprocessing.Process(target=ish, args=(1,))
    p1.start()  # jarayonni boshlash
    p1.join()   # tugashini kutish
# target — bajariladigan funksiya; args — unga beriladigan argumentlar
>>> Jarayon 1 ishladi
Diqqat: __name__ == "__main__"

multiprocessing'da kodni har doim if __name__ == "__main__": blokining ichida yozing. Bu yangi jarayonlar fayl kodini qayta-qayta ishga tushirib yuborishining oldini oladi.

Process Pool (Pool va map)

Ko'p jarayonni qo'lda yaratish o'rniga Pool ishlatamiz. U jarayonlar guruhini (basseyn) yaratadi va ishni ular o'rtasida avtomatik taqsimlaydi. pool.map(funksiya, ro'yxat) ro'yxatdagi har bir elementga funksiyani parallel qo'llaydi.
from multiprocessing import Pool

def kvadrat(x):
    return x * x

if __name__ == "__main__":
    with Pool(4) as p:  # 4 ta jarayon
        natija = p.map(kvadrat, [1, 2, 3, 4])
    print(natija)
# Pool ishni 4 jarayonga bo'lib, har biriga elementlarni beradi
>>> [1, 4, 9, 16]

Queue Orqali Ma'lumot Almashish

Jarayonlarning xotirasi alohida bo'lgani uchun ular oddiy o'zgaruvchi orqali ma'lumot bera olmaydi. Buning uchun Queue (navbat) ishlatamiz. Bir jarayon navbatga .put() bilan qo'yadi, boshqasi .get() bilan oladi.
from multiprocessing import Process, Queue

def ishchi(q):
    q.put("Salom asosiy jarayon!")

if __name__ == "__main__":
    q = Queue()
    p = Process(target=ishchi, args=(q,))
    p.start()
    print(q.get())  # navbatdan o'qish
    p.join()
# .put() — navbatga qo'yish, .get() — navbatdan olish
>>> Salom asosiy jarayon!
Eslab Qoling

• Process — alohida xotirali mustaqil jarayon
• Jarayonlar GIL ni chetlab o'tadi → haqiqiy parallellik
• Process(target=...) + .start() + .join()
• Pool + .map() — ishni avtomatik taqsimlaydi
• Queue — jarayonlar o'rtasida ma'lumot almashish
• CPU-og'ir ishlar → process; I/O kutish ishlari → thread

Misol 1: Thread vs Process Tezligi

CPU-og'ir ishni (katta hisoblash) thread va process bilan bajarishni taqqoslaymiz. Yadrolar sonini tanlang va ishga tushiring.

Misol 2: Pool bilan Ishni Bo'lish

Ro'yxatdagi sonlarni kvadratga ko'taramiz. Pool ishni jarayonlarga bo'lib, har birining hissasini ko'rsatadi.

Misol 3: Queue Orqali Almashish

Ishchi jarayon navbatga xabar qo'yadi, asosiy jarayon uni oladi. Navbatning ishlashini kuzating.
