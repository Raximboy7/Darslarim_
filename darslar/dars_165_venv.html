venv Nima va Nega Kerak?

venv (virtual environment — virtual muhit) — bu bitta loyiha uchun alohida, izolyatsiya qilingan Python muhiti. Unda o'rnatilgan kutubxonalar faqat shu loyihaga tegishli bo'ladi va boshqa loyihalarga aralashmaydi.
Muammo: nega kerak?

Tasavvur qiling: A loyiha Django 3.0 ni, B loyiha esa Django 4.2 ni talab qiladi. Agar hammasini bitta umumiy Python ga o'rnatsangiz — versiyalar to'qnashadi. Har bir loyihaga alohida venv yaratsangiz, ular bir-biriga xalaqit bermaydi.

1. Virtual Muhit Yaratish

Terminal (buyruq qatori) ni loyiha papkasida oching va quyidagi buyruqni yozing. U venv nomli yangi papka yaratadi.
python -m venv venv
# python -m venv — venv modulini ishga tushiradi; oxirgi "venv" — yaratiladigan papka nomi (istalgan nom bo'lishi mumkin)
Eslatma

Ba'zi tizimlarda python o'rniga python3 yozish kerak bo'ladi: python3 -m venv venv.

2. Virtual Muhitni Faollashtirish (activate)

Papka yaratilgani yetarli emas — uni faollashtirish kerak. Buyruq operatsion tizimga qarab farq qiladi.

Windows (CMD / PowerShell)

venv\Scripts\activate
# Windows da Scripts papkasidagi activate ishga tushiriladi

macOS / Linux

source venv/bin/activate
# macOS va Linux da bin papkasidagi activate manba (source) qilinadi
# Faollashgach terminal boshida (venv) paydo bo'ladi:
(venv) user@komputer:~/loyiham$
Muhim belgi

Terminal satri boshida (venv) yozuvi paydo bo'lsa — muhit faol. Endi o'rnatgan har bir kutubxona faqat shu muhitga tushadi.

3. Ishlatish va O'chirish (deactivate)

Muhit faol paytda pip install bilan kutubxona o'rnatsangiz, u global Python ga emas, balki shu venv ga o'rnatiladi. Ishni tugatgach deactivate bilan chiqasiz.
(venv) $ pip install flask
(venv) $ pip list
(venv) $ deactivate
# flask shu venv ichiga o'rnatildi; pip list — o'rnatilganlarni ko'rsatadi; deactivate — muhitdan chiqadi

Maslahat

venv papkasini Git ga qo'shmang. Uni .gitignore ga yozing — chunki har kim o'zida qaytadan yaratib oladi (keyingi darsda requirements.txt yordamida).

Xulosa

Eslab qoling

• venv — har loyihaga alohida, toza Python muhiti
• python -m venv venv — muhit yaratadi
• source venv/bin/activate (yoki Windows da venv\Scripts\activate) — faollashtiradi
• Terminal boshidagi (venv) — muhit faolligini bildiradi
• deactivate — muhitdan chiqadi
