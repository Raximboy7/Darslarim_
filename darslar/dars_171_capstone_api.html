Capstone 1 — REST API

Modul 23 — Pro daraja va Capstone | Dars 171

Bu Loyihada Nima Quramiz?

Bu birinchi capstone (yakuniy katta) loyiha. Biz haqiqiy REST API quramiz — bu boshqa dasturlar (mobil ilova, sayt, boshqa server) ma'lumot olish va yuborish uchun murojaat qiladigan dastur. Misol sifatida vazifalar (to-do) API sini yasaymiz: vazifa qo'shish, ko'rish, yangilash va o'chirish.
API nima?

API — bu ikki dastur o'rtasidagi kelishuv. Mijoz (client) so'rov yuboradi, server javob qaytaradi. REST esa API yasashning eng mashhur uslubi: har bir narsa manzil (URL) bilan ifodalanadi va HTTP metodlari orqali boshqariladi.

REST Tamoyillari

REST API'da har bir resurs (masalan, vazifa) o'z manziliga ega. Resurs ustida amal bajarish uchun mos HTTP metodi ishlatiladi.
MetodManzilVazifasi
GET/tasksBarcha vazifalarni olish
GET/tasks/1Bitta vazifani olish
POST/tasksYangi vazifa qo'shish
PUT/tasks/1Vazifani yangilash
DELETE/tasks/1Vazifani o'chirish
CRUD nima?

Bu 4 ta asosiy amal: Create (yaratish — POST), Read (o'qish — GET), Update (yangilash — PUT), Delete (o'chirish — DELETE). Deyarli har bir API shu 4 amaldan iborat.

JSON Javob

API ma'lumotni JSON ko'rinishida uzatadi. JSON Python lug'atiga (dictionary) juda o'xshaydi: kalit va qiymatlardan iborat. Server JSON yuboradi, mijoz uni o'qiydi.
{
  "id": 1,
  "title": "Python o'rganish",
  "done": false
}
# Bitta vazifaning JSON ko'rinishi

1-qadam: Flask O'rnatish va Boshlash

Biz Flask kutubxonasidan foydalanamiz — bu Python'da API yasashning eng oddiy yo'li. Avval terminalda o'rnatamiz, keyin eng kichik dasturni yozamiz.
pip install flask
# Terminalda Flask kutubxonasini o'rnatish
from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route("/")
def home():
    return "Vazifalar API ishlamoqda!"

if __name__ == "__main__":
    app.run(debug=True)
# Flask ilovasini yaratamiz, "/" manziliga oddiy javob qaytaramiz va serverni ishga tushiramiz
>>> Running on http://127.0.0.1:5000

2-qadam: Ma'lumotlar Bazasi (oddiy ro'yxat)

Boshlash uchun ma'lumotni oddiy Python ro'yxati (list) ichida saqlaymiz. Har bir vazifa — lug'at. Keyingi darsda buni haqiqiy ma'lumotlar bazasiga ulaymiz.
tasks = [
    {"id": 1, "title": "Non olish", "done": False},
    {"id": 2, "title": "Kod yozish", "done": True},
]
next_id = 3
# tasks — vazifalar ro'yxati; next_id — yangi vazifaga beriladigan keyingi raqam

3-qadam: GET — Vazifalarni O'qish

GET so'rovi ma'lumot o'qish uchun. Barcha vazifalarni ham, bitta vazifani ham olishimiz mumkin.
@app.route("/tasks", methods=["GET"])
def get_tasks():
    return jsonify(tasks)

@app.route("/tasks/<int:task_id>", methods=["GET"])
def get_task(task_id):
    for t in tasks:
        if t["id"] == task_id:
            return jsonify(t)
    return jsonify({"error": "Topilmadi"}), 404
# /tasks butun ro'yxatni qaytaradi; /tasks/<id> bitta vazifani izlaydi, topilmasa 404 status kodi

4-qadam: POST — Yangi Vazifa Qo'shish

POST so'rovi yangi narsa yaratadi. Mijoz JSON yuboradi, biz uni o'qib ro'yxatga qo'shamiz va 201 Created status bilan javob qaytaramiz.
@app.route("/tasks", methods=["POST"])
def add_task():
    global next_id
    data = request.get_json()
    if not data or "title" not in data:
        return jsonify({"error": "title kerak"}), 400
    task = {"id": next_id, "title": data["title"], "done": False}
    tasks.append(task)
    next_id += 1
    return jsonify(task), 201
# JSON ni o'qiymiz, "title" bo'lmasa 400 (xato so'rov), aks holda yangi vazifa qo'shib 201 qaytaramiz

5-qadam: PUT va DELETE

PUT mavjud vazifani yangilaydi, DELETE esa o'chiradi.
@app.route("/tasks/<int:task_id>", methods=["PUT"])
def update_task(task_id):
    data = request.get_json()
    for t in tasks:
        if t["id"] == task_id:
            t["title"] = data.get("title", t["title"])
            t["done"] = data.get("done", t["done"])
            return jsonify(t)
    return jsonify({"error": "Topilmadi"}), 404

@app.route("/tasks/<int:task_id>", methods=["DELETE"])
def delete_task(task_id):
    global tasks
    tasks = [t for t in tasks if t["id"] != task_id]
    return jsonify({"ok": True})
# PUT: vazifani topib maydonlarni yangilaydi; DELETE: berilgan id'dan boshqasini qoldirib, vazifani o'chiradi

Status Kodlar

Server har bir javobda status kod qaytaradi — bu so'rov qanday tugaganini bildiradi.
KodMa'nosiQachon
200OK — muvaffaqiyatGET, PUT, DELETE bajarildi
201Created — yaratildiPOST yangi narsa qo'shdi
400Bad Request — xato so'rovNoto'g'ri ma'lumot yuborildi
404Not Found — topilmadiResurs mavjud emas
500Server Error — server xatosiDasturda xatolik

Test Qilish

API'ni test qilish uchun Python'ning requests kutubxonasidan foydalanish mumkin. Quyida API ishlayotganini tekshiramiz.
import requests

# Yangi vazifa qo'shamiz (POST)
r = requests.post("http://127.0.0.1:5000/tasks",
                 json={"title": "Test vazifa"})
print(r.status_code)  # 201
print(r.json())     # {'id': 3, 'title': 'Test vazifa', 'done': False}
# requests bilan serverga POST yuboramiz va javob status kodini hamda JSON'ni tekshiramiz
Eslab Qoling

• REST: har bir resurs URL bilan, amal HTTP metodi bilan
• CRUD = POST (yaratish), GET (o'qish), PUT (yangilash), DELETE (o'chirish)
• JSON — ma'lumot uzatish formati, Python lug'atiga o'xshaydi
• Status kod javob natijasini bildiradi (200, 201, 400, 404)
• jsonify() Python lug'atini JSON javobga aylantiradi

Misol 2: JSON So'rov va Javob

Quyida haqiqiy CRUD ishlaydi. Vazifa qo'shing (Create), ro'yxatni ko'ring (Read), bajarildi deb belgilang (Update) yoki o'chiring (Delete).
