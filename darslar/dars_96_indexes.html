Index Nima?

Index — bu jadvaldagi ma'lumotni tez topish uchun yordamchi tuzilma. U xuddi kitobning mundarijasiga o'xshaydi: butun kitobni varaqlamasdan, mundarijaga qarab kerakli sahifaga to'g'ridan-to'g'ri o'tasiz.
Kitob mundarijasi analogiyasi

Tasavvur qiling, 500 sahifali kitobdan "Index" so'zini qidiryapsiz. Mundarija bo'lmasa — har bir sahifani birma-bir o'qib chiqishingiz kerak. Mundarija bo'lsa — alifbo tartibida topib, sahifa raqamiga sakraysiz. Database'da index aynan shu mundarija vazifasini bajaradi.

Nega Index Kerak?

Jadval kichik bo'lsa (10-20 qator), database hammasini tez o'qiydi. Lekin jadval million qatorli bo'lsa, har bir qatorni tekshirish juda sekin bo'ladi. Index katta jadvalda qidiruvni yuzlab marta tezlashtiradi.

Index'siz: butun jadvalni tekshirish (Full Scan)

SELECT * FROM users
WHERE email = 'ali@mail.com';
# Index bo'lmasa: 1 000 000 qatorning HAMMASINI tekshiradi

Index bilan: to'g'ridan-to'g'ri sakrash

-- Avval email ustuniga index yaratamiz
CREATE INDEX idx_email ON users(email);
# Endi xuddi mundarijadan topgandek tez topadi

CREATE INDEX — Index Yaratish

Index yaratish sintaksisi juda sodda. Tez-tez qidiriladigan ustunlarga index qo'yiladi.
-- Oddiy index (bitta ustun)
CREATE INDEX idx_name ON users(name);

-- Takrorlanmas (unique) index
CREATE UNIQUE INDEX idx_email ON users(email);

-- Bir nechta ustun bo'yicha index
CREATE INDEX idx_city_age ON users(city, age);

-- Index'ni o'chirish
DROP INDEX idx_name;
# idx_ — index nomi uchun keng tarqalgan boshlanish. UNIQUE — qiymatlar takrorlanmasligini ham ta'minlaydi.

EXPLAIN QUERY PLAN — Tekshirish

Database so'rovni qanday bajarayotganini ko'rish uchun EXPLAIN QUERY PLAN ishlatiladi. U index ishlatilayotgani yoki butun jadval skanerlanayotganini ko'rsatadi.

Index'siz natija

EXPLAIN QUERY PLAN
SELECT * FROM users WHERE email = 'ali@mail.com';
# So'rov rejasini ko'rsatadi (qanday bajariladi)
>>> SCAN users

SCAN — yomon belgi: butun jadval o'qilmoqda.

Index'dan keyingi natija

>>> SEARCH users USING INDEX idx_email (email=?)

SEARCH ... USING INDEX — yaxshi belgi: index ishlatilmoqda, tez bajariladi.

Index Kamchiliklari

Index hammasini tezlashtirmaydi

• Qo'shimcha joy: har bir index disk xotirasini egallaydi.
• Yozish sekinroq: INSERT, UPDATE, DELETE da index ham yangilanadi, shuning uchun yozish biroz sekinlashadi.
• Keraksiz index: hech qachon qidirilmaydigan ustunga index qo'yish faqat zarar.

Xulosa: index o'qishni (SELECT) tezlashtiradi, lekin yozishni biroz sekinlashtiradi. Shuning uchun faqat kerakli ustunlarga qo'ying.
Eslab Qoling

• Index — bu kitob mundarijasi: tez qidiruv uchun
• CREATE INDEX idx ON jadval(ustun) — yaratish
• EXPLAIN QUERY PLAN — index ishlayotganini tekshirish
• SCAN — yomon, SEARCH USING INDEX — yaxshi
• Index o'qishni tezlashtiradi, yozishni sekinlashtiradi
