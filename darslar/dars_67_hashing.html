Hashing va Dictionary Ichki Tuzilishi

Modul 11 — Algoritmlar | Dars 67

Hash Funksiyasi Nima?

Hash funksiyasi — bu istalgan ma'lumotni (matn, son) qabul qilib, uni bitta raqamga aylantiruvchi maxsus funksiya. Bu raqam hash qiymati deb ataladi. Bir xil kalit har doim bir xil raqamni beradi — bu juda muhim.
print(hash("olma"))
print(hash("olma"))
# Bir xil kalit → har doim bir xil raqam
>>> 7240398127412341011
>>> 7240398127412341011
Oddiy tilda

Tasavvur qiling, har bir so'zga maxsus "shtrix-kod" beriladi. "olma" so'zi har doim bir xil shtrix-kodga ega bo'ladi. Hash funksiyasi ana shu shtrix-kodni hisoblaydi.

Hash Table (Hash Jadval)

Hash table — bu ma'lumotni saqlash uchun ishlatiladigan tuzilma. U bir nechta kataklar (ingliz tilida "bucket")dan iborat. Ishlash tartibi: kalit → hash raqami → katak indeksi → qiymat.
kalit = "olma"
h = hash("olma")   
indeks = h % 8
# Katta hash raqamini 8 katakka sig'dirish uchun % (qoldiq) ishlatiladi
>>> hash("olma") = 7240398127412341011
>>> 7240398127412341011 % 8 = 3
>>> "olma" → 3-katakka tushadi
Demak qiymat 3-katakka joylanadi. Keyin "olma"ni qidirsak, yana o'sha hisob qaytariladi va to'g'ridan-to'g'ri 3-katakka boramiz — boshqa kataklarni tekshirishga hojat yo'q.

Collision (To'qnashuv)

Ba'zan ikki xil kalit bitta katakka tushib qoladi. Masalan "olma" → 3-katak va "anor" → 3-katak. Bu holat collision (to'qnashuv) deb ataladi. Bu xato emas — tabiiy holat, chunki kataklar soni cheklangan.

Qanday hal qilinadi? — Chaining (Zanjir)

Eng oddiy yechim — chaining (zanjirlash). Bitta katakka bir nechta qiymat tushsa, ular o'sha katak ichida ro'yxat (zanjir) qilib saqlanadi. Qidirishda esa shu kalta ro'yxat bo'ylab kerakli kalit topiladi.
# 3-katak ichida zanjir:
katak[3] = [("olma", 5), ("anor", 9)]
# Ikkalasi ham 3-katakda, lekin alohida juftlik bo'lib saqlanadi
Muhim

Yaxshi hash funksiyasi qiymatlarni kataklarga teng taqsimlaydi, shuning uchun to'qnashuvlar kam bo'ladi va zanjirlar qisqa qoladi.

Nega dict va set Tez Ishlaydi?

Python'dagi dict va set ichida aynan hash table ishlaydi. Kalitni qidirganda Python butun ma'lumotni boshidan oxirigacha tekshirmaydi — u kalitning hashini hisoblaydi va to'g'ri katakka sakraydi. Shu sababli qidiruv tezligi O(1) — ya'ni deyarli bir qadamda.
Amallist (ro'yxat)dict / set
x in data qidiruvO(n) — hammasini tekshiradiO(1) — to'g'ri katakka boradi
Qiymat qo'shishO(1) oxirigaO(1) hash orqali
Kalit bo'yicha olishyo'q (faqat indeks)O(1) hash orqali
narxlar = {"olma": 5, "anor": 9}
print(narxlar["olma"])
# "olma" hashlanadi → katak topiladi → 5 darhol qaytadi
>>> 5
Eslab Qoling

• hash() — kalitni raqamga aylantiradi
• Hash table = kataklar + zanjirlar
• Bir xil katak = collision (to'qnashuv)
• To'qnashuv chaining (zanjir) bilan hal qilinadi
• dict va set tez (O(1)) — hash tufayli

Misol 1: Kalitdan Katak (Hash Demosi)

Kalit kiriting — uning hash qiymati hisoblanadi va qaysi katakka tushishi ko'rsatiladi (8 ta katak, indeks = hash % 8).

Misol 2: To'qnashuvni Ko'rish (Collision)

Ketma-ket kalitlar qo'shing. Agar yangi kalit allaqachon band katakka tushsa — bu to'qnashuv bo'ladi va zanjirga qo'shiladi (qizil rang bilan belgilanadi).

Misol 3: Qo'shish va Qidirish (Hash Table)

Bu kichik dict imitatsiyasi. Kalit va qiymat qo'shing, keyin kalit bo'yicha qidiring — Python qanday qilib to'g'ri katakka sakrashini ko'rasiz.
