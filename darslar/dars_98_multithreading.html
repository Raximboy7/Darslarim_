Multithreading (Ko'p Oqim)

Modul 15 — Concurrency va Advanced | Dars 98

Thread (Oqim) Nima?

Odatda dastur kodni ketma-ket bajaradi: bir ish tugamaguncha, keyingisi boshlanmaydi. Thread (oqim) esa dasturga bir vaqtning o'zida bir nechta ishni bajarish imkonini beradi.
Buni oshxonaga o'xshatamiz: bitta oshpaz hamma taomni navbatma-navbat pishirsa — bu ketma-ket. Agar bir nechta oshpaz birga ishlasa — bu ko'p oqim (multithreading). Thread ayniqsa kutish ko'p bo'lgan ishlarda (fayl o'qish, internetdan ma'lumot olish) juda foydali.

threading Moduli

Pythonda oqimlar bilan ishlash uchun standart threading moduli mavjud. Uni import qilamiz:
import threading
import time
# threading — oqimlar bilan ishlash uchun

Thread Yaratish

Thread yaratish uchun threading.Thread(target=funksiya) ishlatamiz. Bunda target — yangi oqimda bajariladigan funksiya.
import threading

def salomla():
    print("Salom oqimdan!")

oqim = threading.Thread(target=salomla)
oqim.start()   # oqimni ishga tushiradi
oqim.join()    # oqim tugashini kutadi
# .start() — oqimni ishga tushiradi
# .join() — oqim tugaguncha kutadi
>>> Salom oqimdan!
Eng muhim 3 metod

• Thread(target=...) — oqim ob'ektini yaratadi
• .start() — oqimni ishga tushiradi (funksiyani parallel boshlaydi)
• .join() — asosiy dastur shu oqim tugashini kutadi

Bir Nechta Thread va Argumentlar

Funksiyaga argument berish uchun args=(...) ishlatamiz. Bir nechta oqimni sikl orqali yaratish mumkin:
import threading

def ishchi(raqam):
    print(f"Ishchi {raqam} ishladi")

oqimlar = []
for i in range(3):
    t = threading.Thread(target=ishchi, args=(i,))
    t.start()
    oqimlar.append(t)

for t in oqimlar:
    t.join()   # hamma oqim tugashini kutamiz
# args=(i,) — funksiyaga argument beradi (vergul shart!)
# hamma oqimni ro'yxatga yig'ib, keyin .join() qilamiz
>>> Ishchi 0 ishladi
>>> Ishchi 1 ishladi
>>> Ishchi 2 ishladi

Thread Sinxronizatsiyasi: Race Condition

Bir nechta oqim bitta umumiy resursni (masalan, bitta o'zgaruvchini) bir vaqtda o'zgartirsa, muammo yuzaga keladi. Bu race condition (poyga holati) deyiladi — natija har safar boshqacha va noto'g'ri chiqishi mumkin.
hisob = 0

def qoshish():
    global hisob
    for _ in range(100000):
        hisob += 1   # xavfli! umumiy resurs
# Ikki oqim birga hisob += 1 qilsa, ba'zi qo'shishlar yo'qoladi
Muammo

hisob += 1 aslida 3 qadam: o'qish → qo'shish → yozish. Ikki oqim bir vaqtda o'qisa, ikkalasi ham eski qiymatni ko'radi va bir qo'shish yo'qoladi.

Lock va Mutex bilan Himoyalash

Yechim — Lock (qulf, ya'ni mutex). Lock umumiy resursni bir vaqtda faqat bitta oqim o'zgartirishini kafolatlaydi. Boshqa oqimlar qulf ochilguncha kutib turadi.
import threading

hisob = 0
qulf = threading.Lock()   # qulf yaratamiz

def qoshish():
    global hisob
    for _ in range(100000):
        with qulf:   # qulflaymiz
            hisob += 1   # endi xavfsiz
# with qulf: — bloki boshida qulflaydi, oxirida avtomatik ochadi
# endi natija har doim to'g'ri: 200000
Lock metodlari

• qulf = threading.Lock() — qulf yaratadi
• with qulf: — eng oson va xavfsiz usul
• qulf.acquire() / qulf.release() — qo'lda qulflash/ochish

GIL — Qisqacha Eslatma

GIL (Global Interpreter Lock) nima?

Pythonda GIL degan mexanizm bor: u bir vaqtda faqat bitta oqim Python kodini bajarishiga ruxsat beradi. Shuning uchun og'ir hisob-kitoblarda (matematika, sikllar) thread tezlikni oshirmaydi. Lekin kutish bor ishlarda (internet, fayl, kutish vaqti) thread juda foydali — chunki bitta oqim kutayotganda boshqasi ishlay oladi. Og'ir hisob-kitob uchun esa keyingi darsdagi multiprocessing kerak.

Eslab Qoling

• Thread — bir vaqtda bir nechta ishni bajarish
• Thread(target=f) → .start() → .join()
• Umumiy resursni Lock bilan himoyalang (race condition)
• Thread kutish bor ishlarga zo'r; og'ir hisobga — multiprocessing

Misol 1: Ketma-ket vs Parallel Vaqt

Har bir ish 1 soniya kutadi. Ketma-ket bajarsak vaqtlar qo'shiladi. Thread bilan parallel bajarsak — birga ketadi va ancha tez tugaydi.

Misol 2: Lock bilan/holda Race Condition

Ikki oqim bitta hisobchini oshiradi. Lock bo'lmasa, ba'zi qo'shishlar yo'qoladi va natija noto'g'ri chiqadi. Lock bilan natija har doim to'g'ri.

Misol 3: Oqimlar Vizualizatori

Bir nechta oqim bir vaqtda ishlaganini ko'ring. Har bir oqim o'z chizig'ida parallel to'ladi — bu thread bir vaqtda ishlashini ko'rsatadi.
