UPDATE va DELETE

Modul 14 — Database va SQL | Dars 93

Bu darsda nima o'rganamiz?

O'tgan darsda SELECT bilan ma'lumotni o'qishni o'rgandik. Endi ma'lumotni o'zgartirish va o'chirishni ko'ramiz:

• UPDATE — mavjud qatorni o'zgartiradi (yangilaydi)
• DELETE — qatorni jadvaldan o'chiradi
• WHERE — qaysi qatorga ta'sir qilishini belgilaydi (JUDA MUHIM!)

UPDATE — Ma'lumotni o'zgartirish

UPDATE jadvaldagi mavjud qatorlarning qiymatini o'zgartiradi. SET qaysi ustunga qanday yangi qiymat yozilishini ko'rsatadi, WHERE esa qaysi qatorlar o'zgarishini belgilaydi.
UPDATE talabalar
SET yosh = 21
WHERE id = 3;
# id = 3 bo'lgan talabaning yoshini 21 ga o'zgartiradi
>>> 1 qator o'zgardi (1 row updated)

Bir nechta ustunni birga o'zgartirish

UPDATE talabalar
SET yosh = 22, shahar = 'Toshkent'
WHERE ism = 'Ali';
# Ali ning yoshi va shahri birga o'zgaradi (vergul bilan ajratiladi)

DELETE — Ma'lumotni o'chirish

DELETE jadvaldan qator(lar)ni butunlay o'chiradi. Qaysi qator o'chishini yana WHERE belgilaydi.
DELETE FROM talabalar
WHERE id = 5;
# id = 5 bo'lgan talabani o'chiradi
>>> 1 qator o'chirildi (1 row deleted)
Esda tuting

DELETE faqat qatorlarni o'chiradi — jadvalning o'zi va ustunlar joyida qoladi. Bo'sh jadval ham jadval bo'lib qaladi.

⚠️ WHERE — Eng muhim shart!

DIQQAT: WHERE ni unutmang!

Agar WHERE yozmasangiz, UPDATE va DELETE jadvaldagi BARCHA qatorlarga ta'sir qiladi!

• UPDATE talabalar SET yosh = 20; → hammaning yoshini 20 qiladi
• DELETE FROM talabalar; → BUTUN jadvalni bo'shatadi (hamma qator yo'qoladi!)

Bu xatoni qaytarib bo'lmaydi. Shuning uchun WHERE ni doim tekshiring.

# XAVFLI — WHERE yo'q, hamma o'chadi:
DELETE FROM talabalar;

# XAVFSIZ — faqat bitta qator o'chadi:
DELETE FROM talabalar WHERE id = 5;
# Har doim WHERE bilan qaysi qatorni nishonga olganingizni aniqlang

Xavfsiz ishlash qoidalari

QoidaNima uchun
WHERE ni doim yozingFaqat kerakli qatorga ta'sir qiladi
Avval SELECT bilan tekshiringQaysi qatorlar tegishini oldindan ko'rasiz
Asosiy kalit (id) bo'yicha nishonlangid noyob — faqat bitta qator tegadi

Maslahat: avval SELECT, keyin o'zgartiring

O'chirishdan oldin xuddi shu WHERE bilan SELECT qilib ko'ring:
SELECT * FROM talabalar WHERE id = 5;
Agar to'g'ri qator chiqsa, keyin DELETE qilasiz.

COMMIT — O'zgarishni saqlash

Python (masalan sqlite3) bilan ishlaganda, UPDATE va DELETE darhol saqlanmaydi. Ularni doimiy qilish uchun commit() chaqirish kerak.
cursor.execute("UPDATE talabalar SET yosh = 21 WHERE id = 3")
connection.commit()
# commit() o'zgarishni bazaga doimiy saqlaydi
commit() bo'lmasa?

Agar commit() qilmasangiz, dastur tugagach o'zgarishlar yo'qoladi. Xato qilsangiz esa rollback() bilan ortga qaytarish mumkin.

Eslab Qoling

• UPDATE ... SET ... WHERE — qatorni o'zgartiradi
• DELETE FROM ... WHERE — qatorni o'chiradi
• WHERE siz UPDATE/DELETE — BUTUN jadvalga ta'sir qiladi (xavfli!)
• O'chirishdan oldin SELECT bilan tekshiring
• O'zgarishni saqlash uchun commit()

Misol 1: UPDATE — Qator o'zgaradi

Pastdagi soxta jadvalda talabani tanlang va yangi yoshni kiriting. UPDATE bosilganda o'sha qator yashil rangda yangilanadi.

Misol 2: DELETE — Qator o'chadi

Talabani tanlang va DELETE bosing. O'sha qator qizil rangda chiziladi va jadvaldan o'chiriladi.

Misol 3: WHERE bormi yoki yo'qmi?

Bu yerda WHERE ning ahamiyatini ko'rasiz. "WHERE bilan" faqat bitta qatorga, "WHERE siz" esa BUTUN jadvalga ta'sir qiladi — qizil ogohlantirishga e'tibor bering!
