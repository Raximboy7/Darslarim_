Modul 13 — Web Asoslari | Dars 82

requests Nima?

requests — Python da internet bilan ishlash uchun eng mashhur kutubxona. U urllib dan ancha qulayroq va soddaroq. Bir qatorda veb-sahifa yoki API dan ma'lumot olishingiz mumkin. Devlopchilar shu sababli aynan requests ni tanlashadi.
urllib vs requests

urllib bilan so'rov yuborish ko'p qatorli va murakkab.
requests bilan esa bir qatorda — requests.get(url) — tayyor.

O'rnatish

requests standart kutubxona emas, shuning uchun uni avval o'rnatish kerak. Terminal (buyruq qatori) da quyidagini yozing:
pip install requests
# Kutubxonani internetdan yuklab o'rnatadi
O'rnatgandan keyin uni dasturda chaqiramiz:
import requests
# Endi requests barcha funksiyalari tayyor

requests.get() — Ma'lumot Olish

requests.get(url) — serverdan ma'lumot so'rab oladi. Bu eng ko'p ishlatiladigan so'rov turi. Natija response (javob) obyekti bo'ladi.
import requests

javob = requests.get("https://api.misol.uz/users")
print(javob.status_code)
print(javob.text)
# get() serverga so'rov yuboradi va javob obyektini qaytaradi
>>> 200
>>> {"users": ["Ali", "Vali"]}

requests.post() — Ma'lumot Yuborish

requests.post(url, data=...) — serverga yangi ma'lumot yuboradi (masalan, ro'yxatdan o'tish, forma to'ldirish). data yoki json orqali ma'lumot beriladi.
import requests

malumot = {"ism": "Ali", "yosh": 20}
javob = requests.post("https://api.misol.uz/add", json=malumot)
print(javob.status_code)
# post() serverga yangi ma'lumot jo'natadi
>>> 201
get vs post

• get — ma'lumot olish uchun (o'qish)
• post — ma'lumot yuborish uchun (yozish)

Response (Javob) Obyekti

Har bir so'rov response obyektini qaytaradi. Uning eng muhim qismlari quyidagicha:
XususiyatVazifasiMisol
.status_codeSo'rov holati (raqam)200, 404, 500
.textJavobni matn ko'rinishida'{"ism": "Ali"}'
.json()Javobni dict ga aylantiradi{'ism': 'Ali'}
.headersJavob sarlavhalari{'Content-Type': ...}
.urlSo'ralgan to'liq manzil'https://...'
Status kodlari

• 200 — OK, hammasi yaxshi
• 201 — Yaratildi (post muvaffaqiyatli)
• 404 — Topilmadi
• 500 — Server xatosi

.json() — JSON ni Dict ga Aylantirish

Ko'p API lar javobni JSON ko'rinishida qaytaradi. .json() metodi uni to'g'ridan-to'g'ri Python lug'at (dict) ga aylantiradi — ishlatish juda oson bo'ladi.
javob = requests.get("https://api.misol.uz/user/1")
malumot = javob.json() # dict
print(malumot["ism"])
print(malumot["yosh"])
# .json() JSON matnini Python dict ga aylantiradi
>>> Ali
>>> 20

Headers va Cookies

Headers (sarlavhalar) — so'rov bilan birga qo'shimcha ma'lumot jo'natish uchun ishlatiladi (masalan, kalit yoki til). Cookies esa serverdan kelgan kichik ma'lumotlar.
sarlavha = {"Authorization": "Bearer KALIT123"}
javob = requests.get(url, headers=sarlavha)

print(javob.cookies) # server cookie lari
# headers orqali maxsus ma'lumot yuboramiz

params — So'rovga Parametr Qo'shish

params — URL ga qidiruv parametrlarini qo'shadi. requests ularni avtomatik ?kalit=qiymat ko'rinishiga aylantiradi.
sorov = {"q": "python", "sahifa": 2}
javob = requests.get("https://api.misol.uz/search", params=sorov)
print(javob.url)
# params dict ni URL parametrlariga aylantiradi
>>> https://api.misol.uz/search?q=python&sahifa=2
Eslab Qoling

• pip install requests bilan o'rnating
• get() — olish, post() — yuborish
• .status_code 200 bo'lsa hammasi yaxshi
• .json() javobni dict ga aylantiradi
• params va headers qo'shimcha ma'lumot yuboradi

Misol 1: GET So'rov Simulyatsiyasi

URL manzilni tanlang va "So'rov yubor" tugmasini bosing. Demo serverdan keladigan status kod va JSON javobni simulyatsiya qiladi.

Misol 2: POST So'rov — Ma'lumot Yuborish

Ism va yoshni kiriting, so'ngra serverga post() bilan yuboring. Demo serverning javob holatini ko'rsatadi.

Misol 3: .json() — Dict ga Aylantirish

Quyidagi JSON matnni yozing va .json() bilan Python dict ga aylantiring. Keyin alohida kalitlarga murojaat qilamiz.

Misol 4: params bilan URL Yasash

Qidiruv so'zi va sahifa raqamini kiriting. requests ularni qanday URL ga aylantirishini ko'rasiz.
