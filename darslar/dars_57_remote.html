Remote Repository Nima?

Hozirgacha siz Git'ni faqat o'z kompyuteringizda ishlatdingiz. Bu local repository deyiladi — kod faqat sizning kompyuteringizda saqlanadi.

Remote repository — bu loyihangizning internetdagi (bulutdagi) nusxasi. U boshqa serverda saqlanadi va istalgan joydan kira olasiz.
Nega remote kerak?

• Kompyuteringiz buzilsa ham kod yo'qolmaydi (zaxira nusxa)
• Boshqa kompyuterdan ishlay olasiz (uydan, ishdan)
• Boshqalar bilan birga ishlash imkoni (jamoa)
• Kodni dunyoga ulashish (portfolio)

GitHub Nima?

GitHub — bu kodni saqlash va ulashish uchun eng mashhur platforma. U remote repository'larni bulutda saqlaydi. Boshqa platformalar ham bor: GitLab, Bitbucket. Lekin GitHub eng ko'p ishlatiladi.
TushunchaBu nima
GitVersiyalarni boshqaruvchi dastur (kompyuterda)
GitHubKodni bulutda saqlovchi veb-platforma (sayt)
RepositoryLoyiha papkasi (kod + tarix)
originRemote repository'ning standart nomi
Diqqat: Git va GitHub bir narsa emas!

Git — kompyuteringizdagi dastur.
GitHub — internetdagi sayt (github.com). Git'siz ham mavjud.

Local va Remote Farqi

Local — sizning kompyuteringizdagi nusxa. Remote — bulutdagi (GitHub'dagi) nusxa.
Bu ikkalasi buyruqlar orqali bir-biri bilan sinxronlanadi.
BuyruqYo'nalishVazifasi
git cloneRemote → LocalRepo nusxasini olib kelish (birinchi marta)
git remote add—Local'ni remote'ga ulash
git pushLocal → RemoteO'zgarishlarni bulutga yuborish
git pullRemote → LocalBulutdagi o'zgarishlarni olib kelish

git clone — Nusxa Olish

GitHub'dagi tayyor repository'ni o'z kompyuteringizga nusxalash uchun git clone ishlatiladi. Bu repo manzilini (URL) nusxalab oladi.
git clone https://github.com/foydalanuvchi/loyiha.git
# GitHub'dagi repo'ni kompyuterga ko'chiradi
Cloning into 'loyiha'...
remote: Counting objects: 100% done.
Receiving objects: 100% done.

git remote add origin — Ulash

Agar repo'ni o'zingiz kompyuterda yaratgan bo'lsangiz (git init), uni GitHub'ga ulash kerak. Buni git remote add origin qiladi. origin — remote'ning odatdagi nomi.
git remote add origin https://github.com/foydalanuvchi/loyiha.git
git remote -v
# origin nomi bilan remote ulanadi, -v ulanishni tekshiradi
origin  https://github.com/foydalanuvchi/loyiha.git (fetch)
origin  https://github.com/foydalanuvchi/loyiha.git (push)

git push — Bulutga Yuborish

Local'da commit qilgan o'zgarishlaringizni GitHub'ga yuborish uchun git push ishlatiladi. Birinchi marta -u origin main qo'shiladi.
git add .
git commit -m "Yangi sahifa qo'shildi"
git push -u origin main
# O'zgarishlar saqlanadi va bulutga yuboriladi
Enumerating objects: done.
To https://github.com/foydalanuvchi/loyiha.git
   a1b2c3d..e4f5g6h  main -> main

git pull — Bulutdan Olish

Boshqa joyda (yoki jamoadoshingiz) qilgan o'zgarishlarni o'z kompyuteringizga olib kelish uchun git pull ishlatiladi. Bu remote'dagi yangiliklarni local'ga qo'shadi.
git pull origin main
# Bulutdagi yangi o'zgarishlarni kompyuterga oladi
From https://github.com/foydalanuvchi/loyiha
   e4f5g6h..i7j8k9l  main -> origin/main
Updating files: 100% done.
Eslab Qoling

• clone — birinchi marta nusxa olish
• push — yuborish (Local → Remote)
• pull — olish (Remote → Local)
• origin — remote'ning standart nomi
