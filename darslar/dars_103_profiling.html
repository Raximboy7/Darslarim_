Profiling Nima?

Profiling — bu dasturingiz qancha vaqt va qancha xotira sarflashini o'lchash jarayoni. Ya'ni siz "taxmin qilish" o'rniga aniq raqamlar bilan qaysi qism sekin ishlayotganini topasiz.
Eng muhim qoida: avval o'lcha, keyin tezlat. Ko'pincha dasturchi "menimcha shu joy sekin" deb o'ylaydi, lekin o'lchaganda butunlay boshqa joy sekin chiqadi. Shuning uchun optimizatsiyani har doim o'lchashdan boshlaymiz.

Vaqtni O'lchash: time.time()

Eng oddiy usul — kod boshida va oxirida vaqtni olib, farqini hisoblash. time.time() hozirgi vaqtni soniyalarda qaytaradi.
import time

boshi = time.time() # boshlanish vaqti
jami = 0
for i in range(1000000):
    jami += i
oxiri = time.time() # tugash vaqti

print("Sarflangan vaqt:", oxiri - boshi, "soniya")
# Kodning boshi va oxirida vaqtni o'lchaymiz va farqini chiqaramiz
>>> Sarflangan vaqt: 0.052 soniya

timeit Moduli

timeit — kichik kod bo'lagini ko'p marta takror ishga tushirib, o'rtacha vaqtni aniq o'lchaydi. Bu time.time() dan aniqroq, chunki bir martalik tasodifiy sekinlikni hisobga olmaydi.
import timeit

# number=10000 — kodni 10000 marta ishga tushiradi
vaqt = timeit.timeit(
    "sum(range(100))",
    number=10000
)
print(vaqt, "soniya")
# timeit kodni ko'p marta takrorlab, jami vaqtni qaytaradi
>>> 0.0089 soniya
Maslahat

timeit ni terminalda ham ishlatish mumkin:
python -m timeit "sum(range(100))"

cProfile — Qaysi Funksiya Sekin?

Katta dasturlarda "qaysi funksiya ko'p vaqt yeyapti?" degan savol tug'iladi. cProfile butun dasturni kuzatib, har bir funksiya necha marta chaqirilgani va qancha vaqt sarflaganini ko'rsatadi.
import cProfile

def sekin():
    return sum(range(1000000))

def asosiy():
    for _ in range(5):
        sekin()

cProfile.run("asosiy()")
# cProfile har bir funksiya necha marta chaqirilgani va qancha vaqt olganini ko'rsatadi
>>>   ncalls  tottime  cumtime  function
>>>      5    0.180   0.180  sekin()
>>>      1    0.000   0.180  asosiy()
UstunMa'nosi
ncallsFunksiya necha marta chaqirilgan
tottimeFaqat shu funksiyaning o'zida sarflangan vaqt
cumtimeFunksiya + uning ichidagilar bilan jami vaqt

memory_profiler — Xotirani O'lchash

Ba'zan muammo vaqt emas, balki xotira bo'ladi (dastur juda ko'p RAM yeydi). memory_profiler kutubxonasi har bir qator qancha xotira ishlatganini ko'rsatadi.
# Avval o'rnatamiz: pip install memory_profiler
from memory_profiler import profile

@profile
def katta_royxat():
    a = [i for i in range(1000000)]
    return sum(a)

katta_royxat()
# @profile dekoratori har qator qancha xotira ishlatganini jadval qilib chiqaradi
>>> Line  Mem usage   Increment  Line Contents
>>>   4   38.1 MiB   38.1 MiB   a = [...]
>>>   5   38.1 MiB    0.0 MiB   return sum(a)

Optimizatsiya Yondashuvlari

1
Avval o'lcha, keyin tezlat

Taxmin qilmang. Profiling bilan aniq sekin joyni toping, keyin o'shanga ishlang.

2
To'g'ri data strukturasini tanlang

Qidirish uchun list emas, set/dict ishlating — ular ancha tezroq qidiradi.

3
Takror hisoblardan qoching

Bir xil natijani qayta-qayta hisoblamang — bir marta hisoblab, saqlab qo'ying (kesh).

4
Tayyor funksiyalardan foydalaning

sum(), min(), max() kabilar siz yozgan tsikldan ko'ra tezroq ishlaydi.

Eslab Qoling

• time.time() — eng oddiy vaqt o'lchash
• timeit — kichik kodni ko'p marta o'lchab aniqlik beradi
• cProfile — qaysi funksiya sekin ekanini topadi
• memory_profiler — xotira sarfini o'lchaydi
• Qoida: avval o'lcha, keyin tezlat

Misol 1: list vs set Qidiruv

Bir elementni qidirishda list har bir elementni birma-bir tekshiradi, set esa to'g'ridan-to'g'ri topadi. Element sonini o'zgartiring va qaysi biri tez ekanini ko'ring.

Misol 2: timeit Natijasi

Kodni necha marta (number) takrorlashni tanlang. timeit jami va o'rtacha vaqtni hisoblab beradi.

Misol 3: Optimizatsiya — Oldin va Keyin

Quyida bir xil vazifa ikki xil yozilgan: sekin (har safar qayta hisoblaydi) va tez (natijani keshlaydi). Tezlik farqini ko'ring.

Misol 4: Tezlik Diagrammasi

Ikki yondashuvning tezligini diagramma ko'rinishida ko'rish optimizatsiya foydasini yaqqol ko'rsatadi.
