Ilg'or Git Buyruqlari

Siz allaqachon commit, branch va merge ni bilasiz. Endi tajribali dasturchilar har kuni ishlatadigan 4 ta kuchli buyruq bilan tanishamiz: rebase, cherry-pick, stash va tag. Bular sizning ish jarayoningizni ancha tartibli va qulay qiladi.

1. git rebase — Tarixni Qayta Tartiblash

rebase tarmoqning commitlarini boshqa tarmoq ustiga "ko'chirib" joylashtiradi. Natijada tarix to'g'ri chiziq kabi tekis bo'ladi.

merge dan farqi

BuyruqTarix ko'rinishiQachon
mergeTarmoqlarni birlashtirib qo'shimcha "merge commit" yaratadiJamoa ishida, ommaviy tarmoqlarda
rebaseCommitlarni qayta joylashtirib tekis chiziq hosil qiladiShaxsiy tarmoqni tozalashda
git checkout feature
git rebase main
# feature tarmoqdamiz
# main ustiga qayta joylaymiz — tarix tekis bo'ladi
⚠️ Ogohlantirish

rebase tarixni o'zgartiradi — eski commitlar o'rniga yangi commitlar yaratiladi. Boshqalar bilan baham ko'rilgan (push qilingan) tarmoqlarda rebase qilmang, aks holda jamoangizning tarixi buziladi. Faqat o'zingizning shaxsiy tarmog'ingizda ishlating.

2. git cherry-pick — Bitta Commitni Olish

cherry-pick boshqa tarmoqdagi aniq bitta commitni olib, hozirgi tarmog'ingizga qo'shadi. Butun tarmoqni emas, faqat kerakli o'zgarishni "terib olish" kabi.
git log --oneline
a1b2c3d Xatoni tuzatish
git cherry-pick a1b2c3d
# commit kodini (hash) topamiz
# o'sha bitta commitni hozirgi tarmoqqa olamiz
Qachon kerak?

Masalan, feature tarmog'ida muhim xato tuzatildi. U yerdagi hamma o'zgarishlar tayyor emas, lekin faqat o'sha tuzatishni darhol main ga olishingiz kerak — cherry-pick aynan shu ish uchun.

3. git stash — O'zgarishlarni Vaqtincha Yashirish

Ishlab turibsiz, lekin commit qilishga tayyor emassiz. Shu paytda boshqa tarmoqqa o'tish kerak bo'lib qoldi. stash o'zgarishlaringizni vaqtincha chetga olib qo'yadi va ish papkangizni toza qiladi. Keyin stash pop bilan ularni qaytarib olasiz.
git stash
# ... boshqa ish bajaramiz ...
git stash pop
# o'zgarishlarni yashiramiz, papka toza bo'ladi
# ishimizni qaytarib olamiz — hamma narsa joyida
BuyruqVazifasi
git stashO'zgarishlarni yashiradi
git stash listYashirilgan o'zgarishlar ro'yxati
git stash popOxirgisini qaytarib oladi va ro'yxatdan o'chiradi
git stash dropYashirilganini butunlay o'chiradi

4. git tag — Versiyalarni Belgilash

tag muhim commitlarga nom (yorliq) qo'yadi. Odatda dastur versiyalarini belgilash uchun ishlatiladi, masalan v1.0, v2.3. Shunda istalgan paytda "v1.0 qaysi commit edi?" degan savolga oson javob topasiz.
git tag v1.0
git tag -a v1.0 -m "Birinchi rasmiy versiya"
git push origin v1.0
# oddiy yorliq
# izohli yorliq (tavsiya etiladi)
# yorliqni serverga yuboramiz
Versiya nomi qoidasi

Ko'pchilik v1.0.0 ko'rinishini ishlatadi: katta.o'rta.kichik. Katta o'zgarish → 1-raqam, yangi imkoniyat → 2-raqam, kichik tuzatish → 3-raqam ortadi.

Eslab Qoling

• rebase — tarixni tekis qiladi (ehtiyot bo'ling!)
• cherry-pick — bitta commitni olib keladi
• stash — ishni vaqtincha yashiradi, pop qaytaradi
• tag — versiyalarni belgilaydi (v1.0)
