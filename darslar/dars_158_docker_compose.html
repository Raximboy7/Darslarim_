Docker Compose Nima?

Haqiqiy loyihalar ko'pincha bir nechta konteynerdan iborat bo'ladi: bittasi web ilova, ikkinchisi ma'lumotlar bazasi (database), uchinchisi kesh va hokazo. Ularning har birini alohida docker run bilan ishga tushirish juda noqulay.

Docker Compose — bu bir nechta konteynerni bitta fayl orqali birga boshqarish vositasi. Siz hammasini bitta docker-compose.yml faylida ta'riflaysiz, keyin docker-compose up bilan barchasini bir buyruq bilan ishga tushirasiz.
Sodda taqqoslash

Dockerfile — bitta konteyner uchun "retsept".
Docker Compose — bir nechta konteyner uchun "boshqaruv markazi".

docker-compose.yml Fayli

Docker Compose YAML formatidagi fayldan foydalanadi. YAML — bu sodda, o'qilishi oson matn formati. Unda chekinish (bo'shliqlar) juda muhim — har bir daraja 2 ta bo'shliq bilan ichkariga suriladi.
services:
  web:
    build: .
    ports:
      - "8000:8000"
  db:
    image: postgres:16
# services — barcha konteynerlar ro'yxati
# web va db — ikkita alohida konteyner (service)

Asosiy Kalit So'zlar (services)

Har bir service — bu bitta konteyner. Service ichida konteyner qanday ishlashini ko'rsatuvchi sozlamalar yoziladi.
Kalit so'zVazifasi
imageTayyor obrazdan konteyner yaratadi (masalan postgres)
buildDockerfile dan obraz quradi (. = shu papka)
portsPortlarni ulaydi: "tashqi:ichki"
volumesMa'lumotni saqlash uchun papkani ulaydi
environmentMuhit o'zgaruvchilarini (parol, sozlama) beradi
depends_onQaysi service avval ishga tushishini belgilaydi

ports, volumes, environment

ports — portni ochish

ports:
  - "8000:8000"
# Kompyuterdagi 8000-port konteyner ichidagi 8000-portga ulanadi
# Endi brauzerda localhost:8000 ochish mumkin

volumes — ma'lumotni saqlash

volumes:
  - db_data:/var/lib/postgresql/data
# Database ma'lumotlari konteyner o'chsa ham yo'qolmaydi
# db_data — kompyuterda saqlanadigan joy

environment — sozlamalar

environment:
  - POSTGRES_PASSWORD=parol123
  - POSTGRES_DB=mydb
# Database uchun parol va baza nomini beramiz

depends_on — Tartibni Boshqarish

Web ilova database tayyor bo'lishini kutishi kerak. depends_on Docker'ga aytadi: avval db, keyin web ishga tushsin.
services:
  web:
    build: .
    depends_on:
      - db
# web service db service tayyor bo'lgach ishga tushadi

To'liq Misol: web + database

Quyida haqiqiy loyihaga o'xshash to'liq docker-compose.yml. Bunda bitta web ilova va bitta PostgreSQL database birga ishlaydi.
services:

  web:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgres://db:5432/mydb
    depends_on:
      - db

  db:
    image: postgres:16
    environment:
      - POSTGRES_PASSWORD=parol123
      - POSTGRES_DB=mydb
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:
# web: bizning ilova (Dockerfile dan quriladi)
# db: PostgreSQL database (tayyor obrazdan)
# web db ga ulanadi va u tayyor bo'lishini kutadi

Ishga Tushirish (up / down)

Hamma narsa tayyor bo'lgach, faqat ikki buyruq kerak bo'ladi.
BuyruqVazifasi
docker-compose upBarcha service'larni ishga tushiradi
docker-compose up -dFonda (orqa fonda) ishga tushiradi
docker-compose downBarcha konteynerlarni to'xtatib o'chiradi
docker-compose psIshlayotgan service'larni ko'rsatadi
docker-compose logsLoglarni (chiqishlarni) ko'rsatadi
$ docker-compose up
>>> Creating db ... done
>>> Creating web ... done
>>> web | Server running on http://0.0.0.0:8000
Eslab Qoling

• Docker Compose — bir nechta konteynerni birga boshqaradi
• Hammasi docker-compose.yml (YAML) faylida
• Har service — bitta konteyner (web, db ...)
• docker-compose up — ishga tushiradi, down — o'chiradi

Misol 1: docker-compose.yml Tuzilmasi

Kalit so'zni tanlang — u faylda qayerda va nima uchun ishlatilishini ko'ring.

Misol 2: Service'lar Birga Ishga Tushadi

docker-compose up bosing — avval db, keyin web ishga tushganini kuzating (depends_on tartibi).
db
image: postgres:16
port: 5432
to'xtagan
web
build: .
port: 8000
depends_on: db
to'xtagan

Misol 3: up / down Oqimi

Buyruqni tanlang va Docker Compose nima qilishini terminal ko'rinishida ko'ring.
