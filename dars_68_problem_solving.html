Masala Yechish Strategiyasi

Modul 11 — Algoritmlar | Dars 68

Nega Strategiya Kerak?

Yaxshi dasturchi masalani ko'rishi bilan kod yozishga shoshilmaydi. U avval masalani tushunadi, keyin reja tuzadi, so'ngra kod yozadi. Bu tartibni o'rgansangiz, har qanday yangi masalani — hatto avval ko'rmagan masalani ham — bosqichma-bosqich yecha olasiz.
Asosiy fikr

Masala yechish — bu iste'dod emas, bu ko'nikma. Har bir yechilgan masala keyingisini osonlashtiradi.

Masala Yechishning 5 Bosqichi

1
Tushunish

Nima so'ralyapti? Kirish (input) nima, chiqish (output) nima?

2
Misollar va chekka holatlar

Bir nechta misol o'ylab toping: oddiy holat va edge case (bo'sh, bitta element).

3
Yondashuv (reja)

Avval so'z bilan, qadamlar bilan tushuntiring. Kod emas — fikr.

4
Kod yozish

Rejani Python kodiga aylantiring. Sodda boshlang.

5
Test qilish

Misollaringizni va edge case'larni tekshiring. Xato bo'lsa, tuzating.

LeetCode Tipidagi Masalalar Nima?

LeetCode, HackerRank, Codeforces kabi platformalarda algoritm masalalari bor. Ular odatda: aniq kirish beriladi, aniq natija kutiladi. Masalan: "ro'yxatdagi takror sonni toping" yoki "ikki sonning yig'indisi N ga teng bo'lganini toping". Bu masalalar ish suhbatlarida (intervyu) ham ko'p so'raladi.
Maslahat

Masala matnini diqqat bilan o'qing. Ko'pincha "tartiblangan", "takrorlanmaydigan", "musbat sonlar" kabi muhim shartlar yashiringan bo'ladi.

Vaqt va Xotira Murakkabligi (Big O)

Big O — yechim kirish kattalashganda qancha sekinlashishini baholaydigan o'lchov. U aniq sekundlar emas, balki o'sish tezligini ko'rsatadi.

Oddiy qoida

Sikl ichida sikl bo'lsa — odatda O(n²). Bitta sikl — O(n). Maqsad: sekin yechimni tez yechimga aylantirish.

Brute Force → Optimallashtirish

Brute force ("oddiy kuch") — eng sodda, "boshga kelgan" yechim. U ko'pincha sekin, lekin ishlaydi. Avval shuni yozing, keyin tezroq variantini o'ylang. Ishlamaydigan tez yechimdan, ishlaydigan sekin yechim yaxshiroq.
Tartib

1) Ishlaydigan yechim yoz (brute force) → 2) Test qil → 3) Sekin joyini top → 4) Optimallashtir (masalan, set yoki dict ishlat).

To'liq Misol: Takror Sonni Topish

Masala: Ro'yxat berilgan. Unda kamida bitta son takrorlangan. Birinchi takror sonni toping. Masalan [3, 1, 4, 1, 5] → natija 1.

1-bosqich: Tushunish

Kirish va chiqish
Kirish: sonlar ro'yxati. Chiqish: avval ikkinchi marta uchragan son.

2-bosqich: Misollar va edge case

KirishKutilgan natija
[3, 1, 4, 1, 5]1
[2, 2]2
[]None (takror yo'q)
[7]None (takror yo'q)

3-bosqich: Yondashuv (brute force)

Har bir son uchun, undan keyingi sonlarni ko'rib chiqamiz. Agar bir xil son topilsa — javob shu. Bu sikl ichida sikl, ya'ni O(n²) — sekin.
def takror_brute(lst):
    for i in range(len(lst)):
        for j in range(i + 1, len(lst)):
            if lst[i] == lst[j]:
                return lst[i]
    return None
# Har juftni taqqoslaydi — O(n²), sekin lekin ishlaydi

4-bosqich: Optimallashtirish (tez yechim)

Ko'rgan sonlarni set ga qo'shib boramiz. Yangi son set da bo'lsa — bu takror! Bir marta o'tamiz, ya'ni O(n) — tez.
def takror_tez(lst):
    korilgan = set()
    for son in lst:
        if son in korilgan:
            return son
        korilgan.add(son)
    return None
# set'da qidirish O(1) — umumiy O(n), tez yechim

5-bosqich: Test

>>> takror_tez([3, 1, 4, 1, 5])
>>> 1
>>> takror_tez([])
>>> None
Eslab Qoling

• 5 bosqich: tushun → misol → reja → kod → test
• Edge case'larni doim tekshiring: [], [bitta], manfiy, takror
• Big O — yechim qancha tez o'sishini baholaydi
• Avval brute force, keyin optimallashtiring
